= Parking Lot: Documentation Consolidation Candidates

Purpose:

This note captures documentation issues that were identified during the
Functional/Semantic Model cleanup phase but are intentionally deferred.
The goal is retention of context without introducing new Project Memory
entities, concerns, or governance structures.

== Priority Overview

=== Immediate

* Fix malformed relocation xrefs introduced during the Functional/Semantic
  split.
* Treat as a normal BUG lifecycle item.

=== Next Functional-Model Work

* Functional Model ch04 cleanup.
* Capability-family inventory.
* Family-by-family promotion and meta-complexity review.

=== Later

* Fill Technical Model structure/contracts chapters.
* Relocate technical persistence-boundary mechanisms out of the Semantic Model.

=== Long-Term

* Consolidate preserved migration material in Functional Model ch06.
* Warm Storage archival hygiene.

=== Observe Only

* Prevent ai_info from becoming a parallel documentation system.

== Deferred Topics

=== D1. Functional Model ch06 Duplication

Current state:

* ch05 contains a lightweight overview.
* ch06 contains preserved material relocated from the Semantic Model.

Observation:

Both chapters describe largely the same problem space from different
perspectives.

Assessment:

Not currently harmful.

Reason for deferral:

The preserved material still serves as migration context and historical
traceability. Consolidation should happen only after the Functional Model
has absorbed additional capability families and stabilised.

Priority:

Low.

the Functional/Semantic split was performed.

Observation:

Contract belongs in the Semantic Model.
Mechanism belongs in the Technical Model.

Assessment:

Architecturally correct observation.

Reason for deferral:

Depends on Technical Model maturation.

Priority:

Medium.

developed.

Structural documentation gap.

Medium.

Priority:

Low.

=== D6. Warm Storage Archival Hygiene

Observation:

Assessment:

Pure hygiene.

Priority:

Low.

== Suggested Next Functional-Model Track

When Functional Model work resumes:

1. Create capability-family inventory.
2. Map implemented families to Functional Model coverage.
3. Promote families one by one.
4. For each promotion:

   * identify required concepts,
   * review semantic impact,
   * check for meta-complexity growth,
   * prefer reuse of existing concepts over introduction of new vocabulary.

Expected outcome:

Functional coverage catches up with implementation while providing a
continuous stress test for semantic-model complexity growth.
