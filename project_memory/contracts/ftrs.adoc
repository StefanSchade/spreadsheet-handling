= Table Contract: ftrs
:toc:

== Purpose

`ftrs` records FTR-level work items relevant to project memory and review/finding tracking.

This table is a lightweight index, not a full backlog replacement.

== Fields

[cols="1,1,3"]
|===
| Field | Required | Contract

| `id`
| yes
| Stable FTR id.

| `title`
| yes
| Human-readable title.

| `status`
| yes
| Current status. See <<statuses>>.

| `summary`
| yes
| One-sentence or short-paragraph summary.

| `owner`
| yes
| Owning project or area.

| `phase`
| no
| Phase or priority lane such as `P1`, `P5`.

| `priority`
| no
| Priority label. Current data uses `normal`.

| `created`
| no
| Creation date or timestamp.

| `source_path`
| no
| Source FTR artifact or canonical path.

| `current_relevance`
| yes
| Current relevance. See <<current-relevance>>.
|===

== Statuses [[statuses]]

Observed values include:

* `draft`
* `open`
* `in_progress`
* `done`
* `closed`
* `superseded`

Only `draft` and `open` are present in the current sample data.

== Current Relevance [[current-relevance]]

Observed values include:

* `current`
* `historic`
* `historical`

Prefer `current` for active rows. Preserve existing historical spelling until a dedicated normalization slice exists.

== Non-Goals

* No full FTR body duplication.
* No replacement for backlog files.
