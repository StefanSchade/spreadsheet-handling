:toc:
:toclevels: 2

== Purpose

This directory documents the lightweight table contracts for the `project_memory/canonical/*.json` tables.

The contracts are intentionally AsciiDoc-first. They define field meaning, granularity, controlled values, and non-goals. JSON Schema validation can be added later once the model is stable.

== Contract Policy

* Canonical data remains in JSON tables.
* Contract files do not duplicate individual records.
* Controlled values should be changed here before being used broadly in data.
* The model stays flat and spreadsheet-roundtrip-friendly.
* Contract text may be used by agents as implementation guidance, but the JSON files remain the data source.

== Granularity Policy

The project-memory model intentionally separates coarse steering memory from detailed traceability.

* Reviews, findings, FTRs, and Git history remain the detailed evidence layers.
* Concerns and concern events are steering-level memory objects.
* A concern event may summarize a review, finding, decision, release event, manual interpretation, or activity wave.
* Commit references on activity events are anchors, not a complete commit-level provenance model.
