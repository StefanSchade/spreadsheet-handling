= Table Contract: ftr_dependencies
:toc:

== Purpose

`ftr_dependencies` records flat FTR-to-FTR relations.

The table exists to keep dependency information spreadsheet-roundtrip-friendly instead of nesting dependency lists inside `ftrs.json`.

== Fields

[cols="1,1,3"]
|===
| Field | Required | Contract

| `id`
| yes
| Stable dependency id.

| `source_ftr_id`
| yes
| The FTR that has the relation.

| `target_ftr_id`
| yes
| The FTR being referenced.

| `relation`
| yes
| Controlled value. See <<relations>>.

| `status`
| yes
| Controlled value. See <<statuses>>.

| `note`
| no
| Short explanation.
|===

== Relations [[relations]]

[cols="1,3"]
|===
| Value | Meaning

| `depends_on`
| `source_ftr_id` depends on `target_ftr_id`.

| `related_to`
| Reserved for future use if non-blocking relations are needed.

| `supersedes`
| Reserved for future use if FTR lineage needs explicit modeling.
|===

== Statuses [[statuses]]

[cols="1,3"]
|===
| Value | Meaning

| `active`
| Relation is current.

| `historic`
| Relation is retained historically.

| `superseded`
| Relation was replaced by a later relation.
|===

== Non-Goals

* No nested dependency arrays in `ftrs.json`.
* No graph semantics beyond the explicit relation value.
