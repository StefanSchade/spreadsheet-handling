= Table Contract: review_sets
:toc:

== Purpose

`review_sets` defines the coarse groupings of review documents, mirroring the
storage topology under `docs/cold_storage/reviews/`.

A review set is the stable navigation anchor for concern events that describe
the steering impact of a whole review wave. It provides a deterministic path
back to the source review documents without requiring knowledge of individual
review IDs.

== Design Rule: Storage Topology

Review set groupings are derived directly from the directory structure under
`docs/cold_storage/reviews/`:

----
docs/cold_storage/reviews/
  global_reviews/
    review_001/   -> REVSET-GLOBAL-REVIEW-001
    review_002/   -> REVSET-GLOBAL-REVIEW-002
    review_003/   -> REVSET-GLOBAL-REVIEW-003
  topic_reviews/
    domain_reviews/      -> REVSET-TOPIC-DOMAIN-REVIEWS
    fk_helper_reviews/   -> REVSET-TOPIC-FK-HELPER-REVIEWS
    meta_reviews/        -> REVSET-TOPIC-META-REVIEWS
    package_reviews/     -> REVSET-TOPIC-PACKAGE-REVIEWS
----

Do not invent semantic regroupings. The storage structure is the authoritative
grouping because it provides a deterministic, human-navigable path back to
source material.

== Navigation Model

----
Concern
  -> Concern Event  (source_type=review_set, source_id=REVSET-*)
     -> Review Set  (review_sets.json)
        -> Reviews  (reviews.json, filtered by review_set_id=REVSET-*)
           -> Review documents  (evidence_path)
----

== Fields

[cols="1,1,3"]
|===
| Field | Required | Contract

| `id`
| yes
| Stable review set id. Pattern: `REVSET-<CATEGORY>-<NAME>`.
  Categories: `GLOBAL` for global review waves, `TOPIC` for topic-scoped reviews.

| `title`
| yes
| Human-readable name for the review set.

| `category`
| yes
| Controlled value: `global` or `topic`.

| `storage_path`
| yes
| Directory path under the repository root. Must match an actual directory in
  `docs/cold_storage/reviews/`.

| `notes`
| no
| Brief characterization of what the review set covers.
|===

== Relationship to reviews.json

`reviews.json` carries a `review_set_id` field that references `review_sets.id`.
Every review should belong to exactly one review set.

== Non-Goals

* This table does not replace `reviews.json`.
* This table does not replace individual review documents.
* This table does not create new semantic groupings beyond storage topology.
* Do not create one concern event per review document.
  Events point to review sets, not individual reviews.
