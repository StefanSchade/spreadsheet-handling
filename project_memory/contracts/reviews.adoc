= Table Contract: reviews
:toc:

== Purpose

`reviews` records review artifacts and their project-level takeaways.

The table is an index and steering summary, not a replacement for the full review files.

== Fields

[cols="1,1,3"]
|===
| Field | Required | Contract

| `id`
| yes
| Stable review id.

| `title`
| yes
| Human-readable review title.

| `type`
| yes
| Controlled value. See <<types>>.

| `set`
| no
| Review set/group such as `REVIEW-001`, `REVIEW-002`, `TOPIC-META`.

| `focus`
| yes
| Short focus key.

| `date`
| yes
| Review date `YYYY-MM-DD`.

| `reviewer`
| no
| Reviewer or review artifact source.

| `evidence_path`
| yes
| Path to source review artifact.

| `Verdict`
| no
| Legacy/case-variant verdict field. Prefer `verdict` for new data.

| `verdict`
| no
| Current verdict text.

| `current_relevance`
| yes
| Current relevance. See <<current-relevance>>.

| `project_takeaway`
| no
| Coarse project-level takeaway.

| `takeaway_confidence`
| no
| Confidence in the project takeaway. See <<takeaway-confidence>>.
|===

== Types [[types]]

[cols="1,3"]
|===
| Value | Meaning

| `global_review`
| Broad review across major architecture/project areas.

| `topic_review`
| Focused review of a topic, feature family, or cleanup wave.
|===

== Current Relevance [[current-relevance]]

Observed values include:

* `current`
* `historic`

Preserve current spelling unless a dedicated normalization slice changes it.

== Takeaway Confidence [[takeaway-confidence]]

[cols="1,3"]
|===
| Value | Meaning

| `high`
| Takeaway is strongly supported by the review artifact.

| `medium`
| Takeaway is useful but somewhat interpretive.

| `low`
| Takeaway should be manually reviewed before reuse.
|===

== Legacy Field Note

The current table contains both `Verdict` and `verdict`. New rows should prefer `verdict`; do not remove or rename `Verdict` without a dedicated migration slice.

== Non-Goals

* No full review duplication.
* No replacement for finding extraction.
