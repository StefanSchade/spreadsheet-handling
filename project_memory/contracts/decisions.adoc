= Table Contract: decisions
:toc:

== Purpose

`decisions` records explicit project-memory or architecture/modeling decisions that should remain visible across later work.

Decision rows are lightweight ADR-like anchors, not full ADR documents.

== Fields

[cols="1,1,3"]
|===
| Field | Required | Contract

| `id`
| yes
| Stable decision id. Existing pattern: `DEC-*`.

| `key`
| yes
| Stable short key for lookup and references.

| `title`
| yes
| Human-readable decision title.

| `date`
| yes
| Decision date. Existing rows use timestamp strings; future rows should prefer ISO date or consistent project timestamp format.

| `status`
| yes
| Controlled value. See <<statuses>>.

| `owner`
| yes
| Owning project or area.

| `target_type`
| yes
| Type of target the decision governs, such as `ftr`, `release`, `table`, `model`, or `contract`.

| `target_id`
| yes
| Identifier of the governed target.

| `rationale`
| yes
| Why this decision was made.

| `source_path`
| no
| Path to the canonical source or evidence path.
|===

== Statuses [[statuses]]

[cols="1,3"]
|===
| Value | Meaning

| `active`
| Current decision.

| `superseded`
| Replaced by a later decision.

| `retired`
| No longer applicable.
|===

== Non-Goals

* No detailed ADR replacement.
* No duplicate storage of full decision discussions.
