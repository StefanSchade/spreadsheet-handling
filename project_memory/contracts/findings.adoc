= Table Contract: findings
:toc:

== Purpose

`findings` records concrete review findings and their current disposition.

Findings are the detailed evidence layer below concern signals. Concern signals may summarize findings, but should not replace this table.

== Granularity Rule

One row represents one finding or one explicitly retained non-finding row from a review artifact.

Do not collapse findings here merely because a concern signal summarizes them. Use `current_relevance`, `status`, and links to represent closure or supersession.

| Human-readable affected code/model area.

| no

== Severities [[severities]]

Observed values include:

* `critical`
* `major`
* `medium`
* `minor`
* `low`
* `should_fix`
* `nice_to_have`
* `none`

Do not normalize source severity aggressively without preserving source meaning.

== Priority Buckets [[priority-buckets]]

Observed values include:

* `critical`
* `should_fix`
* `architecture`
* `bug`
* `test_gap`
* `hygiene`
* `analysis`
* `watch`
* `nice_to_have`
* `non_finding`

`priority_bucket` is project-level steering, not source severity.

== Statuses [[statuses]]

[cols="1,3"]
|===
| Value | Meaning

| `open`
| Active and unresolved.

| `in_arbeit`
| Active work or tracked work is in progress / intended.

| `done`
| Resolved and retained historically.

| `resolved`
| Resolved; usually used for follow-up or carry-over rows.

| `accepted`
| Accepted as deliberate or not requiring action.

| `superseded`
| No longer the active tracker; replaced or absorbed elsewhere.
|===

== Current Relevance [[current-relevance]]

[cols="1,3"]
|===
| Value | Meaning

| `current`
| Still relevant for current work or steering.

| `historical`
| Retained for history, not a current driver.

| `partial`
| Partially relevant or partially mitigated.
|===

== Description Confidence [[description-confidence]]

[cols="1,3"]
|===
| Value | Meaning

| `high`
| Description is well supported by source fields.

| `medium`
| Description is plausible but may need later review.

| `low`
| Description is weak or should be manually checked.
|===

== Non-Goals

* No replacement for original review artifacts.
* No forced severity normalization across unrelated review styles.
* No automatic task creation from every finding.
