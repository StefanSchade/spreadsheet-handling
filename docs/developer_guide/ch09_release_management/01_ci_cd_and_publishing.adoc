=== GitHub Actions Pipeline

The workflow is defined in `.github/workflows/release.yml`.

==== Trigger Conditions

[source,yaml]
----
on:
  push:
    branches: [ main ]      # runs the test job on every push to main
    tags: [ 'v*' ]          # additionally runs the publish job on version tags
----

==== Jobs

[cols="1,1,3"]
|===
| Job | Runs when | What it does

| `test`
| Every push to `main` and every `v*` tag
| Installs deps with `pip install -e '.[dev]'`, runs `make test`

| `publish`
| `v*` tag only, after `test` succeeds
| Builds sdist + wheel with `python -m build`, publishes to PyPI
|===

=== Dynamic Version from Tag

[source,yaml]
----
- uses: actions/checkout@v4
  with:
    fetch-depth: 0    # CRITICAL — ensures all tags are present
----

Without `fetch-depth: 0` the checkout is shallow and `setuptools-scm` cannot
find the most recent tag, causing the version to fall back to `0.0.0`.

=== PyPI Trusted Publisher

The publish job uses OIDC-based Trusted Publisher authentication — no PyPI
API token stored as a secret.

[source,yaml]
----
permissions:
  id-token: write   # required for OIDC token exchange
  contents: read

- name: Publish to PyPI
  uses: pypa/gh-action-pypi-publish@release/v1
----

The matching Trusted Publisher entry must be configured on PyPI under the
project's publishing settings (publisher: GitHub Actions, repository owner,
repository name, workflow filename).

=== Release Process

. Ensure `dev` is stable: `make test` green, all relevant FTRs done.
. Merge `dev` → `main` (rebase and merge).
. Create and push an annotated version tag:
+
[source,console]
----
git tag -a v1.2.3 -m "Release 1.2.3"
git push origin v1.2.3      # developer executes
----
. GitHub Actions picks up the tag, runs tests, builds the distribution, and
  publishes to PyPI automatically.
. Verify the release at https://pypi.org/project/spreadsheet-handling/.

=== Consumer Repo Integration

The `spreadsheet-handling-demo` repo installs the library from PyPI
(or from a local editable install for development).  To bind it to the local
source during development:

[source,console]
----
make setup-lib-local    # in the demo repo
----

This wires `pip install -e ../spreadsheet-handling` so the demo always
reflects the current working state of the library without a PyPI release.
