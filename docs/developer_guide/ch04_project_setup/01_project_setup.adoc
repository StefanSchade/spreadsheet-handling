=== pyproject.toml

The project uses `setuptools` with `setuptools-scm` for build and versioning.
Hatchling is explicitly *not* used.

==== Key sections

[source,toml]
----
[project]
name = "spreadsheet-handling"
requires-python = ">=3.10"
dynamic = ["version"]           # version derived from git tags by setuptools-scm

[build-system]
requires = ["setuptools>=68", "setuptools-scm>=8", "wheel"]
build-backend = "setuptools.build_meta"

[tool.setuptools]
package-dir = {"" = "src"}      # src layout

[tool.setuptools_scm]
version_scheme = "guess-next-dev"
local_scheme   = "node-and-date"
write_to       = "src/spreadsheet_handling/_version.py"
fallback_version = "0.0.0"
----

==== src/ layout

All installable code lives under `src/spreadsheet_handling/`.
This separation prevents accidental imports of uninstalled source from the
working directory during tests.

==== Dynamic versioning with setuptools-scm

The version is derived from the most recent git tag at build time.
`setuptools-scm` writes `src/spreadsheet_handling/_version.py` — this file
is gitignored and regenerated on every build.

* A tagged commit (`v1.2.3`) produces version `1.2.3`.
* A commit after a tag produces `1.2.4.dev3+g<hash>.<date>`.
* Without any tag the fallback is `0.0.0`.

IMPORTANT: `git fetch --tags` (or `actions/checkout@v6` with `fetch-depth: 0`)
is required for correct version resolution.  Missing tags silently fall back
to `0.0.0`.

==== Runtime Dependencies

[cols="1,2"]
|===
| Package | Purpose

| `pandas >= 2.0`
| DataFrame-based data carrier between all layers (`Frames = Dict[str, DataFrame]`)

| `openpyxl >= 3.1`
| XLSX read/write backend

| `XlsxWriter >= 3.0`
| Alternative XLSX write backend (formula-rich output)

| `odfpy >= 1.4.1`
| ODS (LibreOffice Calc) backend

| `pyyaml >= 6`
| YAML config and pipeline definition parsing
|===

==== Dev Dependencies

Installed via `pip install -e '.[dev]'` (done automatically by `make setup`):

* `pytest`, `pytest-cov` — test runner and coverage
* `ruff` — fast linter
* `black` — formatter
* `autopep8` — supplementary formatter

==== Entry Points

Three CLI scripts are registered:

[source,toml]
----
[project.scripts]
sheets-run                  = "spreadsheet_handling.cli.apps.run:main"
sheets-example-json-to-xlsx = "spreadsheet_handling.cli.apps.example_json_to_xlsx:main"
sheets-example-xlsx-to-json = "spreadsheet_handling.cli.apps.example_xlsx_to_json:main"
----

After `pip install` (or `pip install -e .`) these commands are available on `$PATH`.
