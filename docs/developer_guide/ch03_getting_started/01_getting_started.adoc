=== Prerequisites

* Python >= 3.10 on a POSIX/Linux system
* `make`, `git`
* A virtual env is managed entirely by the Makefile — do not create one manually

=== Clone and Bootstrap

[source,console]
----
git clone https://github.com/StefanSchade/spreadsheet-handling.git
cd spreadsheet-handling
make setup          # creates .venv, installs all dev deps from pyproject.toml
----

`make setup` is idempotent and stamp-driven: it only re-installs when
`pyproject.toml` has changed since the last run.

=== Verify the Installation

[source,console]
----
make test           # unit + integration + architecture tests (active suite)
make run            # demo roundtrip on the bundled example
----

If either fails after a clean clone, `make doctor` shows the environment and
stamp state.  A reliable recovery path is:

[source,console]
----
make clean-stamps && make setup
----
