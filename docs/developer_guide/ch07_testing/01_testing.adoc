=== Test Layers

[cols="1,2,2"]
|===
| Location | Purpose | Marker

| `tests/unit/`
| Fast, in-memory; no filesystem unless testing IO
| `@pytest.mark.unit`

| `tests/integration/`
| End-to-end with real files
| `@pytest.mark.integ`

| `tests/architecture/`
| Import-graph and structural guardrails
| `@pytest.mark.arch`

| `tests/legacy_pre_hex/`
| Quarantined pre-refactor tests; never run by default
| `@pytest.mark.prehex`
|===

=== Running Tests

[source,console]
----
make test                             # active suite: unit + integ + arch
make test-verbose                     # same, with streaming logs
make test-lastfailed                  # re-run last failures
make test-one TESTPATTERN="expr"      # keyword filter
make test-file FILE=path/to/test.py   # single file
make test-node NODE=path::test_name   # single test
make test-unit
make test-integ
make test-arch
make test-prehex                      # quarantined suite, explicit opt-in
make test-full                        # everything including slow
----

Override marker expression: `make test MARK="not slow"` or `make test MARK=` (no filter).
Pass extra flags: `make test PYTEST_OPTS="--maxfail=1 -x"`.

=== FTR Traceability Markers

Every test file declares its FTR via module-level `pytestmark` (after imports,
before any test functions):

[source,python]
----
import pytest

pytestmark = pytest.mark.ftr("FTR-EXAMPLE-NAME")
----

Multiple FTRs: use a list.  Collect by name: `pytest -k "FTR_NAME"`.

=== Test Naming

* File: `test_<subject>_<aspect>.py` — e.g. `test_ir_writepath_semantics.py`
* Function: `test_<expected_behavior>` — reads like the invariant
* Architecture tests name the boundary: `test_rendering_layer_does_not_import_adapters`
* Avoid: `test_misc.py`, vague names like `happy`, `basic`, `works`

=== Coverage

[source,console]
----
make coverage           # terminal report
make coverage-html      # HTML report in build/htmlcov/
----

`pytest-cov` is configured in `pyproject.toml`:
branch coverage enabled, missing lines shown, `_version.py` and `__init__.py` omitted.

=== TDD and Test Quality

* Characterization tests before reshaping dense existing logic
* Prefer small in-memory DataFrames; assert schema, row count, and keys explicitly
* Tests must be hermetic and fast; no network access
* Use filesystem only when the purpose of the test is IO

=== Architecture Guardrail Tests

Tests in `tests/architecture/` enforce layer boundaries automatically.
When a guardrail test fails, identify which architectural boundary was crossed
and move the behavior to the correct layer — do not move code solely to silence the test.

=== Quarantine Lifecycle

Transitions between test states must be deliberate:

* *Quarantined → Active*: validate against current architecture, place in correct layer
* *Quarantined → Archived*: prefer documented deletion; record why behavior was retired
* *Active → Quarantined*: only during active redesign, with a short written rationale
