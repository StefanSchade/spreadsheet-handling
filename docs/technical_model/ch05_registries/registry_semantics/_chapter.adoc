== Registry Semantics

This subsection provides the conceptual foundation for the vocabulary used by
the registries in this chapter — most importantly the meta registry, but the
same terms also appear in surrounding architecture docs and in agent-facing
material under `docs/ai_info/`.

The terms (`canonical`, `derived`, `carrier`, `model`, `presentation`,
`workbook_identity`; `original` vs `derived` data) had been used as
load-bearing distinctions without explicit foundations. The 2026-05-18
REVIEW-002 flagged this as residual semantic density. The files below define
the foundation explicitly and identify the design questions that remain open.

[NOTE]
====
The live `meta_registry.yaml` continues to use the older classification
labels (`meta_canonical` / `meta_rendering` layers; `canonical_meta` /
`derived_operational_view` / `carrier_artifact` / `read_path_hint`
classifications). Re-encoding the YAML to use the foundation vocabulary
directly is *not* part of this foundation work — it is deferred to a separate
FTR. The mapping between the foundation vocabulary and the live registry
labels is documented in `02_meta_layer.adoc` and `03_meta_classification.adoc`.
====
