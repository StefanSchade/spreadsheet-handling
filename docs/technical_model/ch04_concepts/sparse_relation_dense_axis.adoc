[[concept-sparse-relation-dense-axis]]
== Sparse Relation and Dense Axis Reconstruction

_Architectural rule for the boundary between sparse relations and dense
matrices: the system never reconstructs missing rows/columns/combinations by
guessing — reconstruction is allowed only from an explicitly configured or
persisted axis contract._

Status: the principle is in force now. `contract_xref` supports the explicit
`dense_axes` reconstruction mode for generic XRef/crosstable matrix shape
materialization. `CELL-CODEC` does not own dense-axis reconstruction, and
`COMPACT-MULTIAXIS` still rejects `dense_axes` until a narrow composed-transform
follow-up wires the XRef primitive through.

=== Why this matters

Compact spreadsheet transforms move between dense matrix shapes and sparse
relations. An empty cell may be intentionally irrelevant, or it may be part of
the matrix contract that must survive a roundtrip. If the system tried to "fill
in" the dense shape by inference, results would depend on incidental workbook
layout and would not be deterministic. So the rule is: no heuristic
reconstruction; explicit configuration over inference; granular modes instead of
one implicit global roundtrip semantic.

=== The principle

The system must not reconstruct missing matrix rows, columns, or combinations by
guessing from visible labels, table positions, similar names, grouped headers,
or surrounding workbook layout. Reconstruction is allowed only when the complete
axis contract is available from one of:

* explicit pipeline or metadata configuration, or
* persisted contract metadata written by an earlier deterministic step.

If neither source exists, sparse relations remain sparse.

=== Cell-codec boundary

`CELL-CODEC` owns cell-value parsing and serialization only. `drop_empty: true`
is a valid *sparse default*: empty compact cell values produce no decoded rows,
and empty-only groups can disappear from the decoded relation — but this is
strict-lossless *only when empty cells are semantically irrelevant*. For strict
matrix-shape roundtrip without later reconstruction, use `drop_empty: false` so
empty cell values remain explicit rows.

=== Dense-axis contract

Sparse relations may contract back into dense matrices using *explicitly
configured* axis sources:

[source,yaml]
----
dense_axes:
  rows_from:
    frame: Products
    key: product_id
  columns_from:
    frame: Markets
    key: market_id
----

Semantics: the relation may be sparse; the dense row/column space comes from the
configured axis sources; missing combinations receive the configured
`fill_value`; axis identity comes from keys, not display labels; and the mode
must be explicitly configured or loaded from persisted contract metadata.

Current implementation scope: `contract_xref` materializes rows and columns
from explicit sources and stores resolved axis identities in `xref_crosstable`
metadata. `expand_xref` preserves that metadata when a matching transform
configuration exists.

=== Non-goals

* No guessing from visible spreadsheet labels.
* No reconstruction from table position or workbook proximity.
* No name-similarity matching between relation columns and other frames.
* No implicit switch from sparse-relation semantics to dense-matrix semantics.
