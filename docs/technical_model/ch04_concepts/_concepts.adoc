== Concept Index

Foundational technical concepts of the spreadsheet-handling project.
Compact agent-facing distillations of some concepts live under `docs/ai_info/`
and point back here when detail is needed.

Originally migrated from `docs/internal_guide/concepts/`.
Use-case concepts live in `docs/functional_model/ch04_use_case_concepts/`.

=== Reading order for a new reader

. link:metadata_canonical_vs_derived.adoc[Canonical vs. Derived Metadata] — the
  vocabulary every other concept reuses (`canonical` ≠ "input"; `derived` ≠
  "freshly computed this run").
. link:roundtrip_spreadsheet_layout.adoc[Round-trip Strategy for Spreadsheet
  Layouts] — the round-trip contract every feature serves (data + metadata;
  style normalized; flat-table vs. named-range layouts).
. link:fk_helper_metadata.adoc[FK Helper Metadata] — the most concrete feature;
  where the canonical/derived split first bites.
. link:compact_transforms.adoc[Compact Transforms] — the bijective transform
  family (`expand_xref`/`contract_xref`, `expand_compact`/`contract_compact`).
. link:crosstab_binary_xref.adoc[Crosstab for Editing Binary Cross References] —
  the first worked slice of the compact-transform chain.
. link:sparse_relation_dense_axis.adoc[Sparse Relation and Dense Axis
  Reconstruction] — the no-heuristic-reconstruction boundary for those
  transforms.
. link:partition_by_domain.adoc[Partition by Domain] — an independent feature:
  split/merge by component to reduce Git merge conflicts.
. link:pipeline_step_registration.adoc[Pipeline Step Registration]
  (transitional) — the mechanics: how YAML step names become executable steps.

=== Index

[cols="2,4", options="header"]
|===
| Concept | Summary

| link:metadata_canonical_vs_derived.adoc[Canonical vs. Derived Metadata]
| `canonical` = stable spreadsheet-neutral meaning; `derived` = materialized,
  carrier-near view. The distinction is functional, not temporal.

| link:roundtrip_spreadsheet_layout.adoc[Round-trip Strategy for Spreadsheet Layouts]
| Round-trip guaranteed for data + metadata (style normalized); two layout
  modes — flat anchored tables, or named ranges.

| link:fk_helper_metadata.adoc[FK Helper Metadata]
| Redundant spreadsheet-only columns materializing referenced-table fields next
  to an FK column; helper *capability* is canonical, materialized helper
  *columns* are derived.

| link:compact_transforms.adoc[Compact Transforms]
| Bijective transforms between normalized relations and a compact human-editable
  table where logical layers are encoded as tokens in a cell.

| link:crosstab_binary_xref.adoc[Crosstab for Editing Binary Cross References]
| First slice of the compact-transform chain: edit a binary relation as an
  `X`-or-empty matrix while storage stays a normalized PK-pair link table.

| link:sparse_relation_dense_axis.adoc[Sparse Relation and Dense Axis Reconstruction]
| No heuristic reconstruction: sparse relations stay sparse unless the full axis
  contract is explicitly configured or persisted.

| link:partition_by_domain.adoc[Partition by Domain]
| Split frames into deterministic per-component slices before persistence;
  recompose losslessly on read; fewer Git merge conflicts.

| link:pipeline_step_registration.adoc[Pipeline Step Registration]
| Keep a factory-based step registry as the extension point; add a generic
  binding path for ordinary domain-addressable steps. (Transitional.)
|===
