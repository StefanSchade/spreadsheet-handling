== Glossary

This chapter builds the shared language for the technical model.

Focus:

* stable terms for modeling discussions,
* precise distinction between domain, pipeline, render, and spreadsheet terms,
* terminology around metadata and registries,
* less implicit meaning in reviews, tickets, and implementation work.

The glossary belongs near the registries because registry entries depend on
consistent wording for producers, consumers, metadata roots, and projection
boundaries.
