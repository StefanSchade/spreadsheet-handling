=== ADR-DOMAIN-OWNS-VALIDATION-WRITERS-STAY-DUMB -- Domain pipeline owns cleanup and validation

Status: accepted
Date: 2026-06-03
Related backlog: BUG-HELPER-CLEANUP-CONTRACT-P4A

==== Context

`BUG-HELPER-CLEANUP-CONTRACT-P4A` exposed a full-cycle failure: after
`BUG-RUNTIME-META-PERSISTENCE-BOUNDARY-P4A` correctly stopped persisting
transient `_meta.derived` provenance, `apply_derived_column_policy` could no
longer identify helper columns during canonical workbook reimport. FK helper
columns such as `_places_name` reached canonical JSON.

One possible fix would be to filter helper columns in JSON, YAML, XLSX, or ODS
writers, or at the persistence boundary. That would hide the symptom but move
semantic policy into serialization.

==== Decision

Semantic cleanup and validation live in the domain pipeline, triggered by
explicit pipeline steps. Writers serialize the frames they receive.

For helper cleanup this means:

* `apply_derived_column_policy` removes declared helper columns before the
  writer is called.
* `validate_references` can assert the invariant with `no_helper_columns`.
* The persistence boundary may project runtime metadata to a persistable
  contract, but it does not filter payload columns.

==== Alternatives Considered

. *Filter helper columns in writers.* Rejected. It gives writers domain
  knowledge, breaks reader/writer symmetry, and makes it harder to write
  intentionally pathological state for debugging.
. *Filter helper columns at the persistence boundary.* Rejected. The boundary
  owns metadata projection, not payload semantics.
. *Add a reconstruction step before cleanup.* Rejected. The cleanup step can
  resolve the needed drop identity directly from durable helper metadata.
. *Re-persist `_meta.derived` for FK helpers.* Rejected. That would re-open
  the runtime metadata persistence bug this fix must preserve.

==== Rationale

Readers and writers should stay mechanically symmetric: they move carrier
content into and out of frames. Domain steps decide which payload columns are
canonical, which are projections, and which invariants must hold before write.
The project already has a findings-frame validation primitive; using it avoids
duplicating validation semantics in adapter code.

==== Consequences

* Pipelines must declare cleanup when canonical output should not contain
  helper columns.
* Pipelines can add `validate_references` / `no_helper_columns` after cleanup
  to fail or report if helper columns remain.
* Writers remain useful for debug and inspection output because they do not
  silently remove semantic payload.
* Future helper lifecycle work may refine durable helper metadata, but it
  should keep the cleanup/validation responsibility in the domain pipeline.
