=== ADR-META-BLOB-SERIALIZATION -- Serialize workbook meta carrier as JSON

Status: accepted
Date: 2026-05-12
Related review findings: IR-008, A-004
Related FTR: FTR-REVIEW-001-META-BLOB-JSON-P3

==== Context

The hidden spreadsheet `_meta` sheet stores canonical workbook metadata in a
`workbook_meta_blob` carrier cell. Older write paths serialized that blob with
Python `repr` via `str(dict)`. The read path already tried `json.loads` first
and then fell back to `ast.literal_eval`.

That mixed format is fragile. In particular, `float("nan")` can be written as a
Python-literal-looking token that is not valid JSON and can be misread by the
fallback path. The carrier encoding should not change canonical metadata
semantics or require a new metadata model.

==== Decision

New writes serialize `workbook_meta_blob` with deterministic `json.dumps`:
`ensure_ascii=False`, `sort_keys=True`, and compact separators. Python's JSON
reader/writer remains responsible for existing numeric semantics such as NaN.

Read paths remain JSON-first. `ast.literal_eval` is retained only as a legacy
fallback for workbooks written before this decision.

==== Consequences

* Newly written workbook meta carriers are deterministic JSON strings, not
  Python `repr`.
* Existing workbooks that contain legacy Python-literal blobs remain readable.
* The canonical `_meta` structure is unchanged.
* This decision does not introduce a meta engine, schema validator, or new
  public Python API.
