== Verdict

*CELL CODEC PROJECTION-FAMILY SLICE NOT ACCEPTED.*

R002-B1 and R002-B2 are closed. The public signatures no longer expose the
private sentinel, the one-character grammar is bijective for its accepted
string vocabulary, and the corrected live documentation describes those
contracts accurately.

One important R002-I3 residual remains. The effective-default comparator can
treat a non-default, structurally meaningful NumPy collection as equal to a
`None` default. For example, `allowed_tokens=np.array([None], dtype=object)`
alongside `codec_intent` silently selects the position path and ignores the
historical collection instead of rejecting mixed intent. Direct calls and
pipeline binding reproduce the same bypass. This is not a blocking data-loss
defect, but it violates the explicit mutual-exclusion contract and the exact
array-like review requirement, so primitive-family acceptance remains
withheld.

No accepted metadata, separator, typing, physical-label, or legacy-path
architecture needs to be reopened.

== R002-B1 closure -- generated output labels

Both historical paths validate their source-addressing and generated-output
selectors immediately after physical-frame validation:

* decode validates source `value` and generated `code`; and
* encode validates source `code` and generated `value`.

Independent probes passed list, `pd.NA`, `None`, and a multi-element NumPy
array as each generated label. Every case raised a deterministic Cell Codec
`ValueError` containing `cannot address a scalar column`; no raw `TypeError`,
ambiguous-truth error, or pandas-normalized `NaN` label occurred. Validation
runs before collision membership, record construction, and output DataFrame
construction. Ordinary string and numeric generated labels remain supported.

For every rejection the caller's mapping gained no output, no
`_meta.cell_codecs` or `_meta.pipeline_cleanup` appeared, the source DataFrame
was unchanged, and nested sentinel metadata was unchanged. R002-B1 is closed.

== R002-B2 closure -- position grammar

`_position_intent` now rejects a separator unless it is a string of length
exactly one. The exact Review 002 counterexample
(`separator="--"`, `absent_value="~"`, values `A-`, `B`, `C`) therefore fails
at declaration validation before row processing or output.

Independent probes also verified:

* empty and other multi-character separators reject;
* separator equal to the absent marker rejects;
* an absent marker containing the separator rejects;
* a real token containing the separator or equalling the absent marker
  rejects;
* valid one-character separators preserve ordinary and numeric-looking
  strings over multiple rows and columns; and
* empty and missing values both intentionally recreate as `""`.

The narrowed proof is sound. Every accepted real token and absent marker lacks
the one separator code point, so joining cannot create a new occurrence at a
boundary and splitting on that code point recreates the same ordered tokens.
An exhaustive short ordinary-string probe found no counterexample. No escaping
or multi-character support is implied. R002-B2 is closed.

== R002-I3 signature and dispatch assessment

`inspect.signature` exposes normal annotations and defaults, including
`mode: str | None = None`, `delimiter: str = "-"`, `strip: bool = False`,
`code: str = "code"`, and `value: str = "value"`. Source search,
introspection, and error output contain no `_UNSET` or `<unset>`. Historical-
only calls and position-only calls pass; `compact_multiaxis` continues to use
the explicit historical branch.

Default-equal scalar values (`delimiter="-"`, `strip=False`, default
`code`/`value`, `drop_empty=True`, and `mode=None`) may accompany
`codec_intent`. Non-default delimiter, `strip=True`, explicit mode,
`group_by`, passthrough declarations, allowed collections, legend reference,
canonical order, normalization, non-default `code`/`value`, and
`drop_empty=False` reject deterministically. Multi-element ndarray comparison
also safely becomes a conflict instead of raising an ambiguous-truth error.

The comparator is nevertheless incomplete for ordinary array-like values.
At `frame.py` lines 49-60, `_differs_from_default` reduces `value != default`
with `bool(...)`. NumPy comparison of an object array containing one `None`
against the `None` default yields a one-element false array; `bool(...)` is
therefore false. The supplied collection is incorrectly classified as equal
to the scalar default.

Exact direct counterexample:

[source,python]
----
intent = {
    "participating_columns": ["a", "b"],
    "compact_column": "p",
    "separator": "|",
    "absent_value": "-",
}
frames = {
    "s": pd.DataFrame([{"a": "A", "b": "B"}]),
    "_meta": {"sentinel": {"keep": ["unchanged"]}},
}

out = encode_cell_values(
    frames,
    source="s",
    output="out",
    codec_intent=intent,
    allowed_tokens=np.array([None], dtype=object),
)
----

Observed: the call succeeds and returns `out` with `p == "A|B"`; the
historical `allowed_tokens` declaration is silently ignored. The analogous
decode call with `allowed_codes=np.array([None], dtype=object)` also succeeds.
`build_steps_from_config` plus `run_pipeline` reproduces the encode bypass, so
direct and pipeline behavior agree but both violate the contract.

Expected: any actual collection supplied for a structurally meaningful
parameter whose documented default is `None` is distinct historical intent
and must produce the deterministic mutual-exclusion `ValueError` before
position output. The input frames and nested metadata remain unchanged, and
no codec metadata or cleanup is written, but an output frame is returned when
rejection was required. Affected path:
`_reject_conflicting_historical_args` -> `_differs_from_default` -> position
encode/decode dispatch. Severity: important. No human decision is required.
