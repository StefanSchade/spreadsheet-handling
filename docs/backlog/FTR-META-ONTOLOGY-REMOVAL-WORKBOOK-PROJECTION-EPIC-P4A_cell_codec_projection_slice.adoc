= Cell Codec Projection-Family Slice: FTR-META-ONTOLOGY-REMOVAL-WORKBOOK-PROJECTION-EPIC-P4A
:revdate: 2026-07-20
:status: Implemented
:type: Projection-family slice record (cell_codec)

== Purpose

Second projection-family slice, independent of `compact_multiaxis` (which
remains blocked pending XRef acceptance and is treated strictly as a
consumer here). Applies the established method: explicit transformation
parameters, minimal persisted feature-local inverse intent, run-local
Resolution only where a concrete consumer exists, explicit cleanup only
when mechanically safe, and no generic lifecycle, canonicality, provenance,
or Workbook View semantics.

== Field inventory result: no persisted intent at all

The complete producer/consumer trace found that **`_meta.cell_codecs` had
no runtime reader anywhere** — not the codec steps themselves (decode and
encode both *require* explicit `codec_intent` or `mode` and never fall
back to metadata), not `compact_multiaxis` (which passes parameters
explicitly and persists its own `compact_multiaxis` family), not
rendering, schema maintenance (generic convention scan never matched the
real payload shapes), sparse defaults, column roles, or any consumer
repository. Every field in both payload shapes (`operation`, `source`,
`output`, `value`, `code`, `passthrough_columns`, `drop_empty`,
`group_by`, `mode`, `delimiter`, `strip`, `allowed_*`, `canonical_order`,
`normalize_case`, `participating_columns`, `compact_column`, `separator`,
`absent_value`) was descriptive duplication of explicit configuration.

Under the family rule (retain only with a concrete decision consumer) the
producer was removed entirely:

* Cell Codec intent lives in explicit step configuration only. Decode
  without explicit intent fails (pre-existing behavior, now pinned with a
  legacy-payload-present test proving no metadata fallback resurrects).
* There is no run-local Resolution facet: nothing consumed one.
* Legacy `_meta.cell_codecs` payloads are tolerated pass-through sediment
  (persistence boundary passes them unchanged; nothing reads them).
* No cleanup commands are involved: neither codec direction supersedes its
  source frame by contract (encode keeps the structured frame; decode
  keeps the compact frame); frame-set decisions stay with the pipeline
  builder via the generic cleanup contract.

This also removes the `compact_multiaxis`-composed sediment: the wrapper's
inner codec calls no longer persist entries naming
`+__compact_multiaxis_*+` temp frames (question 10 of the slice mandate —
resolved as a side effect of the producer removal, without touching the
wrapper).

== Forward/inverse contract

Two explicit contracts, both symmetric:

* *Position-based* (`codec_intent` on both steps): `participating_columns`
  join into `compact_column` with `separator`; `absent_value` represents
  empty/missing attributes and decodes back to `""`. Encode and decode
  take the identical intent mapping.
* *Historical code-row* (`mode` + codec parameters, consumed primarily by
  `compact_multiaxis`): code rows group into a delimited cell and split
  back. Symmetric parameter vocabulary; `mode` is required explicitly.

Declaration guards (fail before any output):

* duplicate `participating_columns` / `group_by`; missing participating
  columns; `compact_column` that is also participating or collides with a
  passthrough column; `absent_value` containing the separator; duplicate
  physical source labels (equality-based detection, codec diagnostic);
  decode rejects participating columns that already exist beside the
  compact column (silent overwrite).

== Carrier/type contract (string-oriented, enforced)

Cell Codec is intentionally string-oriented
(`ADR-VISIBLE-CELL-TYPING-STRING-SUBSTRATE`); this slice states and
*enforces* that contract instead of adding a typed-value system:

* position-based participating values must be non-empty strings or empty;
  numeric, boolean, date, and mixed values fail with an explicit
  diagnostic (`1` vs `"1"` ambiguity cannot arise: the typed value fails);
* empty string and null/missing are equivalent: both encode as
  `absent_value` and decode to `""` (not null);
* there is no escaping/quoting: separator-in-value fails encoding, and
  `split_tokens` serialization now rejects delimiter-containing tokens
  (previously it silently produced cells that decoded into the wrong
  token count — the one behavioral defect found in the scalar layer);
* `whole_cell_code` cells may contain delimiter punctuation by design;
* XLSX/ODS roundtrip proven for compact cells including absent handling
  and numeric-looking string tokens (`"1"` stays a string across both
  carriers under the string substrate).

The historical code-row path stringifies typed code values (`7` → `"7"`);
that wrapper-era behavior is characterized by test and documented, with
tightening deferred to the `compact_multiaxis` composite slice.

== Schema maintenance

`cell_codecs` moved from the generic blocked roots to the ignored
(out-of-scope) surfaces: a retired family with no producer and no consumer
has no live references, so legacy sediment must not block renames or
drops. `ReferenceRoot.CELL_CODECS` was removed; the meta-reference-surfaces
row records the ignored role with the uninventoried marker. A
characterization test proves a realistic legacy payload blocks nothing and
passes through unchanged.

== Composed-consumer findings (`compact_multiaxis`, consumer only)

* The wrapper passes all codec parameters explicitly; nothing in it reads
  codec metadata. Its behavior is unchanged by this slice (full compact
  suite green).
* Wrapper-owned and deferred to its composite slice: legacy stringification
  of typed codes in the code-row path; the wrapper's own
  `compact_multiaxis` metadata family (including its codec-parameter
  duplication) was not inventoried here.

== Registries and documentation

* Step registry: `codec_intent` parameter documented on both steps;
  `cell_codecs` write refs and the (already false) `encode` read ref
  removed; the compact entries' transitive `cell_codecs` refs pruned to
  match actual behavior (description-only change, no wrapper refactor).
* Meta registry: `cell_codecs` entry removed (with pinned-test updates);
  its consumer list had claimed `layout_composer` / `workbookir_to_frames`
  consumption that never existed.
* User guide: the codec section was substantially wrong (invented
  `delimited`/`positional` modes, wrong `,` delimiter default, wrong
  `drop_empty` default, `codec_intent` undocumented); rewritten to the
  actual contract with the string-contract note.
* Functional model: durable intent clarified as the explicit declaration;
  implementation reference updated. Technical model classification list
  and the string-substrate ADR carry retirement notes (the ADR decision
  itself is unchanged).

== Tests

23 new tests (suite 1552 → 1578): no-metadata pinning for all four paths
plus legacy-payload no-fallback; declaration guards (duplicates, missing,
collisions, absent/separator conflicts, physical duplicates, decode
overlap); string-contract enforcement (numeric/boolean/date/mixed, empty
vs missing, separator-in-value); legacy path characterization
(delimiter-in-token rejection, whole-cell punctuation, stringification,
duplicate group_by/labels); input and metadata non-mutation; XLSX and ODS
compact-cell roundtrips with recreation of structured attributes; legacy
schema-maintenance sediment tolerance.

Validation: full Core suite 1578 passed / 1 xfailed; ruff clean;
architecture/registry suites green; demo ACME XRef roundtrip green;
worldbuilding XLSX, ODS, and workbook-view workflows green (the dino
pipelines exercise the code-row codec through `compact_multiaxis` in both
directions).

== Compatibility

* `_meta.cell_codecs` is no longer written (pre-1.0; no consumer read it;
  legacy payloads tolerated; release-noted).
* New failure modes only where output was previously corrupt or ambiguous:
  typed participating values (silent type change), delimiter-in-token
  serialization (undecodable cells), duplicate labels/fields (non-scalar
  cells), decode overwrite collisions.
* No shipped consumer pipeline uses the removed metadata or the newly
  rejected inputs (demo and worldbuilding verified green).

== Deferred (wrapper-owned or out of scope)

* `compact_multiaxis` composite slice: its own metadata family inventory,
  code-row typed-value tightening, temp-frame naming.
* Category-hinted/typed compact values: future scope per the functional
  model; would be a new explicit contract (see ADR status note).
* Side-effecting writers, metadata pileup, M:N Workbook View, facet
  taxonomy: unchanged deferrals.

== Human decisions

None required. The string-oriented contract is the functional model's MVP
scope plus the accepted string-substrate ADR; the metadata removal follows
mechanically from the no-consumer rule. The removal is prominently
flagged for the independent review: if the review establishes a concrete
future consumer for persisted codec intent, reintroduction would be a
narrow, explicit contract addition.
