= FTR-CELL-CODEC-LEGACY-CODE-ROW-RETIREMENT-P4A
:revdate: 2026-07-20
:status: Open
:type: Follow-up FTR (Cell Codec historical code-row path)

== Purpose

Record, without implementing, the decision to eventually retire or extract the
*historical code-row* Cell Codec path (`mode` / `group_by` / `delimiter` /
`allowed_*` / `canonical_order` / `normalize_case` / `strip` /
`passthrough_columns` and the `whole_cell_code` / `split_tokens` scalar
grammar). This path is retained now only for compatibility with
`compact_multiaxis`; it is not an equal-status, permanent Cell Codec contract
alongside the position-based `codec_intent` contract.

This FTR is created by the Cell Codec projection-slice Review 001 corrections
(see
`FTR-META-ONTOLOGY-REMOVAL-WORKBOOK-PROJECTION-EPIC-P4A_cell_codec_projection_slice_review_001.adoc`).
It is *not* implemented in that correction. Do not remove or restructure the
historical path as part of the Review 001 corrections.

== Current status of the historical path (as of this FTR)

* It is a *compatibility path*, not a permanent contract. The public
  `encode_cell_values` / `decode_cell_values` entry points enforce that the
  position-based `codec_intent` and the historical mode-based parameters are
  mutually exclusive.
* The shared physical-frame label boundary
  (`domain/tabular.ensure_unique_physical_column_labels`) and the historical
  declaration validations (delimiter-before-data-shortcut, duplicate
  passthrough rejection) now apply, so the path is safe while it remains.
* No `_meta.cell_codecs` is produced or consumed; intent lives entirely in
  explicit step configuration.

== Scope this FTR must cover when it is scheduled

=== Current consumers

* `compact_multiaxis` (`expand_compact_multiaxis` /
  `contract_compact_multiaxis`) is the only in-repository consumer; it calls
  the historical path explicitly in both directions.
* Confirm whether any pipeline YAML, demo, or worldbuilding configuration uses
  `encode_cell_values` / `decode_cell_values` in historical mode directly
  (outside `compact_multiaxis`).

=== Public API and registry exposure

* `encode_cell_values` / `decode_cell_values` expose the historical parameters
  on the public transform surface and in `registries/pipeline_step_registry.json`.
* Decide whether the historical parameters should remain on the public surface
  or move behind `compact_multiaxis` only.

=== Whether independent users exist

* Determine whether the historical code-row projection has independent
  functional value (code/token rows <-> compact cells) separate from
  `compact_multiaxis`, or whether it is purely a `compact_multiaxis`
  implementation detail.

=== Migration of `compact_multiaxis`

* If retired: `compact_multiaxis` must be rebuilt on the modern primitive
  contract (position-based codec or a successor primitive) before removal.
* If extracted: define the separately named primitive family and migrate
  `compact_multiaxis` to it.

=== Removal versus extraction decision

Two candidate outcomes, to be decided by this FTR:

. *Retire* the historical path when `compact_multiaxis` is rebuilt on the
  modern primitive contract; or
. *Extract* it into a separately named primitive family if it has independent
  functional value.

=== Typed-code / stringification behavior

* The historical path stringifies typed codes (for example `7` becomes
  `"7"`). Decide whether that behavior is carried forward, changed, or dropped
  in the chosen outcome, consistent with
  `ADR-VISIBLE-CELL-TYPING-STRING-SUBSTRATE`.

=== Documentation and compatibility implications

* Update the Functional Model, Technical Model, User Guide, pipeline registry,
  and release notes when the path is retired or extracted.
* Record migration guidance for any external consumer discovered above.

== Explicitly out of scope

* Do not implement retirement or extraction here.
* Do not redesign the position-based contract.
* Do not reopen the explicit-only metadata direction, the string substrate
  decision, or Tuple/MultiIndex / stable-id-vs-display-label design.

== Relationship to the epic

Companion to the Cell Codec projection-family slice under
`FTR-META-ONTOLOGY-REMOVAL-WORKBOOK-PROJECTION-EPIC-P4A`. `compact_multiaxis`
remains blocked pending XRef acceptance and is the driver for the
removal-versus-extraction decision recorded here.
