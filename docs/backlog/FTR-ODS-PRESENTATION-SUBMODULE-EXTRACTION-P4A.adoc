===== FTR-ODS-PRESENTATION-SUBMODULE-EXTRACTION-P4A -- Extract ODS presentation into an ODS-local module

*Status:* New

*Purpose:*::
Relocate the ODS presentation-extraction logic into an ODS-local submodule.
Behavior-preserving; structure only.

*Context:*::
The io_backends / carrier-boundary review found presentation extraction is the
largest concentration in the ODS parser. Once the cell-walk consolidation has
reduced it to a cohesive block, relocating it to an ODS-local module shrinks the
parser and lets a modularization guard pin the new seam.

*Dependencies:*::
* `FTR-ODS-PRESENTATION-CELL-WALK-CONSOLIDATION-P4A` -- relocate only after the
  cell-walk is consolidated.

*Scope:*::
* Move the presentation extractors and style-map builders into an ODS-local
  module.
* Add a modularization guard mirroring the existing interpretation-seam guard.

*Non-Goals:*::
* No behavior change.
* No new ODS features.
* No change to `_parse_table_grid`.
* No renderer split.

*Acceptance:*::
* Presentation extraction lives in an ODS-local module.
* A modularization guard pins the new seam.
* Existing characterization and parity tests stay green with no behavior change.
