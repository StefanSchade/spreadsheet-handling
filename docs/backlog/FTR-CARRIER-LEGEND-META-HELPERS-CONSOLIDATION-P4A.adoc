===== FTR-CARRIER-LEGEND-META-HELPERS-CONSOLIDATION-P4A -- Consolidate shared legend / store-meta helpers

*Status:* New

*Purpose:*::
Consolidate the byte-identical legend-hint and workbook-meta-store helper logic
shared by the XLSX and ODS parsers into a single carrier-neutral module -- only
where the logic is actually shared. Behavior-preserving.

*Context:*::
The io_backends / carrier-boundary review identified `_legend_table_hints`,
`_apply_legend_table_hints`, and `_store_workbook_meta` as byte-identical copies
across both parsers. Hoisting them to one carrier-neutral home removes the
duplication while keeping each backend's carrier-specific code untouched.

*Dependencies:*::
* `FTR-IO-BACKENDS-DOMAIN-CROSSING-GUARD-P5` -- the shared module must be on the
  guard's allow-list.

*Scope:*::
* Hoist the byte-identical helpers into a shared carrier-neutral module.
* Both parsers delegate to the shared module.

*Non-Goals:*::
* No shared XLSX / ODS renderer.
* No mixing of the XLSX live-object model with the ODS named-style carrier model.
* No behavior change.

*Acceptance:*::
* One copy of each shared helper lives in a carrier-neutral module.
* Both parsers import it.
* Legend roundtrip and parity tests stay green with no behavior change.
