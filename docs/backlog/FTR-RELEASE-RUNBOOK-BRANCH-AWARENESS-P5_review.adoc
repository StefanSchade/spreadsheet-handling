==== Architectural Assessment

The implementation directionally satisfies the FTR. The runbook now makes
release-tag branch topology explicit, adds a pre-tag check path, documents
accidental `dev`-tag recovery, and gives operators a direct way to inspect
canonical tag topology without reading workflow logs.

No scope creep was found into README, Pages output, CI, demo, or public-doc
FTR implementation. This remains release-process hardening, not release
automation.

A normal commit/push to `main` does not publish to PyPI. The release workflow
runs on pushes to `main` and `v*` tags, but the PyPI `publish` job is gated by
`startsWith(github.ref, 'refs/tags/v')`, so ordinary `main` pushes run tests
only.
