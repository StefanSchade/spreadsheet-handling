===== FTR-COMPACT-MULTIAXIS-INVENTORY-P5 -- Inventory the compact_multiaxis composition family

*Status:* New

*Purpose:*::
Inventory the `compact_multiaxis` family before any modeling or runtime work.
Inventory only; no runtime implementation.

*Context:*::
`compact_multiaxis` imports `cell_codec`, `xref_crosstable`, and the shared
`_legend_blocks` reader, which makes it the likely composition capstone and the
likely third Resolution-bearing family. The sequencing review forces `cell_codec`
modeling first; this inventory establishes the facts a later decision needs,
without committing implementation.

*Dependencies:*::
* `FTR-CELL-CODEC-FUNCTIONAL-MODEL-P5` -- `cell_codec` should be modeled before
  the capstone is inventoried against it.

*Scope:*::
* Map the dependencies on `cell_codec`, `legend_blocks`, and `xref_crosstable`.
* Record the current `_meta` usage and any deferred dense-axis-reconstruction
  contract.
* Decide whether `compact_multiaxis` is a composition capstone that warrants a
  later modeling / implementation FTR, or stays green-but-unmodeled.

*Non-Goals:*::
* No runtime implementation.
* No Functional Model promotion in this slice (inventory only).
* No module split.

*Acceptance:*::
* The inventory documents the `cell_codec` / `legend_blocks` / `xref_crosstable`
  dependencies.
* It states whether `compact_multiaxis` is a composition capstone.
* It records a go / defer decision for later modeling or implementation work.
* No runtime code is changed.
