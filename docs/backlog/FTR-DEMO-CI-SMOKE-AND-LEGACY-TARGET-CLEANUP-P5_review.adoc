== Verdict

Ship. The slice deletes targets/scripts whose underlying entrypoints
(`sheets-pack`, `sheets-unpack`) and configuration (`sheets.yaml`) no
longer exist on the supported core surface; everything that survives is
either exercised by `pytest` or by `make run PIPELINE=...`. The
maintained beta-demo flow (`make run PIPELINE=./pipelines/demo_product_business_slice.yaml`)
is unchanged and validated locally.

== Scope

In scope (implemented in `../demo`):

* `Makefile`: removed targets `pack`, `unpack`, `roundtrip`, `verify`,
  `extract`; removed now-unused vars `CONFIG`, `PACK_CMD`, `UNPACK_CMD`,
  `PACK_SETS`, `SHA`, `DATA_DIR`; corrected the misleading
  `quick roundtrip` section comment.
* `scripts/`: deleted orphans `unpack_current_sha.sh`, `sha.py`,
  `verify.py`. Trimmed dead `PACK_CMD`/`UNPACK_CMD`/`RUN_CMD` exports from
  `dev-local.lib.sh` and rewrote its usage comment.
* `docs/reference/commands.adoc`: removed the legacy compatibility-shim
  block and the broken `make verify` / `make extract` sections; added a
  short paragraph stating that maintained flows go through `make run`.
* `.github/workflows/ci.yml`: added an intent header comment naming the
  three layers (install, demo-journey pytest, public CLI smoke) and
  renamed the steps so the layered intent is visible from the workflow
  log. No new long-running steps.

Out of scope (deliberately left alone):

* `pipelines/demo_extraction*.yaml` — still consume the actively-tested
  `branch_manager_summary` plugin via `sheets-run`. Whether they belong
  in the maintained set is a content question, not CI hygiene; see L1.
* The `lint-json` and `deps` Makefile targets — pre-existing, not part of
  the legacy `pack/unpack/extract` family. Left as-is.
* `tools/` snapshot bootstrap — already documented as a manual-copy
  optional helper per `repository_boundaries.adoc`. Unchanged.

== Findings

[cols="1,2,4,1", options="header"]
|===
| ID | Severity | Finding | Status

| M1
| Medium
| The five removed targets had no functioning compatibility-shim path:
  `sheets-pack`/`sheets-unpack` are gone from core's `pyproject.toml`
  (`pyproject` only exports `sheets-run`; `cli/apps/` has no
  `sheets_pack` or `sheets_unpack` module), and `verify`/`extract`
  reference a `sheets.yaml` that no longer exists in demo. Wholesale
  removal is the right disposition under the FTR's
  "removed, isolated, or explicitly documented as compatibility-only"
  rule, since a shim that cannot run is worse than absence.
| Closed — removed.

| M2
| Medium
| `scripts/dev-local.lib.sh` exported `PACK_CMD`, `UNPACK_CMD`, and
  `RUN_CMD` pointing at module paths
  (`spreadsheet_handling.cli.apps.sheets_pack`,
  `spreadsheet_handling.cli.apps.sheets_unpack`,
  `spreadsheet_handling.cli.apps.run`). The first two refer to modules
  that do not exist on the supported core surface; the third was never
  consulted by the Makefile (which hardcodes its own `RUN_CMD`). Leaving
  the exports in place would silently mislead any future helper.
| Closed — exports removed, PYTHONPATH munging and `which-lib` retained.

| L1
| Low
| `pipelines/demo_extraction.yaml`,
  `pipelines/demo_extraction_with_io.yaml`, and
  `pipelines/demo_extraction_with_generic_step.yaml` survive without any
  maintained Make wrapper. They still exercise the
  `branch_manager_summary` plugin (covered by `tests/test_transforms.py`
  and `tests/test_verify_steps.py`), so they are not broken — but their
  status as "maintained demo journey" vs. "kept for plugin coverage
  only" is undocumented.
| Deferred — out of scope. Track as content-curation follow-up if the
  curated-journey catalog under `docs/walkthroughs/` is revisited.

| L2
| Low
| `make help` does not list `lint-json` or `deps` because those targets
  lack `##` comments. Pre-existing condition (predates this FTR); not a
  regression of this slice.
| Acknowledged — left untouched to keep the slice narrow.

| L3
| Low
| CI now relies on `pytest` as the maintained-pipeline smoke layer. This
  is intentional and explicit in the new workflow comment, but it does
  couple "demo-CI intent" to "the pytest suite happens to invoke
  `make run` equivalents through the demo modules". If the demo test
  suite ever shrinks below those journeys, layer 2 of the documented
  intent silently weakens.
| Acknowledged — review the CI intent comment alongside any future
  removal from `tests/`. No action this slice.
|===
