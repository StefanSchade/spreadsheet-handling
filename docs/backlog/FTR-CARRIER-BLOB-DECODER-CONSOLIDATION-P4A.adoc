===== FTR-CARRIER-BLOB-DECODER-CONSOLIDATION-P4A -- Consolidate the hidden-meta blob decoder

*Status:* New

*Purpose:*::
Consolidate the duplicated hidden-meta blob-decode logic into a single
backend-neutral helper. Behavior-preserving.

*Context:*::
The io_backends / carrier-boundary review found the `json` -> `ast.literal_eval`
blob decoder duplicated across both parsers and three rendering modules. This is
carrier-neutral logic copied verbatim; one home removes drift risk without
changing behavior.

*Dependencies:*::
* A focused blob-decode unit test (JSON form, legacy `repr` form, garbage ->
  `None`) should exist before the consolidation so the shared helper has a home
  test.

*Scope:*::
* Introduce one backend-neutral decode helper.
* Make the previously-duplicated call sites delegate to it.

*Non-Goals:*::
* No behavior change.
* No carrier-specific abstraction leak into the neutral helper.
* No change to `_parse_table_grid`.

*Acceptance:*::
* A single backend-neutral blob-decode helper exists.
* The previously-duplicated call sites delegate to it.
* Roundtrip and parity tests stay green with no behavior change.
