===== FTR-CONCEPT-DOCS-CLEANUP-P5 -- Edit and normalize concept documentation

*Status:* New
*Priority:* P5
*Area:* Documentation / semantics

==== Purpose

Clean up the concept documentation after the documentation split.

Scope:

* Technical concepts in `docs/technical_model/ch04_concepts/`.
* Use-case concepts in `docs/functional_model/ch04_use_case_concepts/`.

==== Rationale

The concept documents were moved to better homes before they were fully
edited.

Known state:

* Some text still reflects earlier directory structures.
* Some concepts mix stable semantics, examples, roadmap notes, and exploratory
  material.
* The current move intentionally preserved content instead of rewriting it.

==== Acceptance

* Separate stable concepts from exploratory or historical notes.
* Remove stale path references from active concept documents.
* Align titles, status markers, and cross-references.
* Keep technical concepts in the technical model.
* Keep use-case concepts in the functional model.
* Move obsolete source fragments to cold storage when they no longer serve the
  active documents.
* Do not add new product behavior or architecture changes.
