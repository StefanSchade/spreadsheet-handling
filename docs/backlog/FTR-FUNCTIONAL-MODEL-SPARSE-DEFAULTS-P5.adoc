===== FTR-FUNCTIONAL-MODEL-SPARSE-DEFAULTS-P5 -- Model sparse default crosstable semantics

*Status:* New

*Type:* Functional Model documentation.

*Purpose:*::
Add a narrow Functional Model treatment for the already implemented
`sparse_defaults` family so readers can understand sparse editable matrix
semantics without scanning implementation or cold storage.

*Context:*::
Derived from the June 2026 transformation/capability family inventory:
`docs/cold_storage/working_context/transformation_family_inventory_2026_06/transformation_family_inventory.adoc`.
The inventory classifies `sparse_defaults` as implemented and technically
documented, with missing Functional Model coverage.

This FTR exists because the inventory is a cold-storage reference, not the
active work item. Project Memory only links to the inventory for navigation and
intent. This backlog item is the actual follow-up work.

*Scope:*::
* Add or extend Functional Model prose to describe why sparse default collapse
  exists in human-editable crosstables.
* Clarify the forward and reverse meaning of default values, blanks, and
  explicit non-default values.
* State the relationship to XRef crosstable projection and dense/sparse matrix
  editing.
* Keep the treatment use-case-oriented and bounded; reference the cold-storage
  inventory for implementation evidence rather than copying it.

*Non-Goals:*::
* No runtime implementation.
* No changes to `sparse_collapse` or `sparse_expand`.
* No new pipeline step names or YAML surface.
* No full transformation-family catalog copied into the Functional Model.
* No Project Memory entities.

*Acceptance Criteria:*::
* Functional Model material exists for sparse-default crosstable semantics.
* The material distinguishes visible blank cells from semantic default values.
* The material states how this family composes with XRef crosstable projection.
* The material links to the June 2026 inventory as evidence/provenance but does
  not duplicate the inventory table.
* The FTR remains documentation-only; no runtime implementation is performed
  unless separately approved.

*Relationship to the inventory and Functional Model:*::
The inventory remains the canonical June 2026 reference for implemented-family
status. This FTR promotes only the needed Functional Model explanation for one
missing family. It does not supersede or replace the inventory.
