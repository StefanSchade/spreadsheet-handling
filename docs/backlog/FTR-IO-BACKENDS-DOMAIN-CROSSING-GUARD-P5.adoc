===== FTR-IO-BACKENDS-DOMAIN-CROSSING-GUARD-P5 -- Guard legend_blocks as the only domain crossing

*Status:* New

*Purpose:*::
Add an architecture guard asserting that `legend_blocks` remains the only
intentional domain-family token crossing into `io_backends`. No behavioral
change.

*Context:*::
The io_backends / carrier-boundary review confirmed `legend_blocks` is the single
legitimate domain crossing (a visible legend must be classified so it is not
treated as payload). The existing import-graph guards fence `domain` imports, but
a string-key read of a *new* domain family in a parser would pass unnoticed. This
converts the accepted-exception prose into an enforced invariant.

*Dependencies:*::
* None.

*Scope:*::
* Add an architecture guard (token / AST scan of `io_backends/**`) asserting
  `legend_blocks` is the sole permitted domain-family token.
* Record an allow-list entry for any future shared carrier-neutral helper module.

*Non-Goals:*::
* No behavioral change to parsers or renderers.
* No new boundary mechanism beyond the existing guard style.

*Acceptance:*::
* A guard asserts `legend_blocks` is the only domain-family token in
  `io_backends`.
* A new domain-family token read in a parser fails the guard.
* No runtime behavior changes (a clean pass that flags nothing is success).
