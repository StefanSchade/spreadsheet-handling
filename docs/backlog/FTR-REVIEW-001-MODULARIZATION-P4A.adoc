===== FTR-REVIEW-001-MODULARIZATION-P4A — Stable `__all__` per sub-package, then split oversized modules

*Status:* Open
*Priority:* P4A
*Area:* domain / rendering / io_backends
*Kind:* Refactoring / Hygiene
*Review source:* REVIEW-001, Cluster C7, Finding IDs: D-004, D-010, IR-009

==== Purpose

Nine production modules exceed 440 lines (peaks: 725, 631, 567, 562, 562); one adapter
module (`io_backends/ods/odf_renderer.py`, ~540 lines) is also above the threshold. At
the same time, `domain/transformations/__init__.py`, `domain/__init__.py`, and
`rendering/__init__.py` are empty, and `domain/validations/__init__.py` exports only
half of its package. Without stable public package surfaces, any module split immediately
breaks import paths for callers and agents. This FTR therefore proceeds in two phases:
Phase 1 establishes stable `__all__` exports (making subsequent splits transparent);
Phase 2 splits oversized modules along cohesion boundaries.

Note: `FTR-MAINTAINER-ENTRYPOINTS-AND-MODULE-BUDGETS-P5` covers source-style
*warning tooling* (adding size thresholds to `check_source_style.py`) and entrypoint
documentation. This FTR covers the *structural execution* of establishing public surfaces
and splitting modules — distinct scope.

==== Current State (verified)

* `domain/__init__.py` — empty. (D-010)
* `domain/transformations/__init__.py` — empty. (D-010)
* `domain/validations/__init__.py` — exports only the `ReferenceFinding`/graph
  validation half; `Finding` + `apply_severity_policy` side not re-exported. (D-010)
* `rendering/__init__.py` — empty. (IR-009)
* Nine `domain/`/`core/` modules >440 lines; `io_backends/ods/odf_renderer.py` ~540
  lines. (D-004)
* Phase 1 finding (D-010) explicitly labels stable `__all__` as prerequisite for splits
  ("Phase 1 nennt D-010 ausdrücklich 'Voraussetzung'").

==== Scope

*Phase 1 — Stable public surfaces (prerequisite for Phase 2)*

. `domain/transformations/__init__.py` — add `__all__` listing the public
  transformation callables; callers can then `from domain.transformations import X`
  without knowing the internal module.
. `domain/validations/__init__.py` — extend `__all__` to export both halves:
  `Finding` / `apply_severity_policy` side and `ReferenceFinding` / graph side.
. `domain/__init__.py` — add `__all__` for any top-level domain symbols used
  cross-package; leave empty `__all__` with a comment if none exist today.
. `rendering/__init__.py` — add `__all__` for `compose_workbook`,
  `build_spreadsheet_render_plan`, and any other public entry points.
. Verify that existing callers outside each package reference only names in the declared
  `__all__`; update any deep-path imports to use the stable sub-package path.

*Phase 2 — Module splits (file-by-file, one commit per split, green tests between)*

For each oversized module, create a sub-package replacing the file and split along
cohesion boundaries:

. `domain/transformations/cell_codec.py` (if >440 lines) — target sub-modules:
  `parse`, `serialize`, `decode_encode`, `legend`. Update
  `domain/transformations/__init__.py` re-exports.
. `domain/validations/reference_validations.py` (if >440 lines) — target sub-modules:
  `rules`, `conditions`, `findings_frame`. Update `domain/validations/__init__.py`.
. `io_backends/ods/odf_renderer.py` (~540 lines) — target: `address_helpers`,
  `style_helpers`, `buffered_sheet`, `build_table`, `freeze_settings`, `op_dispatch`.
  Update `io_backends/ods/__init__.py`.
. For remaining oversized modules: identify with
  `wc -l src/**/*.py | sort -rn | head -20`; split only where a natural cohesion
  boundary exists, not mechanically to satisfy a line count.
. After each split: run full test suite before the next split.

==== Non-Goals

* Do not change any public API — all existing import paths must work via `__all__`
  re-exports after Phase 1.
* Do not split modules mechanically without a genuine cohesion boundary.
* Do not address module-header docstrings here (that is
  FTR-REVIEW-001-MODULE-HEADERS-P5).
* Do not add module-size warning tooling here (that is
  FTR-MAINTAINER-ENTRYPOINTS-AND-MODULE-BUDGETS-P5).

==== Dependencies

Phase 1 must complete before Phase 2 commits. No external FTR dependencies.

==== Risk

Medium for Phase 1 (stable `__all__` may surface callers using unexported names);
high for Phase 2 (module splits across packages risk import cycles). Mitigation: one
split per commit; run `mypy --strict` on affected packages after each split; `python -c
"import spreadsheet_handling"` must succeed without circular-import warnings.

==== Acceptance

*Phase 1:*

* `grep -r 'from domain.transformations import' src/ tests/` uses only names in
  `domain/transformations/__all__`.
* `from rendering import compose_workbook` succeeds (rendering `__all__` non-empty).
* `domain/validations/__init__.py` exports both `Finding` and `ReferenceFinding`.
* Full test suite passes.

*Phase 2:*

* `wc -l src/spreadsheet_handling/**/*.py | sort -rn | head -20` shows no production
  file exceeding ~350 lines (justified exceptions documented with a comment).
* `python -c "import spreadsheet_handling"` succeeds with no `ImportError` or circular
  import.
* Full test suite passes after each split commit.
