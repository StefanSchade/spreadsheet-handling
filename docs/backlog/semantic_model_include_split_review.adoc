== Content-Preservation Assessment

The prose text itself appears to have been moved rather than rewritten. The
source reconstruction check found no wording changes, example changes,
glossary-entry changes, warning/note changes, or semantic-claim edits beyond
chapter-boundary whitespace.

The boundary whitespace is not harmless in AsciiDoc. Comparing rendered HTML
from `d51d112^:docs/semantic_model/semantic_model.adoc` against the current
included document shows changed section nesting and numbering. For example:

* `Relation To Other Models` is no longer rendered as top-level chapter 2; its
  heading text is swallowed into the preceding list item in `Management
  Summary`.
* several later chapters are similarly collapsed into prior sections or lists;
* the table of contents changes from 18 top-level chapters to a smaller,
  incorrectly nested hierarchy.

This is a semantic document-structure change, so the reviewed commit is not a
pure structural split as-is.

== Findings

=== Blocker

*Rendered semantic-model structure is changed by missing blank lines at include
boundaries.*

Location: `docs/semantic_model/semantic_model.adoc` include boundaries and the
ends of the included chapter files.

The source split removed the blank line that previously separated a chapter's
final block from the next `[[semantic-model-...]]` anchor and `==` heading.
Asciidoctor accepts the document, but it no longer renders the same hierarchy.
The rendered table of contents and section numbering change, and some top-level
chapter headings are parsed as text inside preceding blocks.

This violates the expected pure documentation-structure split. The fix should
preserve a block-separating blank line at each include boundary, either by
adding blank lines between include directives in the entry file or by ensuring
the included chapter files terminate with the needed separation.

=== Major

None.

=== Minor

None.

=== Informational

* The entry file remains the canonical document entry point and keeps the title,
  attributes, table-of-contents settings, and scaffold note.
* The 18 expected include files exist and are included in the expected order.
* The 18 top-level semantic-model anchors exist exactly once in source.
* The reviewed commit does not touch runtime code, tests, registries, release
  files, backlog/FTR lifecycle state, or other non-semantic-model files.
* Backlog rendering succeeds.

== Final Verdict

Verdict: needs rework.

`d51d112` is not ready to keep as-is because the rendered semantic-model
document is not structurally equivalent to the parent version. The required
follow-up is small and mechanical: restore block-separating blank lines at the
include boundaries and rerun the source reconstruction plus rendered comparison
checks.
