===== FTR-ODS-PRESENTATION-CELL-WALK-CONSOLIDATION-P4A -- Consolidate ODS cell-addressed presentation walk

*Status:* New

*Purpose:*::
Consolidate the repeated cell-addressed presentation-extraction logic inside the
ODS parser path into one shared cell-walk. Behavior-preserving.

*Context:*::
The io_backends / carrier-boundary review found the three cell-addressed
extractors -- horizontal alignment, vertical alignment, and text orientation --
share a large near-duplicated cell-walk that differs only in which style map it
consults and which value it writes. Its characterization already exists, which
makes this the first safe ODS-internal behavioral slice.

*Dependencies:*::
* `FTR-CARRIER-BLOB-DECODER-CONSOLIDATION-P4A` and
  `FTR-CARRIER-LEGEND-META-HELPERS-CONSOLIDATION-P4A` -- do the cross-cutting
  hygiene first to shrink the blast radius.

*Scope:*::
* Fold the cell-addressed extractors (horizontal alignment, vertical alignment,
  text orientation, and similar) onto one shared cell-walk with thin per-family
  classifiers.
* Preserve `ParserLimits`, clipping to the parsed extent, and the pathological
  repeated-cell defenses.

*Non-Goals:*::
* Do not modify `_parse_table_grid`, even cosmetically.
* No new ODS features.
* No behavior change.

*Acceptance:*::
* The cell-addressed extractors share one cell-walk; each family is a thin
  classifier.
* `ParserLimits`, extent clipping, and repeated-cell characterization stay green
  unchanged.
* `_parse_table_grid` is untouched.
* No behavior change.
