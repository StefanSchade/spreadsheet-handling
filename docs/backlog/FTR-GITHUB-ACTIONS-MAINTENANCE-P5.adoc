===== FTR-GITHUB-ACTIONS-MAINTENANCE-P5 -- Bring CI maintenance noise into a reviewed state

*Status:* Proposed
*Priority:* P5
*Area:* CI infrastructure / GitHub Actions

==== Goal

Bring GitHub Actions and dependency-maintenance automation into a reviewed,
non-noisy state after the beta release.

The `v0.1.0b6` release surfaced CI warnings and a failing Dependabot-related
job that were unrelated to the actual release artifacts but reduced
confidence in the repository health signal.

==== Motivation

Observed during the beta release:

* GitHub Actions emitted Node.js 20 deprecation warnings;
* at least one Dependabot job failed with a `403 Token invalid` error;
* release-critical jobs were green, but repository-level status still
  contained red or warning signals;
* it was not immediately obvious which CI signals were release-blocking and
  which were infrastructure-maintenance noise.

This weakens CI as a decision instrument.

==== Scope

This FTR covers:

* GitHub Actions version maintenance;
* Dependabot / dependency-graph configuration review;
* classification of infrastructure-maintenance failures;
* reducing warning noise in release-adjacent workflows.

This FTR does not cover:

* dependency policy redesign;
* package-version pinning strategy;
* release automation redesign;
* replacing Dependabot with Renovate.

==== Acceptance

* All maintained workflows use currently supported GitHub Action major
  versions.
* Node.js runtime deprecation warnings are removed or explicitly documented
  as temporarily accepted.
* Dependabot configuration is located, reviewed, and documented.
* The source of Dependabot jobs is clear:
** explicit `.github/dependabot.yml`, or
** repository Security / Dependency Graph settings.
* Dependabot failures are classified as repository misconfiguration, GitHub
  service issue, permission issue, or intentional disablement candidate.
* The release runbook distinguishes release-blocking CI from non-blocking
  maintenance automation.
* No release-critical workflow depends on unstable Dependabot behavior.

==== Candidate Checks

Review:

* `.github/workflows/*.yml`
* `.github/dependabot.yml`
* repository Security settings
* repository Actions permissions
* dependency-graph settings
* alert and update configuration
