*Reader orientation -- Functional/Semantic/Technical boundary.*

The Semantic Model defines the *contract*: vocabulary, interpretation
rules, producer/consumer responsibility, precedence, lifecycle, and
guard candidates. Problem-space explanation lives in the Functional
Model (`docs/functional_model/`); implementation mechanism lives in the
Technical Model (`docs/technical_model/`).

The chapters below are grouped by role under
`FTR-FUNCTIONAL-SEMANTIC-MODEL-SPLIT-P5`:

* *Contract Core.* 01 Management Summary, 02 Relation To Other Models,
  04 Ambiguity Policy, 06 Project Semantic Model, 07 Overlapping
  Semantic Sets, 08 Lifecycle And Update Semantics, 09 Registry Role,
  11 Conflict And Precedence Questions, 12 Contract Matrices, 13
  Semantic Guards, 14 Versioning And Evolution, 18 Glossary.
* *Explanatory Context (relocated or marked).* 03 Problem Statement
  (relocated to Functional Model ch06; this chapter is a navigation
  stub), 05 External Inspirations (internal split: "relevant ideas"
  blocks are explanatory background that points to the Functional
  Model; "project interpretation" blocks are contract-adjacent and stay
  here), 10 Roundtrip Walkthroughs (relocated to Functional Model ch06;
  this chapter is a navigation stub).
* *Adoption / Planning.* 15 Adoption Strategy, 17 Open Questions.
* *Appendix.* 16 Planned FTRs (most items archived in cold storage; see
  the chapter's own archival note).

This grouping is a navigation aid. The include order below preserves the
original chapter sequence so that incoming references and section
numbering are not disturbed.
