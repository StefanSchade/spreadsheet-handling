===== FTR-REVIEW-001-META-REGISTRY-DERIVED-P3 — Register `derived` meta channel in `meta_registry.yaml`

*Status:* Done
*Priority:* P3
*Area:* meta-registry / domain / rendering
*Kind:* Hygiene / Architecture
*Review source:* REVIEW-001, Cluster C6, Finding ID: IR-005

==== Purpose

`domain/transformations/enrich_lookup.py` and `domain/transformations/fk_helpers.py`
write helper-column provenance into `_meta["derived"]["sheets"]`.
`rendering/passes/_base.py::_derived_helper_column_names` reads it. This channel is
operational — it is transient (never persisted) and is removed by `drop_helpers` — but
it has no entry in `meta_registry.yaml`. `test_meta_registry` therefore cannot verify
its producer/consumer contract, and any refactor touching these files works against an
undocumented inter-layer dependency. The concept-migration of 2026-05-11 already added
a prose description of this channel to `architecture_primer.adoc` with a
`Status: not yet registered` marker; this FTR resolves that marker.

==== Current State (verified)

* `domain/transformations/enrich_lookup.py` — writes
  `_meta["derived"]["sheets"][sheet_name]["helper_columns"]`. (IR-005)
* `domain/transformations/fk_helpers.py` — writes same key path. (IR-005)
* `rendering/passes/_base.py::_derived_helper_column_names` — reads
  `_meta["derived"]["sheets"]`. (IR-005)
* `drop_helpers` — removes `_meta["derived"]` before output; channel is transient and
  not roundtrip-relevant. (IR-005)
* `docs/internal_guide/architecture/meta_registry.yaml` — no entry for `derived`;
  `test_meta_registry` does not cover this channel. (IR-005)
* `docs/ai_info/architecture_primer.adoc` § "meta_canonical vs meta_rendering" —
  `derived`-channel paragraph present with `Status: not yet registered`
  (added 2026-05-11 concept migration). This FTR resolves that status.
* `docs/ai_info/concepts_fk_helpers.adoc` — describes the
  `_meta.derived.sheets.helper_columns` shape (same migration).

==== Scope

. Open `docs/internal_guide/architecture/meta_registry.yaml`; add an entry for
  `derived` with at minimum:
  * `name: derived`
  * `layer: meta_rendering`
  * `classification: derived_operational_view`
  * `lifecycle: transient` — written before pass execution; removed by `drop_helpers`;
    never persisted to XLSX/output.
  * `producer` — importable refs to `enrich_lookup` and `fk_helpers` write sites.
  * `consumer` — importable ref to
    `rendering.passes._base._derived_helper_column_names`.
. Extend `test_meta_registry` (or its YAML fixture) to assert that the `derived` entry
  exists with `layer == "meta_rendering"` and non-empty `producer` and `consumer`
  lists.
. Update the `Status: not yet registered` note in
  `docs/ai_info/architecture_primer.adoc` § "meta_canonical vs meta_rendering" to
  `Status: registered (meta_registry.yaml)`.
. If `docs/ai_info/concepts_fk_helpers.adoc` or
  `docs/human_info/concepts/fk_helper_metadata.adoc` also carry a `Status:` marker for
  this gap, update those too.

==== Non-Goals

* Do not change the `derived` channel's runtime behaviour or the code that writes/reads
  it.
* Do not register other currently unregistered channels in this FTR.
* Do not alter the `meta_registry.yaml` schema or structure.

==== Dependencies

None. Should be executed **before** any C2/FK refactor touching `fk_helpers.py` (a
producer), so the contract is on record during refactoring.

==== Risk

Negligible — YAML edit and test extension only; no runtime code changes.

==== Acceptance

* `meta_registry.yaml` contains an entry with `name: derived`,
  `layer: meta_rendering`, non-empty `producer` list, and non-empty `consumer` list.
* `test_meta_registry` passes and exercises the `derived` entry.
* `grep 'not yet registered' docs/ai_info/architecture_primer.adoc` returns no matches.
* Full test suite passes.
