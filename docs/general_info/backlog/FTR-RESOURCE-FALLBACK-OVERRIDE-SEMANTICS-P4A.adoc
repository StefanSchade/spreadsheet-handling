===== FTR-RESOURCE-FALLBACK-OVERRIDE-SEMANTICS-P4A -- Define fallback and empty-cell semantics for resource overrides

*Status:* Open
*Priority:* P4A
*Area:* Domain semantics / localized resources

==== Purpose

Define explicit fallback and override semantics for localized resource
roundtrips.

In a business-facing workbook, context-specific resource columns often behave
as overrides of a default text. Empty cells are ambiguous unless the pipeline
declares what they mean.

Note: `FTR-LOCALIZED-RESOURCE-MATRIX-ROUNDTRIP-P4A` treats empty cells
mechanically and does not define default/override semantics. Those policies
remain owned by this FTR.

This FTR should deliver a separate normalization step such as
`normalize_resource_overrides`. A later composite localized-resource roundtrip
may call that step, but the policy should remain testable on its own.

==== Semantics To Define

* default context versus non-default override context
* missing override versus explicit empty text
* deleting an override
* keeping an override that equals the default
* required default values per locale
* optional text values where empty text is valid

==== Example Policy

[source,yaml]
----
resource_override_policy:
  default_context: default
  default_required: true
  empty_override: omit_tuple
  explicit_empty_marker: "<empty>"  # configurable example marker
  collapse_override_equal_to_default: true
----

==== Goals

* Make empty-cell behavior explicit and testable.
* Normalize workbook cells into canonical tuple output according to policy.
* Produce findings for ambiguous or invalid states.
* Allow conservative defaults for business editing workflows.
* Keep the policy generic for any localized or context-overridden resource.

==== Non-goals

* Do not implement application-specific fallback chains beyond configured axes.
* Do not translate texts or validate linguistic quality.
* Do not require all consumers to use the same empty-text marker.

==== Acceptance

* Policy-driven normalization distinguishes missing overrides from explicit
  empty text.
* Required defaults are validated per configured locale/resource set.
* Duplicate or conflicting overrides produce clear findings.
* Tests cover omitted override, explicit empty override, override equal to
  default, required default missing, and optional empty text.
* The policy can be used by localized resource matrix recomposition.
