===== FTR-RESOURCE-FALLBACK-OVERRIDE-SEMANTICS-P4A -- Define fallback and empty-cell semantics for resource overrides

*Status:* Implemented (minimal normalization slice)
*Priority:* P4A
*Area:* Domain semantics / generic resource overrides

==== Purpose

Define explicit default-context and override semantics for generic resource
tuple roundtrips.

In a business-facing workbook, context-specific resource columns often behave
as overrides of a default text. Empty cells are ambiguous unless the pipeline
declares what they mean.

Note: `FTR-LOCALIZED-RESOURCE-MATRIX-ROUNDTRIP-P4A` treats empty cells
mechanically and does not define default/override semantics. Those policies
remain owned by this FTR.

*Scope Boundary:* This FTR assumes the matrix/crosstable frame already contains
the configured row and matrix axes (all rows and columns exist as structure).
Generating or preserving full sparse matrix shape from explicit axis sources
belongs to `FTR-XREF-DENSE-AXES-P4A2`. This FTR works on the *values* within
an already-shaped matrix, not on the matrix *structure* itself.

This FTR delivers a separate normalization step,
`normalize_resource_overrides`. A later composite localized-resource roundtrip
may call that step, but the policy remains testable on its own.

The first implementation slice is intentionally generic. The localized-resource
shape (`resource_key`, `locale`, `context_id`, `text`) appears only in tests and
examples. Production code uses configured `row_keys`, `discriminator_column`,
`context_column`, and `value_column`.

==== Semantics To Define

* default context versus non-default override context
* missing override versus explicit empty text
* deleting an override
* keeping an override that equals the default
* required default values per locale
* optional text values where empty text is valid

==== Implemented Minimal Slice

* Pure domain helper: `normalize_resource_override_frame`.
* Pipeline step: `normalize_resource_overrides`.
* Supported configured columns: `row_keys`, `discriminator_column`,
  `context_column`, `value_column`.
* Supported policy fields:
  `default_context`, `default_required`, `empty_override`,
  `explicit_empty_marker`, `collapse_override_equal_to_default`,
  `allow_empty_default`.
* Supported `empty_override` values: `omit_tuple`, `keep_tuple`.
* Supported modes: `fail`, `warn`, `ignore`.
* The step preserves extra columns on retained tuples and only removes tuples
  because of explicit override policy.

Dense/full matrix shape reconstruction remains owned by
`FTR-XREF-DENSE-AXES-P4A2`; this FTR does not change `contract_xref` or
`expand_xref`.

==== Example Policy

[source,yaml]
----
resource_override_policy:
  default_context: default
  default_required: true
  empty_override: omit_tuple
  explicit_empty_marker: "<empty>"  # configurable example marker
  collapse_override_equal_to_default: true
----

==== Goals

* Make empty-cell behavior explicit and testable.
* Normalize workbook cells into canonical tuple output according to policy.
* Produce findings for ambiguous or invalid states.
* Allow conservative defaults for business editing workflows.
* Keep the policy generic for any localized or context-overridden resource.

==== Non-goals

* Do not implement dense-axis reconstruction or matrix shape materialization.
* Do not alter `contract_xref` / `expand_xref` matrix-shape behavior.
* Do not implement `.properties` or artifact export.
* Do not introduce a localized-resource-specific production framework.
* Do not implement application-specific fallback chains beyond configured axes.
* Do not translate texts or validate linguistic quality.
* Do not require all consumers to use the same empty-text marker.

==== Acceptance

* Policy-driven normalization distinguishes missing overrides from explicit
  empty text.
* Required defaults are validated per configured locale/resource set.
* Duplicate or conflicting overrides produce clear findings.
* Tests cover omitted override, explicit empty override, override equal to
  default, required default missing, and optional empty text.
* The policy can be used by localized resource matrix recomposition.
