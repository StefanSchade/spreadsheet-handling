===== FTR-REVIEW-001-PASSES-CLEANUP-P3 — Split `passes/core.py`, delete stubs, single `default_passes()` source

*Status:* Open
*Priority:* P3
*Area:* rendering/passes
*Kind:* Refactoring / Hygiene
*Review source:* REVIEW-001, Cluster C3, Finding IDs: IR-003, IR-004

==== Purpose

`rendering/passes/core.py` (518 lines, 8 pass classes) is the only real implementation
of IR passes, but the `passes/` package contains per-pass stub files implying a
module-per-pass layout. `formula_pass.py` and `post_pass.py` are empty;
`meta_pass.py` and `style_pass.py` are No-op stubs; `validation_pass.py` (80 lines) is
an outdated parallel implementation. Additionally, `passes/__init__.py::apply_all` and
`flow.py::default_p1_passes` maintain two manually-synchronized pass lists with no test
enforcing their equality. A developer or agent creating a new `passes/<name>.py` will not
wire it up — all 8 passes execute through `core.py`. This FTR makes the actual structure
match the implied one.

==== Current State (verified)

* `rendering/passes/core.py` — 518 lines; contains `IRPass` base class, 8 pass classes
  (`MetaPass`, `ValidationPass`, `StylePass`, `ProtectionPass`, `FilterPass`,
  `FreezePass`, `ColumnWidthPass`, `NamedRangePass`), and shared helpers. (IR-003)
* `rendering/passes/formula_pass.py` — empty file. (IR-003)
* `rendering/passes/post_pass.py` — empty file. (IR-003)
* `rendering/passes/meta_pass.py` — No-op stub. (IR-003)
* `rendering/passes/style_pass.py` — No-op stub. (IR-003)
* `rendering/passes/validation_pass.py` — 80 lines, outdated parallel implementation
  superseded by the class in `core.py`. (IR-003)
* `rendering/passes/__init__.py::apply_all` and `rendering/flow.py::default_p1_passes`
  — two manually-maintained pass lists; no test verifies their equality. (IR-004)
* `test_ir_architecture_clarity` — whitelist that requires updating when module layout
  changes.

==== Scope

. Extract the following from `rendering/passes/core.py` into new per-pass modules:
  * `_base.py` — `IRPass` base class + shared helpers
  * `style_pass.py` (replaces No-op stub) — `StylePass`
  * `protection_pass.py` (new) — `ProtectionPass`
  * `filter_pass.py` (new) — `FilterPass`
  * `freeze_pass.py` (new) — `FreezePass`
  * `column_width_pass.py` (new) — `ColumnWidthPass`
  * `validation_pass.py` (replaces outdated impl) — `ValidationPass`
  * `meta_pass.py` (replaces No-op stub) — `MetaPass`
  * `named_range_pass.py` (new) — `NamedRangePass`
. Delete `formula_pass.py`, `post_pass.py`, and `core.py` once all classes are extracted.
. Update `passes/__init__.py` to re-export all pass classes from the new modules; define
  a single `default_passes()` function returning the canonical ordered list.
  Correct complete order: `MetaPass → ValidationPass → StylePass → ProtectionPass →
  FilterPass → FreezePass → ColumnWidthPass → NamedRangePass`.
. Update `rendering/flow.py::default_p1_passes` to call `default_passes()` from
  `passes/__init__.py` — remove the duplicate list.
. Add a test asserting `apply_all` ordering == `default_passes()` ordering.
. Update `test_ir_architecture_clarity` whitelist for the new module layout.
. Run full test suite.

==== Non-Goals

* Do not change the behavior of any pass — structural split only.
* Do not add new passes or modify pass logic.
* Do not rename pass classes.

==== Dependencies

None.

==== Risk

Medium — touching 518 lines of active render logic. Mitigation: extract one pass class
per commit with green tests between steps; all passes are exercised by integration tests.

==== Acceptance

* `rendering/passes/core.py` does not exist (file deleted after full extraction).
* `rendering/passes/formula_pass.py` and `rendering/passes/post_pass.py` do not exist.
* `grep -rl 'class.*Pass' src/spreadsheet_handling/rendering/passes/` returns 8
  distinct files (one class per file).
* `passes/__init__.py` defines exactly one `default_passes()` function;
  `flow.py` calls `default_passes()` rather than maintaining a separate list.
* Test `test_apply_all_order_equals_default_passes` exists and passes.
* Full test suite passes.
