===== FTR-REVIEW-001-BACKEND-DISPATCH-P4A — Consolidate backend dispatch to a single registry

*Status:* Open
*Priority:* P4A
*Area:* io_backends / application / pipeline
*Kind:* Refactoring / Architecture
*Review source:* REVIEW-001, Cluster C5, Finding IDs: A-001, A-002, A-003, A-010, IR-010, IR-012

==== Purpose

Three independent `kind → backend` dispatch tables exist: `router.py`
(`LOADERS`/`SAVERS`), `io_backends/__init__.py` (`_BACKEND_SPECS`/`make_backend`), and
`application/orchestrator.py` (`_load_frames`/`_save_frames` if/elif). These have
diverged: `csv_dir` is reachable via `router` but not via `orchestrate()`, causing a
`ValueError` crash on `sheets-pack --input-kind csv_dir`. Additionally,
`pipeline.runner.run_pipeline(AppConfig)` and `pipeline.registry.run_pipeline(frames,
steps)` share a name with incompatible signatures, and `pipeline/registry.py` conflates
registry data, YAML build logic, and a re-export. This FTR consolidates all dispatch
to a single authoritative registry and eliminates the ongoing drift surface.

==== Current State (verified)

* `io_backends/router.py` — `LOADERS`/`SAVERS` dicts; used by `pipeline/runner.py`.
  (A-001)
* `io_backends/__init__.py` — `_BACKEND_SPECS` / `make_backend`; used in tests and
  some internal paths. (A-001)
* `application/orchestrator.py::_load_frames` / `_save_frames` — if/elif dispatch
  hardcoded; no `csv_dir` branch → `ValueError` crash when
  `sheets-pack --input-kind csv_dir` is invoked. (A-001, A-002)
* `application/orchestrator.py` and `pipeline/runner.py` — two orchestrator entry
  points with diverging `kind` key sets. (A-001)
* `pipeline/runner.py::run_pipeline(AppConfig)` shadows
  `pipeline/registry.py::run_pipeline(frames, steps)` — same name, incompatible
  signatures; `pipeline/__init__.py` exports the registry variant. (A-010)
* `pipeline/registry.py` (261 lines) — mixes `REGISTRY` mapping,
  `build_steps_from_config` / `build_steps_from_yaml` build logic, and a re-export of
  `run_pipeline`. (IR-010)
* `pipeline/config.py` — defines `AppConfig`/`PipelineConfig`/`StepRef.dotted` parallel
  to the `step:`-based `build_steps_from_config` dialect; two YAML dialects coexist
  with `runner.py` bridging between them. (IR-012)
* Lazy import mechanism (`router._lazy_callable`,
  `io_backends/__init__.__getattr__`) — delays `openpyxl` / `odfpy` import until first
  backend use. Must be preserved; ADR to be written in `FTR-OPTIONAL-BACKEND-DEPS-P3K`
  before this registry merge proceeds.

==== Scope

. Designate `io_backends/router.py` as the single canonical `kind → {loader, saver}`
  registry (`LOADERS`, `SAVERS`).
. Rewrite `io_backends/__init__.py::make_backend` to delegate to `router` rather than
  maintaining `_BACKEND_SPECS`; remove `_BACKEND_SPECS` from `__init__.py`.
. Rewrite `application/orchestrator._load_frames` and `_save_frames` to use
  `router.LOADERS` / `router.SAVERS`; remove the if/elif dispatch; add the missing
  `csv_dir` branch (full A-002 fix).
. Split `pipeline/registry.py`:
  * `pipeline/registry.py` — `REGISTRY` dict + `resolve_registration` only.
  * `pipeline/build.py` (new) — `build_steps_from_config`, `build_steps_from_yaml`.
. Rename `pipeline/runner.py::run_pipeline(AppConfig)` → `run_app(AppConfig)`; update
  all callers; export `run_app` from `pipeline/__init__.py`; remove the shadowing name.
. Converge on the `step:`-based YAML dialect as the canonical form; trim
  `pipeline/config.py::AppConfig` to I/O endpoints and Excel options only.
. Verify lazy import behaviour is preserved: add a test asserting that `openpyxl` and
  `odfpy` are not in `sys.modules` after `import spreadsheet_handling` before any
  backend call is made.
. Update all internal callers, architecture guard whitelists, and any tests that
  reference `_BACKEND_SPECS` or the old `run_pipeline(AppConfig)` name.
. Run full test suite including roundtrip integration tests.

==== Non-Goals

* Do not remove the `orchestrate()` public surface —
  `interfaces_and_gates.adoc` gates it.
* Do not change any backend's read/write behaviour.
* Do not remove `AppConfig` — only trim it to I/O + Excel options.
* Do not delete `io_backends/_interfaces.py` here — that is a Quick Win (A-006).

==== Dependencies

* `FTR-OPTIONAL-BACKEND-DEPS-P3K` — the lazy-import decision and ADR must be recorded
  before the registry merge, so the invariant is on paper and cannot be accidentally
  removed during consolidation.

==== Risk

High — touches four subsystems (io_backends, application, pipeline/runner,
pipeline/registry) with multiple integration and architecture tests requiring updates.
Mitigation: keep `orchestrate()` signature unchanged throughout; work one subsystem per
commit; the lazy-import test (Scope step 7) is the key regression guard for the most
subtle invariant.

==== Acceptance

* `grep -rn '_BACKEND_SPECS' src/` returns no matches.
* `grep -rn 'if.*csv_dir\|elif.*csv_dir' src/spreadsheet_handling/application/
  orchestrator.py` returns no matches (if/elif dispatch removed).
* `orchestrate(input_kind="csv_dir", ...)` completes without `ValueError`.
* `pipeline/registry.py` contains only `REGISTRY` + `resolve_registration`;
  `pipeline/build.py` contains `build_steps_from_config` / `build_steps_from_yaml`.
* `grep -rn 'run_pipeline.*AppConfig' src/` returns no matches; `run_app` is callable
  from `pipeline`.
* Lazy-import test: `openpyxl` and `odfpy` are absent from `sys.modules` after bare
  `import spreadsheet_handling`.
* Full test suite passes.
