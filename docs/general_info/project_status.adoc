=== Project Status Notice

*Spreadsheet-Handling* is an **early beta** project.

- Current public version: pre-release on PyPI (0.1.x-beta).
- Primary user and maintainer: internal
- No external adoption known at this point; interfaces may change without deprecation periods.
- The focus is on establishing clean architecture, test coverage, and documentation rather than release cadence.
- Structural refactors and feature deprecations are permitted freely until the first stable release (≥ 1.0.0).
- Compatibility guarantees will start only once the domain and adapter layers have stabilized.

*Implication for planning:*  Features marked as *deprecated* or *to-be-merged* can be removed without migration work.  All roadmap phases (1-4) are internal milestones only and may be reordered as the design evolves.

*API Evolution Policy (Beta)*::

Minor renames and structural refactors are allowed, but every public-facing API change
(domain, adapter, or renderer) should either:
- update all internal callers within the same change set, or
- introduce a short-lived compatibility shim marked `# Deprecated` and schedule its removal in the next FTR-CLEANUP phase.

Frequent spontaneous renames without tracking create technical debt; therefore,
changes to stable module boundaries (e.g., io_backends.*, rendering.*) should be
recorded through an FTR entry.
