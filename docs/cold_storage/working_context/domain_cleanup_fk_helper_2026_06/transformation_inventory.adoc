= Transformation Inventory

== xref_crosstable

Description::
Generic bidirectional matrix ↔ long-form relation conversion.  Treats cell
values as opaque.  Supports dense-axis reconstruction from auxiliary frames.

Public API::
`expand_xref`, `contract_xref`
(module: `domain/transformations/xref_crosstable.py`)

Input Frames::
`matrix` (wide DataFrame with flat columns; e.g. columns `feature_id`,
`P-001`, `P-002`).  For `contract_xref`: `relation` (long-form with
`row_keys`, `column_key`, `value`).  Optional dense-axis frames referenced
by `dense_axes.rows_from.frame` and `dense_axes.columns_from.frame`.

Output Frames::
`output` (long-form relation for `expand_xref`; matrix for `contract_xref`).

Modified Frames::
None; always returns a new frames dict.

Reads Meta::
yes — reads `_meta.xref_crosstable.<config_id>` to recover stored `column_keys`
and `dense_axes.resolved` on `contract_xref`.

Writes Meta::
yes — writes `_meta.xref_crosstable.<config_id>` with `operation`, `matrix`,
`relation`, `row_keys`, `column_keys`, and optional `dense_axes` payload.

Important Columns::
`row_keys` (pk of the row axis), `column_key` (discriminator for matrix
columns), `value` (cell value).

Local Helpers::
None; self-contained in the single file.

Direct Dependencies::
None outside standard library and pandas.

Comprehension Notes::
`dense_axes` adds significant complexity: two optional source frames define
the complete row and column domains so the contracted matrix is fully dense
even when the relation is sparse.  Dense-axis resolved values are stored in
`_meta` to allow offline roundtrip when the axis frames are not present at
contraction time.  `_values_equal` uses a try/except instead of `==` to
handle numpy scalars without dtype coercion failures.

== cell_codec

Description::
Generic cell-value ↔ structured code/token conversion.  Validates syntax and
configured value sets.  Does not assign business meaning to codes.

Public API::
`parse_cell_value`, `serialize_cell_value`, `decode_cell_values`,
`encode_cell_values`, `ParsedCellValue`
(module: `domain/transformations/cell_codec.py`)

Input Frames::
`source` (flat-column DataFrame with at least a `value` column for
`decode_cell_values`; `source` with `group_by` + `code` columns for
`encode_cell_values`).

Output Frames::
`output` (decoded rows with passthrough columns + `code`;
or encoded rows with `group_by` columns + `value`).

Modified Frames::
None.

Reads Meta::
yes — reads `_meta.legend_blocks.<name>` when `allowed_from_legend` is set.

Writes Meta::
yes — writes `_meta.cell_codecs.<config_id>` with operation, source, output,
mode, delimiter, and the resolved allowed values/tokens.

Important Columns::
`value` (raw compact cell string), `code` (output token column),
`group_by` (grouping identity for encoding).

Local Helpers::
All helpers (`_cell_to_text`, `_canonicalize_values`, `_legend_tokens`, etc.)
are private within the same file.

Direct Dependencies::
None outside pandas.

Comprehension Notes::
Two modes: `whole_cell_code` (entire cell is one code) and `split_tokens`
(cell contains delimiter-separated tokens).  Mixing `allowed_codes` with
`allowed_tokens` is not guarded — callers must use the right parameter for
the mode.  `_ordered_groups` uses a linear scan instead of a dict to preserve
insertion order and handle unhashable identity values.

== compact_multiaxis

Description::
Composes `xref_crosstable` and `cell_codec` to handle compact matrix forms
where cell values carry multi-token codes.  Adds optional group-column
derivation from legend.

Public API::
`expand_compact_multiaxis`, `contract_compact_multiaxis`
(module: `domain/transformations/compact_multiaxis.py`)

Input Frames::
`matrix` (for expand); `relation` (for contract).

Output Frames::
`output`.

Modified Frames::
None; intermediate `__compact_multiaxis_*` frame is created and then popped
before returning.

Reads Meta::
yes — via `cell_codec` (legend_blocks) and via `xref_crosstable` (column_keys).

Writes Meta::
yes — writes `_meta.compact_multiaxis.<config_id>` and delegates meta writes
to `cell_codec` and `xref_crosstable`.  Also writes `_meta.frame_lifecycle`.

Important Columns::
`row_keys`, `column_key`, `value`, `code`, optional `group`.

Local Helpers::
`_with_group_column`, `_group_lookup`, `_legend_groups` — all private in the
same file.

Direct Dependencies::
`domain/transformations/cell_codec.py` (`decode_cell_values`, `encode_cell_values`)
`domain/transformations/xref_crosstable.py` (`contract_xref`, `expand_xref`)
`domain/frame_lifecycle.py` (`mark_source_if_unclassified`, `write_frame_lifecycle`)

Comprehension Notes::
`dense_axes` parameter is declared but raises `NotImplementedError` — it is

check (`_temp_frame_name`) guards against accidental overwrites but adds
surprising logic for callers who reuse the same frame names.  Group lookup
duplicates much of `cell_codec._legend_tokens` logic locally.

== discriminator_split

Description::
Partition one frame into per-discriminator-value frames (split) and reverse
(merge).  Preserves original row order across split/merge when
`preserve_row_order=True`.

Public API::
`split_by_discriminator`, `merge_by_discriminator`
(package: `domain/transformations/discriminator_split/`)

Input Frames::
`source_frame` (flat-column DataFrame; discriminator must be scalar
and non-empty).  For merge: frames matching `source_pattern` or listed in
`source_frames`.

Output Frames::
Per-discriminator-value frames (split); `target_frame` (merge).

Modified Frames::
None.

Reads Meta::
yes — `merge_by_discriminator` reads `_meta.split_by_discriminator.<config_id>`
to recover `values`, `column_order`, and `row_order` when neither `value_map`
nor `source_frames` is given.

Writes Meta::
yes — both write `_meta.split_by_discriminator.<config_id>` with operation,
source/target, values, column_order, and row_order.

Important Columns::
`discriminator_column` (the column whose values drive the partition).

Local Helpers::
`framecheck.py` (structural guards), `metadata.py` (read/write plumbing),
`naming.py` (pattern validation, safe segment derivation, value→frame name),
`values.py` (scalar normalization, equality, sort tokens).
These are internal package implementation details; callers use only the two
re-exported names.

Direct Dependencies::
None outside pandas.

Comprehension Notes::
Split columns exclude the discriminator; merge re-introduces it.  This means
the discriminator column disappears from the split frames — a source of
confusion.  `merge_by_discriminator` enforces uniform column sets across all
source frames and raises on any deviation rather than attempting to reconcile.
The `values.py._sort_token` distinguishes numeric vs. string values for
stable sort ordering across Python type changes.

== fk_helpers

Description::
Full FK-helper lifecycle: enrichment (materializing helper columns driven by
v2 policy), cleanup (dropping helper columns after reimport), and reordering
(positioning helpers next to their FK column).

Public API::
`enrich_helpers`, `drop_helpers`, `resolve_v2_fk_relations`,
`derived_helper_columns_by_sheet`, `MissingFkRelationPolicyError`,
`missing_fk_policy_error`, `build_v2_target_registry`, `_write_helper_provenance`
(package: `domain/transformations/fk_helpers/`)

Also exposed via `domain/transformations/helpers.py`:
`mark_helpers`, `clean_aux_columns`, `flatten_headers`, `unflatten_headers`,
`reorder_helpers_next_to_fk`

Input Frames::
All data frames in the pipeline for `enrich_helpers`/`drop_helpers`.

Output Frames::
Same frame names (columns added or removed in place within new frames dict).

Modified Frames::
Semantically the same frames are returned with modified column sets.

Reads Meta::
yes — `_meta.helper_policies.fk.relations` (schema_version 2) and
`_meta.derived.sheets.<sheet>.helper_columns`.

Writes Meta::
yes — `enrich_helpers` writes `_meta.derived.sheets.<sheet>.helper_columns`
via `_write_helper_provenance`; `drop_helpers` cleans those same entries.

Important Columns::
`source_column` (FK column on source frame), `target_key` (PK column on
target frame), helper columns named per `helper_columns[*].column`.

Local Helpers::
`enrich.py`, `drop.py`, `policy.py`, `provenance.py`, `formula_provider.py`.
`helpers.py` in the parent `transformations/` is related but is technically
a sibling module, not an internal helper.

Direct Dependencies::
`core/fk.py` (`apply_fk_helpers`, `build_id_value_maps`, `FKDef`,
`normalize_sheet_key`)
`core/formulas.py` (`lookup_formula`) via `formula_provider.py`
`core/indexing.py` via `validations/fk_helpers.py`
`domain/frame_keys.py` (`copy_reserved_frames`, `iter_data_frames`)

Comprehension Notes::
`prefix` parameter in `drop_helpers` and `reorder_helpers_next_to_fk` is
retained for YAML backward compatibility but is no longer used on any normal
path — this is a silent dead parameter that could mislead authors.
`enrich_helpers` silently skips relations whose `source_column` is not yet
present in the frame; this is by design for partial pipeline states but
introduces non-obvious silent no-ops.  The v2 policy requirement is
intentionally strict: missing policy raises `MissingFkRelationPolicyError`
with an actionable message.

== enrich_lookup

Description::
Joins a source frame with a lookup frame by explicit keys and projects
configured helper fields.  Supports value mode (merged copy) and formula
mode (stores `LookupFormulaSpec` objects for live XLOOKUP in rendered
workbooks).

Public API::
`enrich_lookup`
(module: `domain/transformations/enrich_lookup.py`)

Input Frames::
`source`, `lookup`.

Output Frames::
`output`.

Modified Frames::
None.

Reads Meta::
yes — reads `_meta.helper_policies.lookup.<lookup>` for key, missing mode,
order, allowed/default helpers.

Writes Meta::
yes — writes `_meta.derived.sheets.<output>.enrich_lookup` with lookup,
join keys, and helper columns.  Also writes `_meta.frame_lifecycle`.

Important Columns::
Join key columns (declared via `key`/`keys`/`on`), helper `fields` from the
lookup frame.

Local Helpers::
All helpers are private within the same file.

Direct Dependencies::
`core/formulas.py` (`lookup_formula`)
`domain/frame_lifecycle.py` (`mark_source_if_unclassified`, `write_frame_lifecycle`)

Comprehension Notes::
Three join key parameters (`key`, `keys`, `on`) are all valid but mutually
exclusive — the `on` spelling is discouraged in YAML due to YAML 1.1 boolean
coercion.  Policy-based conflict checking: inline key/missing/order that
differ from a matching policy entry raise rather than silently override.
Formula mode and `missing='fail'` are incompatible (raises immediately).
Sort by lookup-frame columns is also incompatible with formula mode.

== derived_column_policy

Description::
Reimport cleanup: drops helper/derived columns from a payload frame after
spreadsheet editing, optionally checking `enrich_lookup` helper values
against their source lookup frame.

Public API::
`apply_derived_column_policy`, `enforce_derived_column_policy_frame`,
`DerivedColumnFinding`
(module: `domain/transformations/derived_column_policy.py`)

Input Frames::
`source` (payload frame after user editing).  Lookup frames referenced
by `_meta.derived.sheets.<source>.enrich_lookup.lookup` are read by name
from the frames dict.

Output Frames::
`output` or `source` (in-place replacement by default);
`findings` (only in `warn_on_mismatch` mode).

Modified Frames::
`_meta` is mutated to strip consumed `derived.sheets[source]` when the
source frame is replaced in-place.

Reads Meta::
yes — `_meta.derived.sheets.<source>` (transient provenance),
`_meta.sheets.<source>.helper_columns` (durable fallback),
`_meta.helper_policies.fk` (v2 policy fallback via `resolve_v2_fk_relations`).

Writes Meta::
yes — strips `_meta.derived.sheets.<source>.helper_columns` and
`enrich_lookup` after cleanup.

Important Columns::
Helper column names drawn from metadata; no column names are structurally
hardcoded.

Local Helpers::
All helpers are private within the same file.

Direct Dependencies::
`domain/finding_frame.py` (`Finding`, `findings_to_frame`, `FINDING_COLUMNS`,
`simple_failure_message`)
`domain/transformations/fk_helpers/policy.py` (`resolve_v2_fk_relations`)

Comprehension Notes::
Three-layered fallback for identifying helper columns: (1) transient
`_meta.derived`, (2) durable `_meta.sheets.*.helper_columns`, (3) v2 FK
policy.  Value checking is only available for `enrich_lookup` helpers and only
when transient provenance is present.  FK helpers are dropped silently without
value comparison in all cases.

== join_views

Description::
Bounded declarative join of two frames into a materialized view frame.
Supports `left`, `inner`, and `semi` join modes.  Handles column collision via
fail-or-suffix policy.  Supports row filtering via `where` predicate.

Public API::
`join_frames`
(module: `domain/transformations/join_views.py`)

Input Frames::
`left`, `right`.

Output Frames::
`output`.

Modified Frames::
None.

Reads Meta::
no.

Writes Meta::
yes — writes `_meta.frame_lifecycle`.

Important Columns::
Join key columns (various spellings: `key`, `keys`, `left_key`/`right_key`,
`left_keys`/`right_keys`).

Local Helpers::
`_ColumnSpec`, `_side_work_frame`, `_semi_join`, `_apply_where`, `_join_token`
— all private in the same file.

Direct Dependencies::
`domain/frame_lifecycle.py` (`mark_source_if_unclassified`, `write_frame_lifecycle`)

Comprehension Notes::
Empty cells are treated as non-matching join keys (`_join_token` returns
a fresh `object()` so they never match, even NULL==NULL).  This is a
deliberate design choice to avoid false FK matches.  `validate=m:1` is only
applied when `right_unique` is True or defaulted on non-semi joins.  Semi join
uses an internal numbered-row approach to handle duplicate left rows correctly.

== project_by_role

Description::
Role-based column projection: reorder or drop columns of a named frame
according to the `row_identity`, `display_helper`, `matrix_value` taxonomy.
Supports `outbound` (all three roles) and `inbound` (drop display_helper)
directions.

Public API::
`project_by_role`
(module: `domain/transformations/project_by_role.py`)

Input Frames::
`frame` (any DataFrame in the pipeline frames dict).

Output Frames::
Same `frame` name (the frame is replaced with the reordered version).

Modified Frames::
The named frame is replaced in the returned frames dict.

Reads Meta::
yes — indirectly, via `resolve_column_roles` which reads `_meta.xref_crosstable`
(for `row_identity`), `_meta.workbook_view.sheets`, `_meta.sheets.*.helper_columns`
(for `display_helper`), and `_meta.derived.sheets.*.helper_columns` (FK helpers).

Writes Meta::
no.

Important Columns::
`helper_columns` (explicit override for display_helper role),
`key_columns` (explicit override for row_identity role).

Local Helpers::
`_outbound_columns`, `_inbound_columns`, `_validate_direction`,
`_validate_role_order` — all private in the same file.

Direct Dependencies::
`domain/column_roles/__init__.py` + `resolver.py`
(`resolve_column_roles`, `ROLE_DISPLAY_HELPER`, `ROLE_MATRIX_VALUE`,
`ROLE_ROW_IDENTITY`, `ROLE_NAMES`, `ColumnRoles`)

Comprehension Notes::
Step is declared as in-place ("no source:/output: parameter") but returns a
new frames dict — this is terminology only; no mutation occurs.  Missing
`row_identity` is a warning, not an error; the step proceeds with an empty
identity set.  Role ordering for `outbound` defaults to
`(display_helper, row_identity, matrix_value)`; callers may permute via
`role_order`.

== column_roles

Description::
Foundation column-role taxonomy: classifies frame columns into
`row_identity`, `display_helper`, and `matrix_value` by reading scattered
upstream metadata.  Single source of truth for role detection consumed by
`project_by_role` and `add_validations`.

Public API::
`resolve_column_roles`, `ColumnRoles`, `UnknownRoleError`,
`ROLE_ROW_IDENTITY`, `ROLE_DISPLAY_HELPER`, `ROLE_MATRIX_VALUE`, `ROLE_NAMES`
(package: `domain/column_roles/`)

Input Frames::
Any frames dict; the named `frame` must be a DataFrame.

Output Frames::
None (returns a `ColumnRoles` dataclass).

Modified Frames::
None.

Reads Meta::
yes — reads `_meta.xref_crosstable` (row_keys), `_meta.workbook_view.sheets`
and `_meta.sheets.*.helper_columns` (view helpers), and
`_meta.derived.sheets.*.helper_columns` (FK provenance helpers).

Writes Meta::
no.

Important Columns::
None structurally fixed; classification comes entirely from metadata.

Local Helpers::
`resolver.py` (all logic); `__init__.py` re-exports.

Direct Dependencies::
`domain/transformations/fk_helpers/policy.py` (`derived_helper_columns_by_sheet`)

Comprehension Notes::

is absent.

== schema_maintenance

Description::
Column-level schema operations on data frames (add, drop, rename, reorder)
with automated propagation of changes into `_meta` reference paths
(constraints, helper_policies.fk, workbook_view, frame_lifecycle, etc.).

Public API::
`apply_schema_maintenance`, `add_column`, `drop_column`, `rename_column`,

`SchemaOperationKind`, `WriteIntent`, `ColumnPlacement`, `ReorderSpec`
(package: `domain/schema_maintenance/`)

Input Frames::
`target_frame` (the frame being modified).

Output Frames::
Same frame name with the column change applied (or unchanged on failure).

Modified Frames::
`target_frame` (column add/drop/rename/reorder) and `_meta` (reference
propagation).

Reads Meta::
yes — `_meta` is traversed to find all structured references to the affected
column.

Writes Meta::
yes — updates or prunes references across all known `_meta` root paths.

Important Columns::
`source_column` (old name), `target_column` (new name for rename/add).

Local Helpers::
`columns.py` (frame mutations), `meta_update.py` (metadata rule application),

`model.py` (dataclasses).

Direct Dependencies::
None outside pandas and standard library (self-contained domain package).

Comprehension Notes::

schema maintenance decisions (recorded as `IGNORED_DERIVED` in the report).
`WriteIntent` is modeled but DRY_RUN vs WRITE behavior must be enforced by

== fk_relations / helper_policies

Description::
Heuristic FK relation inference (`infer_fk_relations`) and explicit FK/lookup
helper policy configuration (`configure_lookup_helpers`, `configure_fk_helpers`).
Produces the v2 `_meta.helper_policies.fk` policy block consumed by all
FK-helper primitives.

Public API::
`infer_fk_relations`
(module: `domain/fk_relations.py`)

`configure_lookup_helpers`, `configure_fk_helpers`
(module: `domain/helper_policies.py`)

Input Frames::
All data frames (for `infer_fk_relations`); named `lookup` or source frames
(for configure steps).

Output Frames::
Returns frames dict with `_meta.helper_policies` updated.

Modified Frames::
`_meta` only.

Reads Meta::
yes — existing `_meta.helper_policies.fk` is read and replaced.

Writes Meta::
yes — writes `_meta.helper_policies.fk` (schema_version 2) with
`relations` list, `produced_by`, and `schema_version`.

Important Columns::
`id_columns` (target PK patterns), `fk_patterns` (FK column naming patterns).

Local Helpers::
`domain/fk_relations.py` owns `apply_v2_relations`, `build_v2_relation`.

Direct Dependencies::
`core/fk.py` (`normalize_sheet_key`)
`domain/frame_keys.py` (`iter_data_frames`)

Comprehension Notes::
`infer_fk_relations` is the only heuristic step in the domain; all FK-helper
primitives are deterministic given policy.  The heuristic uses
`naming_convention` mode only.  The pattern `id_({target})` matches FK columns
like `id_customer` that reference a frame named `customer`.  Ambiguous
inference (multiple matches) raises unless `on_ambiguous='ignore'`.

== frame_lifecycle

Description::
Shared utility for writing and reading `_meta.frame_lifecycle` entries.
Every transformation that produces a named output frame writes a lifecycle
entry describing its role, canonical flag, render hint, and provenance.

Public API::
`write_frame_lifecycle`, `mark_source_if_unclassified`, `frame_lifecycle`,
`FRAME_LIFECYCLE_KEY`, `WORKBOOK_VIEW_KEY`
(module: `domain/frame_lifecycle.py`)

Input Frames::
Any frames dict (via `MutableMapping` signature).

Output Frames::
Mutates `_meta.frame_lifecycle` in the frames dict.

Modified Frames::
`_meta`.

Reads Meta::
yes — reads existing `_meta.frame_lifecycle.<frame>` to honor
`preserve_existing_canonical`.

Writes Meta::
yes — writes `_meta.frame_lifecycle.<frame>`.

Important Columns::
None; operates on frame names, not column names.

Local Helpers::
`_meta_dict`, `_unique_strings`, `mark_source_if_unclassified`.

Direct Dependencies::
Standard library only.

Comprehension Notes::
`preserve_existing_canonical=True` (default) means a frame already marked
canonical cannot be demoted to non-canonical — subsequent calls are silently
ignored for the `canonical` field.  This is load-bearing for pipeline
step ordering safety but can mask bugs where a step tries to reclassify a frame.
