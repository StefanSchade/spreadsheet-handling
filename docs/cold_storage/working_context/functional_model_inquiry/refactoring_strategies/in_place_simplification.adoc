= In-Place Simplification

== Description

Refactor and delete inside the current implementation without introducing a new
domain/meta module boundary. Use the Functional Model to identify concepts that
can be merged, removed, or renamed in place.

== Pros

* Least disruptive.
* Can quickly remove obvious duplication.
* Preserves all current tests and call paths.
* Useful before a rewrite when a path is clearly obsolete.

== Cons

* May spend too much effort fighting current structure.
* Risk of preserving architectural sediment because edits remain local.
* Harder to establish a clean new model.
* May not reduce complexity enough if the problematic boundaries are deep.

== Assessment

This is useful as supporting work, especially for obvious deletion candidates.
It is probably too weak as the main strategy for domain/meta simplification.
