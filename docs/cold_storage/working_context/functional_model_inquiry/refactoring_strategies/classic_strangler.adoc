= Classic Strangler

== Description

Keep the existing public runtime surface and gradually route individual old
paths through new implementations. Old and new implementations coexist until
each old path is replaced behind stable contracts.

== Pros

* Low compatibility risk.
* Allows replacement one vertical slice at a time.
* Keeps old behavior available while new contracts mature.
* Useful when external adoption or API stability is important.

== Cons

* Long coexistence can preserve the old complexity for too long.
* Compatibility layers may encode the same sediment the rewrite is meant to
  remove.
* Slower than necessary if breaking changes are acceptable.
* Can become a second architecture instead of a replacement.

== Assessment

This strategy is safer than required for the current project. It remains useful
as a technique for risky slices, but should not dominate the rewrite strategy.
