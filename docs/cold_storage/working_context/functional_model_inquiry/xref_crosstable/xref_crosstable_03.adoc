== XRef and Projection Scope Finding

compare document app_boundary_meta_authority

The XRef reformation discussion exposed a broader Functional Model gap.

XRef empty-cell semantics can be controlled by `drop_empty`:

* `drop_empty=True`: empty matrix cells produce no relation rows;
* `drop_empty=False`: empty matrix cells are preserved as empty-valued relation
  rows.

However, missing matrix rows or columns are different from empty cells.

A missing row or column during reimport may mean:

* it was outside the original projection scope;
* it was in scope and the user intentionally deleted it;
* the workbook projection structure was damaged;
* the corresponding entity was deleted or renamed in a source/entity frame;
* a relation exists but its axis entity no longer exists.

Without projection/writeback scope information, recomposition cannot always
distinguish these cases.

Therefore resolved XRef information such as `column_keys` or
`dense_axes.resolved` may be legitimate projection-scope evidence. It must not
be promoted to durable canonical truth, but it also must not be deleted merely
because it is not transformation intent.
