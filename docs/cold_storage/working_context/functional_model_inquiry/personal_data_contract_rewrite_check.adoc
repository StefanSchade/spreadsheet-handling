= Personal Data Contract Rewrite Check

== Purpose

Short operational note on whether a contract-first domain/meta rewrite is
expected to support the adjacent `personal_data` consumer repository.

This note is temporary working context. It is useful while choosing rewrite
contracts and can become obsolete once the consumer flow is covered by tests.

== Current Consumer Usage

The `personal_data` repository uses a deliberately simple spreadsheet
roundtrip pattern:

* `json_dir` input;
* reference validation before workbook export;
* workbook-view declaration for frame-to-sheet selection and order;
* simple workbook validation constraints, mostly `in_list`;
* spreadsheet export, currently XLSX by default;
* workbook-view sheet/frame remapping on reimport;
* reference validation after reimport;
* `json_dir` output;
* external semantic comparison that ignores `_meta`, normalizes list order, and
  normalizes date-like midnight timestamps.

Observed pipeline steps:

* `validate_references`;
* `configure_workbook_view`;
* `add_validations`;
* `apply_workbook_view_sheet_mappings`.

No current use was found for FK helpers, XRef, Cell Codec, Legend Blocks,
Compact Multiaxis, joins, extracts, pivots, discriminator split/merge,
hierarchical config projection, artifact publishing, or pipeline plugins.

== Expected Rewrite Coverage

This consumer should be covered by the minimal contract-first rewrite path if
the rewrite preserves:

* JSON directory frame IO;
* spreadsheet rendering and reading;
* workbook-view boundary semantics;
* simple workbook allowed-value constraints;
* primary-key, unique, and foreign-key validation with `allow_empty`;
* `_meta` isolation from ordinary payload data.

`personal_data` is therefore a good early adoption contract. It exercises the
core roundtrip without the heavier transformation families used by
Worldbuilding.

== Operational Gaps To Watch

* Workbook-view sheet/frame identity must remain deterministic.
* Simple `in_list` constraints must survive the rewrite.
* Reimport output must remain comparable after ignoring `_meta`.
* Adapter behavior for dates and empty cells must not be accidentally changed.
* Breaking public step syntax is possible, but migration should be cheap and
  explicit because this consumer has many small dataset pipelines.

== Assessment

`personal_data` is a strong minimal consumer contract for the rewrite. It
should probably run before the Worldbuilding contract because it tests the
basic roundtrip boundary, workbook view, validations, and adapters without
mixing in advanced transformations.
