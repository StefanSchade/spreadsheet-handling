== Typed Cell Values

The current MVP stance is string-first.

Visible workbook values are treated as strings unless a specific transformation
or backend contract explicitly says otherwise. This keeps roundtrip behavior
simple and avoids accidental type coercion.

Future support may gradually add a small set of spreadsheet-oriented scalar
types, especially:

* date;
* number;
* boolean, if needed later.

The type model should follow ordinary spreadsheet functionality. It should not
introduce an overly rich object model for values.

## [source,text]

The intended future direction is pragmatic: no complex data structures (rgb colours etc) just the typical date / text / number separation found in spreadsheets.

## [source,text]

Typed values must not be inferred heuristically from cell text in a way that
changes meaning across roundtrips. Type handling should be configured or derived
from explicit schema/metadata, and unsupported types should remain strings or
fail with clear diagnostics.
