== Import Graph

New domain-root utilities:

* ``domain/_cell_primitives.py`` - leaf module, imports only ``pandas`` and
  ``typing``. Defines ``_is_empty_cell``, ``_values_equal``. 14 importers
  across the domain (transformations, validations, structured_yaml,
  xref_crosstable sub-package). All one-way.
* ``domain/_where_predicates.py`` - depends only on
  ``domain/_cell_primitives``. Defines ``_apply_where``,
  ``_WHERE_PREDICATES``, ``_ensure_columns``, ``_ensure_boolean_predicate``,
  ``_duplicate_column_names``. Three importers (``extractions/frame_extract``,
  ``transformations/join_views``, ``transformations/tabular_views``). One-way.

New packages:

* ``transformations/xref_crosstable/`` DAG:
+
----
__init__  → operation
operation → dense_axes, primitives
dense_axes → primitives
primitives → (no package-internal imports)
----
+
The earlier ``operation``/``dense_axes`` load-order workaround was
removed when ``primitives.py`` was introduced. No cycle remains.

* ``transformations/enrich_lookup/`` DAG:
+
----
__init__  → operation
operation → policy
policy   → (no package-internal imports)
----

Cross-cluster edges that existed before the wave (``compact_multiaxis ->

strings, ``derived_column_policy`` reading ``_meta.derived.sheets.*.

unchanged.

Programmatic verification (``importlib.import_module`` of every new

``contract_xref``, ``enrich_lookup``) succeeds at module-load time. No
cycle was introduced by any slice.

== Readability

Net change is positive on every metric the slices targeted:

* **Verbatim duplication removed**: ``_is_empty_cell`` (8 copies →
  1), bare ``_values_equal`` (4 copies → 1), ``_apply_where`` and its
  three companions (2 copies → 1), ``_duplicate_column_names`` (3
  copies → 1).
* **Two large flat modules became packages** with one clearly-named
  secondary submodule each. The xref ``dense_axes`` chunk (~335 LOC, the
  most intricate corner of the file) reads as a unit; the enrich_lookup
  policy-resolution cluster (~190 LOC, the densest concentration of
  conflict-checking logic) does the same.
* **No body or signature was rewritten** for readability. Function
  ordering inside ``xref_crosstable/operation.py`` and
  ``enrich_lookup/operation.py`` was preserved relative to the original
  source order, modulo the dense_axes/policy moves.
* One "rewrite" did slip in during the enrich_lookup slice
  (three ``_check_*`` bodies). It was caught by the existing test
  suite and reverted to the verbatim originals before the commit
  landed. Closing-state ``operation.py`` matches the pre-slice bodies.
* The five-slice cadence (small, behaviour-preserving, test-green) is
  itself a readability win for future contributors: the audit log
  records why each slice was small.

Readability cost: small. The new ``primitives.py`` in
``xref_crosstable/`` is an extra file. The Opus enrich_lookup review
predicted no ``primitives.py``; the actual implementation required a
constant-relocation deviation (Option 1) but did not need a third
submodule. Both packages are still under three Python files plus
``__init__.py``.

== Correctness Concerns

Visible from static inspection: none.

* All 1087 tests pass; 1 xfailed marker is pre-existing.
* Import smoke test of every new module is green. Registry-style
  ``getattr(import_module(path), symbol)`` resolves correctly for the
  three repackaged registry bindings.
* No ``__init__.py`` was modified outside the two new packages.
* No registry, YAML, or test file was edited.
* No public symbol was renamed, removed, or had its signature
  changed.
* No ``_meta`` key shape was changed. The
  ``_meta.derived.sheets.<output>.enrich_lookup`` consumer surface in
  ``derived_column_policy``, ``fk_helpers/provenance``, and
  ``rendering/passes/_base`` continues to work unchanged.
* The ``primitives.py`` deviation (xref) and the constant-relocation
  deviation (enrich_lookup) are documented in their respective
  ``audit/*_review.adoc`` files; both preserve identical run-time
  behaviour.

Potential future concerns flagged in earlier reviews but **not**
addressed in this wave (intentionally - they were deferred):

* ``column_roles/resolver.py`` is still flagged as "interim".
* ``schema_maintenance`` still has the unresolved ``WriteIntent``
  enforcement question.
* The two ``Finding`` dataclasses still coexist.
* ``compact_multiaxis._legend_groups`` ↔ ``cell_codec._legend_tokens``
  duplication is still present.
* ``transformations/helpers.py`` still mixes FK-aware reordering with
  header utilities.

None of these is a regression; all were explicitly deferred per the
Opus review.
