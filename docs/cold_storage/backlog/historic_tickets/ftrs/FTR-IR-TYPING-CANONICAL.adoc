===== FTR-IR-TYPING-CANONICAL — Replace duck-typed interfaces with typed contracts

*Purpose:*::
Ensure all IR-related components use canonical, type-safe interfaces instead of duck-typed objects. This builds on the shim cleanup (FTR-IR-SHIM-CLEANUP) and enforces clear contracts across the renderer, pipeline, and adapters.

*Scope:*::
- Introduce formal `Protocol`/dataclass definitions for key IR constructs (`RenderPlan`, `RenderStep`, `RenderWorkbook`).
- Update renderer, router, and tests to require these types explicitly instead of inferring shape at runtime.
- Add static type validation (mypy/pyright) to CI and enforce at least one strict-mode run.
- Adjust integration and test fixtures to provide canonical plan objects.

*Acceptance:*::
- All renderer and adapter entry points accept/return declared IR types only.
- No remaining `isinstance`/`hasattr` duck-type checks.
- CI includes type-checking job; mypy passes with zero errors.
- Downstream libraries and demo repos compile cleanly under strict type checking.

*Extension (cleanup from FTR-IR-WRITEPATH-P1):*::
During the IR migration, `TableBlock` temporarily accepted a `top_left` alias to keep legacy tests compatible with older fixture data.
As part of this feature, remove that alias and enforce canonical positional attributes (`top`, `left`) only.
Deprecation warnings may be introduced in one intermediate release before complete removal.

*Acceptance for cleanup:*::
- `TableBlock` no longer defines or accepts the `top_left` argument.
- All tests and fixtures use canonical `top` / `left` initialization.
- No `DeprecationWarning` emitted in strict CI mode.
- Document the change in `CHANGELOG` under “Removed legacy constructor aliases”.
