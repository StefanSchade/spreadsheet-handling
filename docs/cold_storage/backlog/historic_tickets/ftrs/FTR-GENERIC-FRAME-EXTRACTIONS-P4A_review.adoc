== Implementation Quality

=== Positiv

* **Saubere Separation**: `frame_extract.py` (290 Zeilen) ist ein eigenständiges Modul ohne unerwartete Abhängigkeiten
* **Defensive Validierung**: MultiIndex-Frames, Duplicate Columns, leere Column-Names, Non-Scalar Constants — alles geprüft
* **Predicate-Vocabulary klar begrenzt**: `_WHERE_PREDICATES = {"equals", "in", "non_empty", "is_null", "not_null"}` — exakt die im FTR definierten
* **Immutabilität**: Source-Frame wird nie mutiert (`.copy()` an allen relevanten Stellen)
* **Deterministische Sortierung**: `kind="mergesort"` für stabile Reihenfolge bei Gleichheit
* **Lifecycle-Integration**: Korrekte `mark_source_if_unclassified` + `write_frame_lifecycle` Nutzung
* **Registry-Konformität**: `make_frames_target_step` wie bei allen vergleichbaren Steps
* **Meta-Registry aktualisiert**: `frame_lifecycle` Producer-Liste erweitert

=== Anmerkungen (kein Blocker)

* [A1] `result = result.where(pd.notnull(result), "")` (Zeile 57): NaN→"" Normalisierung.
  Das ist konsistent mit dem bestehenden Rendering-Pfad, aber es bedeutet,
  dass `is_null`-gefilterte Ergebnisse keine Null-Werte mehr enthalten.
  Der Test `test_extract_frame_supports_null_predicates` zeigt das korrekt:
  `{"deprecated_at": ""}` statt `None`. Das ist ein bewusster Design-Choice,
  kein Bug — Rendering erwartet Strings.

* [A2] `not_null` und `is_null` mit `False` sind äquivalent zu `is_null: true`
  bzw. `not_null: true`. Die doppelte Negation ist etwas verwirrend,
  funktioniert aber korrekt durch `~mask` Invertierung.

* [A3] Nachprüfung: Der `pipeline_step_registry.json`-Eintrag dokumentiert
  auch `rename`, `constants`, `sort_by`, `lifecycle` und `name`. Die
  ursprüngliche Review-Notiz zu fehlenden optionalen Parametern ist damit
  erledigt.
