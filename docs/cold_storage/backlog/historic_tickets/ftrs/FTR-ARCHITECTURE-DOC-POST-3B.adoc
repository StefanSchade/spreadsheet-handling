===== FTR-ARCHITECTURE-DOC-POST-3B — Document target architecture after 3b/3c stabilization

*Status:* Open
*Epic:* Phase 3b — Pipeline & Orchestration Consolidation

*Purpose:*::
Capture the post-3b target architecture in concise project documentation once the most important responsibility cuts are stable in code and tests.

*Context:*::
During migration it is easy to document intended architecture too early and thereby freeze assumptions instead of actual structure. This documentation should therefore follow stabilization work, not precede it.

*Scope:*::
- Document the responsibility split between `domain`, `pipeline`, `orchestrator`, `rendering`, and concrete backends.
- Document the role of meta/config in transformation, validation, and rendering.
- Document the validation-finding/severity model if adopted during 3b.
- Use one representative feature slice (such as FK-helper behavior) to illustrate the architectural cuts.

*Non-Goals:*::
- No broad user documentation rewrite.
- No speculative architecture narrative for features not yet stabilized.

*Acceptance:*::
- The architecture documentation reflects implemented responsibility boundaries rather than migration intent alone.
- At least one representative end-to-end feature slice is documented across layers.
- The documentation is short enough to support onboarding and future feature placement decisions.
