===== BUG-RELOCATION-STUB-XREF-SYNTAX-P5 -- Relocation stubs contain backtick paths instead of clickable xref links

*Status:* Done
*Priority:* P5 (low — documentation hygiene)
*Area:* documentation / cross-reference syntax / semantic model / functional model
*Origin:* FTR-FUNCTIONAL-SEMANTIC-MODEL-SPLIT-P5

==== Summary

Several relocation stubs and attribution notices created under
`FTR-FUNCTIONAL-SEMANTIC-MODEL-SPLIT-P5` use backtick-enclosed filesystem
paths (AsciiDoc monospace code) where `link:` cross-references were intended.
The paths render as literal inline code rather than clickable hyperlinks.

==== Affected files

Six files were identified:

*Navigation stubs (semantic model side) — "see" links should be clickable:*

* `docs/semantic_model/03_problem_statement.adoc` — two paths
* `docs/semantic_model/10_roundtrip_walkthroughs.adoc` — two paths
* `docs/semantic_model/05_external_inspirations.adoc` — one path

*Attribution notices (functional model side) — "formerly at" references:*

* `docs/functional_model/ch06_preserved_problem_space_context/01_problem_statement.adoc` — one path
* `docs/functional_model/ch06_preserved_problem_space_context/02_roundtrip_walkthroughs.adoc` — one path
* `docs/functional_model/ch06_preserved_problem_space_context/_chapter.adoc` — two paths

==== Root cause

The stubs were authored with backtick notation (`` `docs/path/to/file.adoc` ``),
which AsciiDoc renders as monospace code. The correct cross-reference form for
navigating to another `.adoc` file is `link:relative/path.adoc[display text]`
with a path relative to the current file's location.

==== Fix

Replace each backtick-wrapped path with a `link:` reference using a relative
path from the file's own directory. For directory references (no filename),
link to the chapter entry point `_chapter.adoc`.

==== Verification

Built HTML with `asciidoctor` for all six affected files and confirmed that
each formerly-literal path renders as a clickable hyperlink. No broken-link
errors were reported.
