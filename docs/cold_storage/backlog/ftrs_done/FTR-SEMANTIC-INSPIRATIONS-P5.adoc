===== FTR-SEMANTIC-INSPIRATIONS-P5 -- Land external inspirations as project interpretations

*Status:* New
*Priority:* P5
*Area:* Semantic model / documentation

==== Goal

Capture the external inspirations -- specification thinking, data
contracts, schema evolution, consumer-driven contracts, deprecation /
migration policies -- and record how each is interpreted inside this
project. The result must remain inspirational, not normative.

==== Scope

* Elaborate the External Inspirations chapter so each inspiration has a
  concise "relevant ideas" block and a "project interpretation" block.
* State explicitly that the project does not import any heavyweight
  methodology wholesale; only the named project interpretations are
  load-bearing.
* Cross-link to existing project docs (registries, FTR workflow,
  interface gates) where an inspiration is already partially realised.

==== Out of scope

* No introduction of a formal specification language.
* No new metadata fields, registry entries, or guards.
* No claim that binds the project to one external framework.

==== Seed sections elaborated

* External Inspirations

==== Acceptance (rough)

* Each inspiration carries an explicit "project interpretation"
  paragraph.
* The chapter opens with a NOTE that the section is inspirational, not
  normative, consistent with the seed.
* The document still renders cleanly via `make docs-html`.

==== Code and guards

* No production code. No tests. No guards. Documentation only.
