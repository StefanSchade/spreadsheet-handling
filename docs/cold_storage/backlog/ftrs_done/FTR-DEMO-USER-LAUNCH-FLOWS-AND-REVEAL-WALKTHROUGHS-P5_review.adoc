== Findings

[cols="1,2,4,1", options="header"]

| ID | Severity | Finding | Status

| H1
| High
| `demo/CLAUDE.md:22` still references `../spreadsheet-handling` after the
  repo rename to `../core`. The commit migrated lines 8–11 but missed this
  one. Agent guidance silently degrades to fallback rules on a fresh
  workspace.
| Fixed

| M1
| Medium
| `run_demo.sh` and `run_demo.ps1` install `pip install -e .[dev]`,
  pulling pytest, pyyaml, pandas into a user's venv. Core already declares
  pyyaml/pandas as runtime deps; pytest is test-only and not needed for
  the user-facing launcher path.
| Fixed — switched to `pip install -e .` (no extras).

| M2
| Medium
| Launchers compute `SCRIPT_DIR` but do not `cd` to it before invoking
  Python. Pipeline YAMLs use relative paths (`./data/...`, `./tmp/...`),
  so any invocation from a non-repo CWD silently misresolves the paths.
  Particularly hazardous for Windows users invoking the `.ps1` from a
  shell whose CWD is the user's home.
| Fixed — added `cd "$SCRIPT_DIR"` (sh) and `Set-Location $ScriptDir` (ps1).

| M3
| Medium
| Shell launcher forwards `"$@"` (so users can append extra flags). PS
  launcher only forwarded `$Pipeline`; extra args were silently dropped.
| Fixed — added `[Parameter(ValueFromRemainingArguments=$true)] [string[]] $Extra`
  to the PS launcher and forwarded with `@Extra`.

| M4
| Medium
| `.gitattributes` had no rule for `*.ps1`. New `run_demo.ps1` was
  committed under the implicit `text=auto` rule, leaving its line endings
  dependent on the cloner's platform / `core.autocrlf` setting.
| Fixed — added `*.ps1 text eol=crlf` to `.gitattributes`.

| L1
| Low
| `make clean` removes `$(BUILD_DIR)` (= `./target`) and `$(TMP_DIR)`,
  but `scripts/render_slides.sh` writes to `./build/slides/`. Rendered
  HTML survives `make clean`.
| Deferred — gitignored, not blocking the slice. Track as follow-up.

| L2
| Low
| `pip install --quiet` hides setup-time errors. End users get a
  downstream `ModuleNotFoundError` instead of the original pip failure.
| Fixed — dropped `--quiet` together with the M1 change.

| L3
| Low
| `scripts/render_slides.sh` hardcodes a fallback path under
  `~/.local/share/gem/ruby/3.3.0/bin/`. Breaks on Ruby 3.2 / 3.4 /
  Homebrew / rbenv.
| Deferred — affects developer render path only. Track as follow-up.

| L4
| Low

| Deferred — explicitly within scope of "scaffold". Track on Pages FTR.

| L5

== Confirmed clean (no action)

* No generated HTML tracked in demo source. `.gitignore` covers `build/`,
  `target/`, `tmp/`.
* Launchers invoke `python -m spreadsheet_handling.cli.apps.run` directly;
  no `make`, no WSL, no entry-point coupling.
* Source fixtures under `data/acme_plain/` are never overwritten — forward
  writes to `tmp/acme_01_plain.xlsx`, reverse writes to
  `tmp/acme_01_roundtrip/`.
* Repo-path migration (`../spreadsheet-handling` → `../core`) is consistent
  in `Makefile`, `scripts/dev-local.lib.sh`, `AGENT.md`. (Demo `CLAUDE.md`
  fixed under H1.)
* CI (`.github/workflows/ci.yml`) installs from PyPI and never references
  the sibling-core path; unaffected by the rename.
* Output layout (`build/slides/<name>.html`) is compatible with the future
  Pages mapping `/latest/demo/slides/<name>.html`. No Pages root is
  overwritten or assumed.
* Idempotent venv setup — launchers only install when `.venv` is missing or
  `--refresh-venv` / `-RefreshVenv` is passed.
