== Delivered change

The pipeline now takes a private structural snapshot of runtime `_meta` before
and after each successfully completed step when `sheets.pipeline` is enabled
for DEBUG. The resulting immutable `MetaDiff` reports sorted tuple paths in
`added`, `changed`, and `removed`, with an easy-to-test `unchanged` property and
a bounded `limited` marker.

The helper recursively snapshots ordinary mappings, lists, and tuples without
using unrestricted deep copy. Sequence members provide mutation detection, but
the diff reports a changed sequence only at its containing path. Safe scalar
sets are snapshotted atomically. DataFrames and arbitrary unsupported values
are identity-only opaque nodes. Unsupported mapping keys are retained only for
identity matching and never rendered or compared by value; a change collapses
to the nearest safe parent and carries the limitation marker.

`run_pipeline` remains a small linear loop. Its existing configuration DEBUG
message is unchanged in purpose and text. The new work is gated immediately
before the pre-step snapshot, surrounds the unchanged `out = step(out)` call,
and emits a summary only after successful return.

== Implementation decisions

=== Diagnostic snapshot representation

The private helper uses frozen, representation-disabled node dataclasses and a
frozen public-to-the-module `MetaSnapshot`. Mappings copy safe keys and child
nodes; lists and tuples copy child nodes recursively. This is an ephemeral
diagnostic tree, not a serializer, hash, persistent carrier, or metadata model.

=== Opaque-value behavior

DataFrames and unsupported values are stored as identity-only opaque nodes.
Replacement is observable; arbitrary internal in-place mutation is deliberately
best-effort and may remain invisible. No arbitrary equality or representation
is invoked.

=== Safe mapping-key behavior

Only non-empty printable built-in strings become path segments. Unsupported
keys are never converted, rendered, or value-compared. Identity matching allows
the helper to notice a safely snapshotted child change and collapse it to the
nearest supported parent; otherwise the result records only a limitation.

=== Subtree collapsing

An absent mapping key is added or removed once. Comparison never enters its
subtree. Final path tuples are deduplicated and sorted by dotted rendering.

=== Disabled DEBUG fast path

The exact guard is in the central loop after the existing pre-step
configuration `log.debug(...)` call and before `_snapshot_before_step(out)`.
When false, `before` is `None`, the normal step call is unchanged, and the
after-step diagnostic branch is skipped.

=== Failure isolation

The before-snapshot wrapper converts diagnostic exceptions to an internal
unavailable marker without touching the step. The after wrapper contains
snapshot, diff, formatting, and new summary logging, then attempts one fixed,
value-free limitation message if any stage fails. Failure of that fallback is
also swallowed. `Exception` is caught, not `BaseException`; step exceptions are
outside the wrapper and propagate unchanged.

=== Log formatting

The formatter emits one multiline DEBUG record per successful step. It renders
only safe dotted paths, empty category lists, `meta: unchanged`, or a bounded
unsupported-structure limitation. The format is intentionally human-readable
and unstable as a machine contract.

=== Developer-documentation placement

The small operational section lives in
`docs/developer_guide/ch03_getting_started/01_getting_started.adoc`, near setup
and verification guidance. It links to existing architectural material rather
than duplicating the pipeline or metadata models.

=== Release-note checkpoint

No release-note entry was added. Repository conventions reserve release notes
for public or release-relevant behavior. This change is an internal DEBUG
diagnostic with no new CLI argument, YAML option, public Python export, plugin
contract, metadata schema, stable log format, migration behavior, known-
limitation change, or release-positioning change.
