== Root Cause Note

The residual XLSX workbook-view readback failure was not caused by
`project_by_role` and not by the workbook-view contract itself.

The failure came from the XLSX hidden `_meta` carrier boundary.

`MetaPass` writes sheet-level export marker keys such as `author`,
`exported_at`, and `version` into the hidden `_meta` sheet as empty-default
placeholders. After the first blob-unwrapping fix, a corrupt or empty inner
`workbook_meta_blob` could still leave an outer carrier wrapper that contained
only those marker keys plus `sheets`.

`canonicalize_workbook_meta` then treated that outer wrapper as canonical
metadata. As a result, readback produced a mapping shaped like:

[source,text]
----
author
exported_at
sheets
version
----

but still lacked top-level `workbook_view`.

The fix strips carrier-layer marker keys on every canonicalization recursion
level when they carry the MetaPass-default empty signature (`None` or empty
string). Non-empty caller-assigned values remain preserved.

The bug was therefore a carrier-boundary canonicalization defect, not a
projection-feature defect.
