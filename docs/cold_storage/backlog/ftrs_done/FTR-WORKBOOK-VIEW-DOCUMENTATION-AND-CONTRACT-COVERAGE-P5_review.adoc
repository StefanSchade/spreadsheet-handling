== Semantic and Technical Model decision

No Semantic Model expansion was made. The Semantic Model's general lifecycle
and persistence material does not need a duplicate Workbook View concept.

No Technical Model cross-reference was added. The existing
`pipeline_semantics.adoc` Workbook View boundary section already explains
linear execution, explicit view metadata, and the prohibition on guessing by
name, shape, or position. The meta-classification chapter and glossary already
cover the metadata and terminology. The new User Guide section links directly
to the Functional Model's conceptual source, so no unresolved navigation gap
justifies another Technical Model edit.

== Domain-contract placement decision

No registry changed. Current machine-readable coverage is already aligned with
the Functional Model:

* `registries/pipeline_step_registry.json` represents
  `configure_workbook_view` as configuration-only Workbook View policy and
  `apply_workbook_view_sheet_mappings` as the relation-preserving inbound
  remap;
* `registries/meta_registry.yaml` represents `workbook_view` and the
  authoritative persisted `workbook_view.sheet_mappings` child;
* `registries/domain_contracts/canonical/implementation_units.json` contains
  `IMPL-STEP-CONFIGURE-WORKBOOK-VIEW` and
  `IMPL-STEP-APPLY-WORKBOOK-VIEW-SHEET-MAPPINGS`;
* `registries/domain_contracts/canonical/meta_reference_surfaces.json` contains
  the supported `MRS-WORKBOOK-VIEW-SHEET-MAP` surface.

Workbook View remains absent as a dedicated row in
`registries/domain_contracts/canonical/transformations.json`. That registry is
transformation-centric, while the Functional Model defines Workbook View as a
boundary/supporting capability over already prepared frames. Adding a row for
apparent symmetry would misclassify it and is not justified.

== Guard decision

No guard was added. Existing coverage includes:

* `tests/unit/domain/test_workbook_views.py` for declarations, mappings,
  validation failures, authoritative remapping, and order independence;
* `tests/unit/rendering/test_frame_selection.py` for selection, configured
  names and order, and rejection of missing or duplicate sheets;
* `tests/integration/pipeline/test_configured_workbook_view_reimport_roundtrip.py`
  for XLSX and ODS reimport;
* `tests/integration/io_backends/test_ods_xlsx_parity.py` and the XLSX metadata
  carrier tests for carrier mapping persistence and parity;
* architecture guards for registry classification, carrier-boundary intent,
  and the prohibition on pipeline routing through Workbook View metadata.

Implementation found no stable machine-checkable contract that is currently
unprotected and has a credible drift path. A guard for documentation presence
or prose wording would encode an incidental documentation shape rather than a
product invariant.
