== Step 3 — Execute migration (only after confirmation)

After confirmation, migrate the concepts according to the approved
routing table. Rules:

=== For human_info/ targets (prose docs)

- Language: English throughout. Translate any German content.
- Format: AsciiDoc, consistent with existing `human_info/` style.
- Structure per concept doc:
  * Title + one-line summary
  * "Why this matters" — motivation in 2-3 sentences
  * Core concept explained in prose (no bullet dumps)
  * Concrete example (ideally from the actual codebase)
  * "How it connects" — cross-references to related concepts and
    relevant code modules
  * "Further reading" — pointer to relevant ai_info section if any
- Quality bar: complete enough to be useful; if a concept needs
  substantial domain knowledge you don't have, create a well-structured
  stub with explicit TODO markers rather than inventing content.
- Do NOT copy-paste from the source if it is incomplete or in German —
  rewrite in English.

=== For ai_info/ targets (agent-facing)

- Language: English.
- Format: compact, bullet-oriented, imperative where possible.
- Structure:
  * 1-sentence definition
  * Key invariants / rules (bullet list, ≤ 8 items)
  * "What breaks this" — failure modes (≤ 4 items)
  * "Where in code" — module references (file paths, not prose)
  * Cross-references to related ai_info sections
- If extending an existing file (e.g. `architecture_primer.adoc`):
  add a clearly delimited new section, do not rewrite existing content.
- If creating a new `ai_info/concepts_<name>.adoc`: follow the
  `ai_info/_ai_info.adoc` index convention.

=== For stubs

Create the file with the full structure skeleton, fill in what you
can confidently, and mark uncertain/incomplete sections with:
`// TODO: <what is missing and why>`

=== After all files are written

1. Update `docs/human_info/_human_info.adoc` to include the new
   concepts/ section in its index.
2. Update `docs/ai_info/_ai_info.adoc` if new ai_info files were added.
3. Add a `_concepts.adoc` index to `docs/human_info/concepts/` listing
   all concept docs with one-line descriptions and the reading order
   from Step 2c.
4. Do NOT delete the source files in `docs/internal_guide/concepts/`
   yet — add a deprecation notice at the top of each source file:
   `// MIGRATED — see <target path>. Remove after next release.`
