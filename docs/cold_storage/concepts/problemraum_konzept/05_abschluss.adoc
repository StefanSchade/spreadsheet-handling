== Abschluss

Das Problemraumkonzept liefert keine neue allgemeingültige Metadatenregel.
Sein Zweck ist, bestehende und zukünftige Modellentscheidungen an konkreten
Prozessgrenzen zu prüfen.

Die leitende Frage lautet nicht:

[source,text]
----
Welche technische Metadatenklasse hat diese Information?
----

Sondern zuerst:

[source,text]
----
An welcher Stelle des Prozesses wird diese Information gebraucht?
Was passiert, wenn sie dort fehlt?
Kann sie rekonstruiert werden?
Muss sie zurückgeführt, lokal erhalten, geprüft oder verworfen werden?
----

Erst aus diesen Antworten sollte später eine technische oder semantische
Klassifikation abgeleitet werden.

Damit stabilisiert das Dokument die Arbeitsteilung:

* Das Functional Model erklärt den Problemraum, die Arbeitsformen und die
  Motivation.
* Das Semantic Model definiert daraus abgeleitete Verträge, Begriffe,
  Garantien und Interpretationsregeln.
* Das Technical Model beschreibt die konkrete technische Struktur, in der diese
  Verträge umgesetzt und geschützt werden.
