=== ADR-002 — Pandas as Core Data Representation

**Status:** Accepted

**Context**::
A theoretically 'pure' domain would avoid depending on Pandas and define its own entities and value objects.
In practice, nearly every layer of the system—domain logic, transformations, validations, and adapters—relies on Pandas DataFrames.

**Decision**::
- Pandas is accepted as part of the *domain model*.
- DataFrames are the shared representation between all layers (domain, transformations, rendering, I/O).
- No artificial abstraction is added as long as Pandas meets the requirements.

**Consequences**::
- High productivity and compatibility with the Python data ecosystem.
- Loss of theoretical domain purity (domain ≠ library-independent).
- Replacing Pandas in the future would be costly but this is accepted due to the ubiquity of the lib.
