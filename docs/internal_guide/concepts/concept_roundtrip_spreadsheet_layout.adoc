===== Concept: FTR-ROUNDTRIP-SAFE-P1 — Round-trip Strategy for Spreadsheet Layouts

Intent:::
Create a predictable, maintainable contract for authoring, editing, and re-importing spreadsheet workbooks. We target our own exported structures; arbitrary third-party workbooks are best-effort only.

Contract:::
- We guarantee round-trip for data + metadata. Style is normalized.
- We support two clear layout modes:
 * Scenario A (Flat tables): table starts at an anchor and expands until a sentinel empty row/column.
 * Scenario B (Named ranges): each table is a named range; multiple tables per sheet are supported and safe.

Why named ranges?:::
- They’re supported in Excel, LibreOffice/ODS, and Google Sheets.
- They provide stable, tool-readable boundaries that survive most user edits.

Embedded Metadata:::
- Writer emits a hidden _meta sheet or a named range_ `sh_meta` with JSON/YAML describing:
 * workbook version,
 * tables (id, sheet, start cell, named range if any),
 * typing/keys/validation descriptors (optional).
- Reader (parse_ir) prioritizes embedded meta when reconstructing IR.

Bootstrap without meta:::
- CLI or YAML pipeline can declare anchors and options (Scenario A) or named range mappings (Scenario B).
- A sidecar file *.xlsx.meta.json can carry the same info for batch imports.

Editing Rules (for users):::
- You may append rows and insert columns inside table bounds.
- You may modify cell values and keep formula columns.
- Avoid merges across header rows unless covered by the multi-header feature.
- For multiple tables on one sheet, use named ranges.

Testing Strategy:::
- Golden fixture: render → edit (scripted) → parse → compare IR (data + shapes).
- Negative tests: demonstrate why certain edits invalidate round-trip (documented as non-supported).

Roadmap Position:::
- Land after IR parity P1/P2 work; before IR becomes default and legacy removal.
- Related features: FTR-IR-WRITEPATH-P1, FTR-MULTIHEADER-P2, FTR-IR-TYPING-CANONICAL.
