= ACME Progressive Tutorial Structure

== Status

* Concept draft
* Intended as bridge between core capabilities, interaction patterns, and consumer-facing tutorials
* Intended to guide upcoming demo-repository integration work

== Purpose

This document proposes a progressive tutorial structure for the ACME demo universe.

The goal is not only to demonstrate isolated spreadsheet features, but to show how increasingly complex business-facing workbook projections can be derived from canonical structured repository models and roundtripped back safely.

The tutorials are intentionally cumulative.

Each tutorial introduces:

* one or more new interaction patterns
* additional projection complexity
* additional roundtrip requirements
* additional business-facing workbook structures

The tutorials should reuse one coherent ACME business universe instead of unrelated toy datasets.

== Architectural Framing

The tutorial structure should explicitly distinguish three layers:

[cols="1,3"]
|===
|Layer |Meaning

|Core Capabilities
|Low-level transformation primitives and workbook mechanics

|Interaction Patterns
|Reusable business-facing spreadsheet interaction structures built from capabilities

|Use Case Tutorials
|Concrete end-to-end business stories demonstrating patterns in practice
|===

The ACME tutorials belong primarily to the third layer, but should explicitly reference the interaction patterns they exercise.

== ACME Demo Universe

=== Business Theme

ACME Inc. develops and distributes software products globally.

The company operates several product lines:

* ACME Home
* ACME Personal
* ACME Business

Products are offered in different countries and regions.

Some offerings require additional metadata depending on:

* deployment model
* product category
* country or region
* customer or context variation

Business users maintain and review this information through spreadsheet-oriented workbook projections.

Canonical repository data remains normalized and explicit.

== Tutorial Progression

=== Tutorial 01 -- FK Helper Enrichment

==== Purpose

Demonstrate readable business projections derived from normalized FK relations.

==== Canonical Model

[source,text]
----
products
product_managers
branches
----

Relations:

* product n:1 product_manager
* product_manager n:1 branch

==== Workbook Representation

Sheet: `products`

[source,text]
----
product_id | product_name | manager_id | _manager_name | _branch_name
----

==== Interaction Patterns

* FK helper enrichment
* helper-column projection
* workbook readability enhancement

==== Core Capabilities

* apply_fks
* helper metadata
* flatten_headers
* validation

==== Goal

Show that normalized repository structures can still produce business-readable workbook views.

'''

=== Tutorial 02 -- Product Availability Relation Matrix

==== Purpose

Demonstrate editable cross-reference matrices for many-to-many relations.

==== Canonical Model

[source,text]
----
products
regions
countries
product_country_offerings
----

==== Workbook Representation

[source,text]
----
                Germany   Japan   Brazil
Meditation App  SaaS     SaaS
SecureDocs      SaaS     OnPrem  SaaS
----

Cell values represent relation markers or deployment variants.

==== Interaction Patterns

* relation matrix
* sparse relation projection
* multidimensional workbook view

==== Core Capabilities

* contract_xref
* expand_xref
* sparse reconstruction
* workbook view composition

==== Goal

Show how normalized many-to-many relations can be maintained ergonomically through matrix projections.

'''

=== Tutorial 03 -- Conditional Offering Detail Clusters

==== Purpose

Demonstrate conditionally active workbook sections driven by relation semantics.

==== Business Rule Examples

* Business products offered as SaaS in Germany or Japan require a regional service owner.
* Mobile applications require app-store metadata.
* On-prem offerings require installation metadata.

==== Canonical Model

[source,text]
----
product_country_offerings
offering_service_owners
mobile_app_metadata
onprem_installation_metadata
----

==== Workbook Representation

Main matrix:

[source,text]
----
                Germany   Japan   Brazil
Meditation App  SaaS     SaaS
SecureDocs      SaaS     OnPrem  SaaS
----

Additional detail areas become relevant only for subsets of rows.

Examples:

[source,text]
----
regional_service_owner | sla_tier
apple_app_id | age_rating
installation_partner | supported_os
----

==== Interaction Patterns

* conditional detail clusters
* typed relation payloads
* subset-specific workbook regions

==== Core Capabilities

* conditional workbook sections
* relation expansion
* validation rules
* metadata-driven workbook composition

==== Goal

Show how workbook projections can expose conditional business structures without abandoning normalized repository models.

'''

=== Tutorial 04 -- Contextual Projection Variants

==== Purpose

Demonstrate multiple workbook variants derived from one canonical relation using discriminator/context dimensions.

==== Business Story

Different customers or locales may use different business terminology for the same conceptual resources.

==== Canonical Model

[source,text]
----
resources
locales
resource_contexts
localized_resource_values
----

==== Workbook Representation

Workbook sheets:

[source,text]
----
Labels_EN
Labels_DE
Labels_FR
----

Columns:

[source,text]
----
resource_key | default | customer_a | customer_b
----

==== Interaction Patterns

* contextual projection variants
* locale/context matrix projection
* override management
* fallback semantics

==== Core Capabilities

* split_by_discriminator
* contract_xref
* expand_xref
* fallback policies
* deterministic recomposition

==== Goal

Show how multiple human-oriented workbook variants can coexist over one canonical tuple relation.

'''

=== Tutorial 05 -- Rich Relation Payload Matrices

==== Purpose

Demonstrate matrices where cells contain semantically meaningful payload values instead of simple relation markers.

==== Example Theme

Worldbuilding continuity tracking.

==== Canonical Model

[source,text]
----
characters
timeline_phases
character_timeline_states
----

==== Workbook Representation

[source,text]
----
Character | Childhood | War | Exile | Return
Arin     | naive     | traumatized | cynical | reconciled
----

==== Interaction Patterns

* relation payload matrices
* temporal projections
* semantic cell payloads

==== Core Capabilities

* payload-aware matrix projection
* roundtrip-safe cell mapping
* structured relation expansion

==== Goal

Show that workbook cells may themselves represent structured business payloads rather than simple relation existence.

'''

=== Tutorial 06 -- Planning And Timeline Projections

==== Purpose

Demonstrate projections where the same canonical planning model can produce multiple human-oriented planning views.

==== Example Themes

* release planning
* project planning
* timeline coordination

==== Canonical Model

[source,text]
----
work_items
dependencies
teams
milestones
iterations
----

==== Workbook Representation

* planning matrix
* release coordination sheet
* gantt-style projection

==== Interaction Patterns

* temporal projections
* planning projections
* multi-view projection families

==== Core Capabilities

* projection composition
* timeline rendering
* multi-axis workbook generation

==== Goal

Show that multiple planning-oriented workbook views can coexist over the same canonical dependency model.

== Rationale

The ACME tutorial progression is intended to demonstrate that spreadsheet-oriented workbook editing can remain compatible with normalized, testable, repository-driven canonical models.

The tutorials should progressively reveal increasingly complex projection semantics while preserving deterministic roundtrip behavior.
