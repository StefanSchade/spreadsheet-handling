=== Concept: Sparse Relation and Dense Axis Reconstruction

==== Purpose

Compact spreadsheet transforms often move between dense matrix shapes and sparse
relations. Empty cells may be intentionally irrelevant, or they may be part of
the matrix contract that must survive a roundtrip.

This concept defines the architectural rule for that boundary:

* no heuristic reconstruction
* deterministic behavior
* explicit configuration over inference
* granular modes instead of one implicit global roundtrip semantic

==== Principle

The system must not reconstruct missing matrix rows, columns, or combinations by
guessing from visible labels, table positions, similar names, grouped headers,
or surrounding workbook layout.

Reconstruction is allowed only when the complete axis contract is available
from one of these sources:

* explicit pipeline or metadata configuration
* persisted contract metadata written by an earlier deterministic step

If neither source exists, sparse relations remain sparse.

==== Cell-Codec Boundary

`CELL-CODEC` owns cell value parsing and serialization only.

`drop_empty: true` is a valid sparse default:

* empty compact cell values produce no decoded rows
* empty-only groups can disappear from the decoded relation
* this is strict-lossless only when empty cells are semantically irrelevant

For strict matrix-shape roundtrip without later reconstruction, use
`drop_empty: false` so empty cell values remain explicit rows.

==== Future Dense Axis Contract

A future reconstruction mode may make sparse relations contract back into dense
matrices by using explicitly configured axis sources.

Example sketch:

[source,yaml]
----
dense_axes:
  rows_from:
    frame: Products
    key: product_id
  columns_from:
    frame: Markets
    key: market_id
----

Semantics:

* the relation may be sparse
* the dense row/column space comes from the configured axis sources
* missing combinations receive the configured `fill_value`
* axis identity comes from keys, not display labels
* the mode must be explicitly configured or loaded from persisted contract meta

This is a future contract shape only. It is not implemented by the current
`XREF-CROSSTABLE`, `CELL-CODEC`, or `COMPACT-MULTIAXIS` slices.

==== Non-Goals

* No guessing from visible spreadsheet labels.
* No reconstruction from table position or workbook proximity.
* No name-similarity matching between relation columns and other frames.
* No implicit switch from sparse relation semantics to dense matrix semantics.
