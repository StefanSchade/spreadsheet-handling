= Completed Wave Notice 3r
:date: 2026-05-13

This note records what the REVIEW-001 Architecture Hygiene wave already
addressed so later review waves can avoid re-opening the same ground.

== Already addressed in the consolidated follow-up FTRs

* `FTR-REVIEW-001-META-REGISTRY-DERIVED-P3` — the transient `derived` meta
  channel is now registered with explicit scope, lifecycle, producer, and
  consumer contract entries.
* `FTR-REVIEW-001-META-BLOB-JSON-P3` — `workbook_meta_blob` now writes as JSON
  deterministically, while legacy literal reads remain accepted as a fallback.
* `FTR-REVIEW-001-FORMULAS-CORE-MOVE-P3` — formula-intent types moved out of
  the rendering layer into a neutral core module so domain code no longer
  imports rendering formula types.
* `FTR-REVIEW-001-PASSES-CLEANUP-P3` — rendering passes were split into real
  per-pass modules, the dead stub-era files were removed, and a single
  canonical `default_passes()` source now defines ordering.
* `FTR-REVIEW-001-QUICK-WINS-P3` — the remaining small review items stay
  tracked separately and should not be duplicated in future waves.
