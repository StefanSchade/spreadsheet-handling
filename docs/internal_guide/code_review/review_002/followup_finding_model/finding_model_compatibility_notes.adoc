= Finding-Model Compatibility Notes (FTR-FINDING-MODEL-CONSOLIDATION-P5)

Date: 2026-05-18
Purpose: enumerate every externally observable surface and prove the
consolidation preserved it (or document the one intentional, importer-verified
relocation).

== 1. Emitted finding frames — byte-identical

[cols="1,3", options="header"]
|===
| Producer | Why output is unchanged

| `resource_overrides`
| Shared `Finding.to_record` is the *same* body the old
  `ResourceOverrideFinding.to_record` had (`", ".join` columns; `row_index`
  None→`""`, tuple/scalar→`row_index_label`; `value` None→`""`,
  tuple→`" | ".join(map(str,…))`). Same 7-col `FINDING_COLUMNS`, passed
  explicitly. Empty case → same zero-row 7-col frame.

| `derived_column_policy`
| Same argument; old `DerivedColumnFinding.to_record` used `_row_index_label`,
  identical to the shared `row_index_label`. 7-col `FINDING_COLUMNS` preserved
  and still importable.

| `reference_validations`
| `ReferenceFinding.to_record` and the 9-col `FINDING_COLUMNS` are *unchanged*.
  `findings_to_frame` delegates to the shared serializer, which runs the same
  `pd.DataFrame([f.to_record() …], columns=FINDING_COLUMNS)` — identical rows,
  identical column order, identical schema-stable empty frame.

| `graph_validations`
| Not modified. Still imports `ReferenceFinding` + `findings_to_frame` from
  `reference_validations`; both still defined there; output unchanged.
|===

Column order, category strings, severity values, and `severity`/`mode`
branching are untouched in every path.

  them green.

== 4. The one intentional change: `resource_overrides.row_index_label`

* Grep over `src/` and `tests/` confirmed *zero* importers of
  `resource_overrides.row_index_label` (and of its `FINDING_COLUMNS` /
  `findings_to_frame` / `ResourceOverrideFinding`).
* `FINDING_COLUMNS`, `findings_to_frame`, `ResourceOverrideFinding` are still
  exposed from `resource_overrides` (re-export / alias) — surface fully
  preserved.
* `row_index_label` is *not* re-exported from `resource_overrides`; the
  canonical helper now lives at `domain.finding_frame.row_index_label`.
* This is a zero-importer surface relocation with no observable behavior
  change. Documented here and in the review note for the independent reviewer
  rather than masked behind a compatibility alias (the project discourages
  cosmetic compat shims; D-009).
