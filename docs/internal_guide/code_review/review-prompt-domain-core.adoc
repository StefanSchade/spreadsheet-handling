== Kontext (einmalig am Session-Anfang einfügen)

----
Projekt: spreadsheet-handling — Python-Framework zur Transformation von
strukturierten Daten (JSON/XML/YAML) in tabellenorientierte Spreadsheets
(XLS/ODS). Roundtrip-fähig, deterministisch, CLI-Tool.

Architektur: Hexagonal + IR-Layer (Intermediate Representation für
Tabellenlayout-Operationen) + Pipeline/Workflow-Engine.

src-Struktur:
  spreadsheet_handling/
    domain/       ← fachliche Logik (Entities, Value Objects)
    core/         ← shared utilities
    application/  ← Use Cases / Application Services
    pipeline/     ← Workflow-Engine
    rendering/    ← IR-Layer
    io_backends/  ← Adapter (Reader/Writer)
    cli/          ← CLI-Adapter
    orchestrator.py

Tests:
  architecture/   ← Architecture-Tests (dependency_guards, carrier_boundary,
                     formula_boundary, parse_contract, pipeline_semantics,
                     semantic_invariants, spreadsheet_contract)
  unit/domain/
  unit/core/
  integration/
----

== Schritt 3 — Domain & Core Code reviewen

Lies jetzt den Code:

----
src/spreadsheet_handling/domain/
src/spreadsheet_handling/core/
----

Reviewe gegen das Erwartungsmodell aus Schritt 2. Fokus:

1. *Hexagonale Grenzverletzungen* — Importiert `domain/` etwas aus
   `io_backends/`, `rendering/`, `pipeline/`, `cli/`?
   Gibt es direkte Framework-/Library-Abhängigkeiten wo keine sein sollten?

2. *Fehlplatzierte Konzepte* — Gibt es fachliche Logik in `core/`
   die eigentlich nach `domain/` gehört? Umgekehrt: Utilities in
   `domain/` die nach `core/` sollten?

3. *Port-Konsistenz* — Sind Interfaces/Ports sauber als abstrakte
   Klassen definiert? Kennt `domain/` seine eigenen Ports oder
   referenziert es bereits Implementierungen?

4. *Determinismus-Risiken* — Gibt es nicht-deterministische Konstrukte
   in der Domain-Logik (Dict-Iteration ohne Sortierung, datetime.now(),
   random, uuid ohne seed)?

5. *Fehlermodell* — Exceptions vs. Result-Typen: konsistent?
   Domain-spezifische Exceptions oder generische?

6. sind die Fiels so geschnitten, dass sie gut verstaendlich durch mensch und ki sind? ein klarer purpouse pro modul / vermeidung hoher komplexitaet oder grosser laenge?
