== Spreadsheet-Meta-Registry

Die Datei `meta_registry.yaml` ist das manuell gepflegte Register der aktuell
bekannten spreadsheet-relevanten Meta-Eintraege im Pfad
`meta_canonical -> meta_rendering -> RenderPlan`.

Zweck der Registry:

* Sie inventarisiert die heute vorhandenen Meta-Eintraege explizit, statt ihr Verhalten nur aus Code und Tests abzuleiten.
* Sie beschreibt pro Eintrag kurz Bedeutung, Scope, Herkunft, Persistenz, Roundtrip-Relevanz sowie Produzenten und Konsumenten.
* Sie markiert abgeleitete IR-/Carrier-Eintraege wie `options`, `__freeze` oder `workbook_meta_blob` bewusst als operative bzw. quarantänisierte Formen und nicht als zweite kanonische Quelle.

Lesart:

* `meta_canonical` beschreibt persistierbare Workbook-Semantik.
* `meta_rendering` beschreibt die abgeleitete operative Sicht im IR.
* `RenderPlan` bekommt keine freien Meta-Maps mehr, sondern konkrete Operationen; die Registry verweist deshalb bei renderwirksamen Eintraegen auf die jeweiligen Plan-Consumer statt selbst neue Plan-Felder zu definieren.

Abgrenzung:

* Die Registry ist in diesem FTR bewusst manuell und review-orientiert.
* Sie erzeugt noch keinen Code und ersetzt keinen spaeteren typisierten Meta-Contract.
* Struktur-Semantiken wie Tabellengeometrie oder `named_ranges` ohne stabile freie Meta-Quelle werden in der Registry nur dort aufgefuehrt, wo bereits echte Meta-Eintraege oder Carrier-Hinweise existieren.
