== Architecture Guard Inventory

:updated: 2026-04-24
:status: active-guardrail-map

This inventory maps the current architecture guardrail suite to the seams it
protects and makes the remaining follow-up gaps explicit.

This inventory is descriptive. It does not define new enforcement rules by
itself; enforcement belongs to the listed follow-up FTRs.

The primary inventory unit is the guard module under `tests/architecture/`.
Integration tests outside that tree may support the same seams, but they are
not counted as primary architecture guardrails unless the test itself is
primarily enforcing a boundary or invariant.

'''

=== Inventory Scope

Included as primary guardrails:

* `tests/architecture/dependency_guards/`
* `tests/architecture/spreadsheet_contract/`
* `tests/architecture/parse_contract/`
* `tests/architecture/semantic_invariants/`
* `tests/architecture/current_state/`

Supporting but not counted as primary guardrails:

* `tests/integration/spreadsheet_contract/test_ir_readpath.py`
* `tests/integration/roundtrip/test_parse_ir_roundtrip.py`

These integration tests are useful seam evidence, but they remain behavior- and
roundtrip-first rather than guardrail-first.

| Gap | Classification | Why it is classified that way

| needs-design
