=== FTR Conventions

==== Lifecycle

Every FTR starts with `*Status:* New` in the `ftrs/open/` folder and moves
to `ftrs/done/` when implementation and acceptance are complete.

Allowed status values:

[cols="1,3"]
|===
| New | Not yet started
| Open | Acknowledged, may be in progress
| Open (Umbrella) | Coordination FTR that delegates to sub-features (see below)
| Superseded | Replaced by another FTR (kept for traceability, not implemented)
| Done | All acceptance criteria met, code merged to `dev`
|===

==== Umbrella FTRs

An umbrella FTR groups related sub-features under a single coordination ticket.

*Rules:*

. The umbrella carries `*Status:* Open (Umbrella)` and its own
  *integration-level* acceptance criteria — things that only make sense when
  all sub-features work together.
. Each sub-feature is a normal FTR with its own scope, acceptance, and
  lifecycle.
. All FTRs (umbrella + subs) appear in `_ftrs_open.adoc` for full backlog
  transparency.  Sub-features are listed directly below the umbrella with a
  comment line.  The following is schematic example syntax, not a literal file
  set:
+
[source,asciidoc]
----
include::FTR-UMBRELLA-P4.adoc[]

// Sub-features of FTR-UMBRELLA-P4:
include::FTR-SUB-A-P4.adoc[]

'''

include::FTR-SUB-B-P4.adoc[]
----
. Sub-features move to `ftrs/done/` individually as they are completed.
. The umbrella moves to `ftrs/done/` only when *all* sub-features are done
  *and* the umbrella's integration acceptance is met.

*When to use an umbrella:*

- A topic is too large for a single FTR but the parts share a coherent theme.
- The sub-features have meaningful dependency ordering.
- There is genuine integration acceptance beyond the sum of the parts.

Do *not* create an umbrella just to group loosely related items — use the
existing Epic/Phase structure in `_ftrs_open.adoc` for that.

==== Superseding FTRs

When a new FTR replaces an older one:

. Add `*Status:* Superseded by <<FTR-NEW#,FTR-NEW>>` to the old FTR.
. Keep the old file in `ftrs/open/` for traceability.
. Add a note below the include in `_ftrs_open.adoc`:
  `_Superseded by FTR-NEW — kept for traceability._`
