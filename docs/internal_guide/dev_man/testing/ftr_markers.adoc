==== FTR Traceability Markers

Every test file should declare which Feature Ticket (FTR) it belongs to via a
module-level `pytestmark`:

[source,python]
----
import pytest

pytestmark = pytest.mark.ftr("FTR-EXAMPLE-NAME")
----

The `ftr` marker is registered in `pyproject.toml` so that pytest does not emit
unknown-marker warnings.

===== Convention

* Place the `pytestmark` assignment *after all imports*, before any test
  functions or classes.
* Use the exact FTR identifier from the backlog (e.g. `FTR-ROUNDTRIP-SAFE-P1`).
* If a test file covers multiple FTRs, use a list:
+
[source,python]
----
pytestmark = [
    pytest.mark.ftr("FTR-FOO"),
    pytest.mark.ftr("FTR-BAR"),
]
----

===== Usage

*Run all FTR-marked tests:*

----
pytest -m ftr
----

*Collect tests for a specific FTR (keyword filter):*

----
pytest -m ftr -k "FTR_MULTIHEADER_P2"
----

NOTE: Pytest marker expressions do not support argument-level filtering
(`-m "ftr('FTR-XXX')"`).  Use `-k` to narrow down by FTR name.

===== When to add the marker

* Every new test file created for an FTR gets its marker immediately.
* Existing test files were retroactively marked based on git-history analysis
  (which commit introduced the file → which FTR branch was active).
