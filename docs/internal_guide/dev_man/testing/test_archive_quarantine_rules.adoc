==== Test Archive and Quarantine Rules

Every test must have exactly one lifecycle state. The state is determined first
by location, then by marker when an explicit non-active slice needs one.

===== Active

Active tests live under:

* `tests/unit/`
* `tests/integration/`
* `tests/architecture/`

They are authoritative for current behavior and are part of the normal active
development suite. Use `make test` for the default active slice; it excludes
`legacy`, `prehex`, and `slow`.

===== Quarantined

Quarantined tests are isolated from the active suite. The current quarantine
carrier is:

* `tests/legacy_pre_hex/`

Quarantined tests are not run by default. Run the explicit pre-hex quarantine
slice with:

[source,sh]
----
make test-prehex
----

If the quarantine directory is empty, that target may report a deferred/no-tests
outcome. Any future quarantined test group must have a short README or
classification note explaining why it is quarantined and whether the expected
next decision is `promote`, `rewrite`, or `retire`.

Quarantine is temporary. Do not use it as a permanent hiding place for failing
or outdated tests.

===== Archived

Archived tests are no longer normative. Prefer documented deletion over keeping
dead tests in the repository. If historical source is retained, it must live in
a clearly labeled archive location or be documented as deleted/retired, and it
must not be collected by pytest or referenced by active documentation as current
behavior.

===== Transitions

Move tests between states deliberately:

* Quarantined -> Active: validate against the current architecture and move the
  test into the correct active layer.
* Quarantined -> Archived: retire the behavior and document the decision.
* Active -> Quarantined: only when behavior is under active redesign or
  investigation, and only with a short rationale.

Avoid ambiguous states such as ignored active tests, collected files under
support carriers, or undocumented parked test modules.
