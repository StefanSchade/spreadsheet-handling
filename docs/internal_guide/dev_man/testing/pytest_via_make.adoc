==== How to execute the Pytests via Make

The test suite is orchestrated through the project’s `Makefile` for convenience and consistency.
All targets automatically prefer the virtual-environment’s pytest (`$(VENV)/bin/pytest`) if present;
otherwise the system `pytest` will be used.

By default, *pre-hex legacy tests* and intentionally slow tests are excluded
from normal runs to keep the signal clean during active refactoring.
Quarantined pre-hex tests can be explicitly included when needed.

===== Default behaviour

* Default marker expression: `MARK ?= not legacy and not prehex and not slow`
* Default active roots: `tests/unit tests/integration tests/architecture`
* Pre-hex tests live in: `tests/legacy_pre_hex/` and are outside the default
  active roots.

To verify what will run, invoke `make help`; the Makefile automatically lists all test targets with short descriptions.

Example output:

----
Available targets:
test: Run the normal active development slice
test-all: Run all active tests, including slow tests
test-integ: Integration tests only
test-ir: IR-only tests (SH_XLSX_BACKEND=ir)
test-legacy: Legacy-only (pre-hex tests, normally excluded)
test-lastfailed: Re-run only last failed tests (verbose)
test-node: Run a single test node (make test-node NODE=file::test_name)
test-one: Run tests filtered by TESTPATTERN (make test-one TESTPATTERN="expr")
test-unit: Unit tests only (exclude integ)
test-verbose: Verbose run with inline logs
----

===== Typical workflows

*Run the normal active development slice*

----
make test
----

*Run with a custom marker expression (overrides MARK)*

----
make test MARK="not slow"
make test MARK="integ"
make test MARK=
----

*Re-run last failed tests*

----
make test-lastfailed
----

*Run a single test by pattern, file, or node*

----
make test-one TESTPATTERN="helpers and not slow"
make test-file FILE=tests/unit/pipeline/test_runner.py
make test-node NODE=tests/unit/pipeline/test_runner.py::test_happy_path
----

*Run only a specific slice*

----
make test-unit       # unit tests (excludes integ)
make test-integ      # integration tests
make test-ir         # IR-only (backend = ir)
make test-prehex     # quarantined pre-hex tests only
make test-legacy     # compatibility alias for test-prehex
make test-all        # absolutely everything
----

===== Notes

* The `MARK` variable controls which tests are selected (`-m` expression).
Default active paths keep `tests/legacy_pre_hex/` outside normal collection.
* The `help` target auto-generates documentation from each target’s `##` comment — no duplication required.
* All commands honour additional pytest flags via `PYTEST_OPTS`:
----
make test PYTEST_OPTS="--maxfail=1 -x"
----
* To enforce use of the virtual environment, the `deps-dev` target ensures
`$(VENV)/bin/pytest` exists before invoking any tests.

===== Purpose

This convention standardises test execution across developers and CI, prevents accidental inclusion of quarantined pre-hex tests,
and supports reproducible runs for active backend slices.
