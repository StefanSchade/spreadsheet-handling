=== Early Project Evolution

Before starting to track via the backlog presented in this section we first relied on GitHub' s ticketing system for a while and then worked without strict ties to tickets for a while. This seemed pragmatic at the time but it obscures the purpose of the early commits. This brief section recounts the origin of the project history. For deep dives, read commit messages linked above and the section on **Concepts**.

*2025-08–09 — Proof of Concept & Early Structure*

- First JSON↔Spreadsheet transforms;
- tests and robustness passes.

*2025-09-02…09 — Modularization & Multi-backend groundwork*

- Modularize conversion logic;
- non-XLS backends start;
- README and repo structuring.

*2025-09-19 — Pipeline primitives*
- Introduce `BoundStep` and `run_pipeline`
- YAML-defined pipelines
- backend/config tidy
- merge pipeline refactor to `dev`.

*2025-10-01 — Hexagonal redesign milestone*

- Router (get_loader/get_saver), config shim, consistent `read/write_multi`, tests green.
- `pipeline/config.py`, `pipeline/runner.py`, `pipeline/types.py` added.
- isolate tests under `tests/legacy` (later moved to `tests/legacy_pre_hex/`)

*2025-10-02 — Thin CLI adapters over runner to complete the hexagonal architecture.*

- `sheets-pack/unpack` shims; XLSX styling quick-wins; minimal integration via runner.
- prevent legacy tests from running by default

IMPORTANT: The legacy test suite (pre_hex predates the IR. It is due to be decomissioned in Phase 6)

*2025-10-10…14 — Excel validations & meta preservation*

- Column validations (DataValidation) and hidden `_meta` handling.

*2025-10-16…17 — IR introduction & backend reorg*

- IR dataclasses, composer, render passes; openpyxl renderer; move XLSX backend; `header_map` for precise targeting; IR tests.


*2025-10-25 — Backlog consolidation*

- Backlog + concepts aligned for phases 1–4.
