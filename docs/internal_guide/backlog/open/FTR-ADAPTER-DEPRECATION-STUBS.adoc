=== FTR-ADAPTER-DEPRECATION-STUBS — Quarantine legacy adapters behind clear stubs

** *Purpose:*  Remove dead/obsolete adapter code and replace with minimal stubs that fail fast with actionable messages, reducing maintenance and drift.

** *Scope:*  Affects legacy adapters that are no longer exercised in tests or docs. Only keep router registration + thin stubs.

** *Solution:*
- Inventory adapters and tag each as active|deprecated.
- Move deprecated implementations to `adapters/_legacy/` (or delete) and register **stubs** in the router that raise a typed exception with guidance (e.g., `NotImplementedAdapterError(adapter='xml', hint='use json/xlsx or enable new IR path')`).
- Keep adapter-facing **interfaces** stable to avoid ripple effects.
- Update docs to list supported adapters and show deprecation message text.

** *Acceptance:*
- All deprecated adapters resolve to stubs that error deterministically with a helpful hint.
- Test suite includes one smoke test per stubbed adapter asserting the error contract.
- CI disallows references to removed modules (import checks).
- Docs reflect the supported set; examples don’t rely on removed code.
