=== FTR-NESTED-LISTS — Extract and Recompose Nested Lists from Hierarchical JSON

** *Purpose:*  Enable roundtripping between hierarchical JSON structures (objects containing lists of objects) and tabular frames suitable for spreadsheet or CSV representation.

** *Scope:*  Applies to adapters handling hierarchical formats (JSON, XML). Lists are extracted into separate frames, preserving 1-to-many and nested relationships through synthetic foreign keys.

** *Solution:*
- Introduce a transformation step `normalize_nested_lists(frames, meta)` that:
  - Scans JSON-derived frames for columns containing lists of objects or primitives.
  - Emits new frames for each list field (e.g., `orders.items` → frame `orders__items`).
  - Propagates parent identifiers into child frames to preserve referential integrity.
- Provide the inverse `compose_nested_lists(frames, meta)` to re-embed child tables into their parent JSON objects during export.
- Handle recursive/nested lists by repeating the pattern with deterministic naming (`__` or configurable separator).
- Capture the mapping manifest in `meta` for traceability (parent→child frame map).

** *Acceptance:*
- JSON with nested lists can be unpacked into multiple tables and re-composed without information loss.
- Each emitted table includes a parent-ID column referencing the originating object.
- Roundtrip tests cover simple and multi-level nesting (1-to-many-to-many).
- Spreadsheet outputs show one sheet per extracted list with consistent naming and keys.
- Config allows custom naming templates and maximum recursion depth.
