=== FTR-EXCEL-ADDIN — In-sheet buttons to trigger processing

** *Purpose:*  Let analysts run pack/unpack or custom pipelines from Excel via ribbon/buttons, avoiding shell/Make usage.

** *Scope:*  Excel-only convenience layer that shells out to the installed runner (CLI or `.pex`). No new business logic.

** *Solution:*
- Provide a minimal add-in approach (choose one):
  - **VBA + shell**: Ribbon button calls a signed VBA macro that executes the local runner with a selected profile.
  - **xlwings** (Python-backed): Button triggers a small Python bridge that calls `orchestrate(...)`.
- Add a sample workbook template (`/examples/excel-runner-template.xlsx`) with the button wired.
- Document how to point the button to profiles/pipelines (per-file or per-folder).

** *Acceptance:*
- Clicking the button runs a demo pipeline and updates/exports artifacts as expected.
- Works on a stock Windows Excel environment without requiring dev tools.
- Template + short setup guide included; simple smoke test in CI (headless where possible).

** *Epic:*  Phase 7 — Future

** *Notes:*  Prefer the simplest path first (VBA + shell to `.pex`) to minimize dependencies; add xlwings later if needed.
