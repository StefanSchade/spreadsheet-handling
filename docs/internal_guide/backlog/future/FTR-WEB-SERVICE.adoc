=== FTR-WEB-SERVICE — FastAPI front-end and optional static UI

** *Purpose:*  Provide an HTTP API (and optional lightweight UI) as an alternative ingress to the CLI, so analysts or other systems can submit jobs without local setup.

** *Scope:*  New process alongside the CLI. Reuses the shared `orchestrate(...)` engine; no business logic in controllers.

** *Solution:*
- Implement a small FastAPI app exposing endpoints:
  - `POST /run`  — accepts profile/pipeline + I/O descriptors (paths/URIs).
  - `GET  /status/{job_id}` — job state/logs (if async mode is added).
  - `GET  /healthz` — readiness probe.
- Add a minimal static client (optional): single-page form that calls `/run`.
- Package service with uvicorn and a Dockerfile for easy deployment.

** *Acceptance:*
- `POST /run` triggers the same path as `sheets-run` and returns results/artifacts or a job id.
- Unit tests for request validation; integration test that compares CLI vs API outputs for a sample pipeline.
- Service can be run locally (`uvicorn app:app`) and via Docker.

** *Notes:*  Keep the API thin; all orchestration stays in the core. Consider auth later if exposed beyond local use.
