===== FTR-TEST-HARNESS — Dual-backend markers & CI targets

* `@pytest.mark.xlsx_ir` and `@pytest.mark.xlsx_legacy`; Make targets `test-ir`, `test-legacy`, default excludes IR-only.
* Acceptance: CI can run legacy-only, IR-only, and full suites; flakes eliminated.
* “Introduce @pytest.mark.xlsx_ir and @pytest.mark.xlsx_legacy; add make test-ir → -m xlsx_ir and make test-legacy → -m xlsx_legacy; document that default make test runs not legacy.” (Matches what’s already half-there.)

**current state in the code**::

**What’s in code now:**::
a solid, fast test runner setup with default marker filtering and convenience targets (test, test-verbose, test-one, test-integ, test-all). Legacy tests are excluded by default via MARK_EXPR=not legacy plus --ignore=tests/legacy when applicable.

**What’s not present:**::
I can’t find the explicit dual markers @pytest.mark.xlsx_ir / @pytest.mark.xlsx_legacy, nor dedicated make test-ir / make test-legacy targets that switch on those markers. The acceptance text for FTR-TEST-HARNESS mentions those specifically, but they aren’t in the code I see.

**Partially done:**::
(runner + legacy filtering are there) but not fully “Done” per the feature’s acceptance (missing the xlsx_ir/xlsx_legacy markers and matching make targets).


*Adjustment of Acceptance Criteria:*

_Purpose:_ Run fast, focused test slices without accidental interference; keep historical tests quarantined.

*Why this is better now*

* Reflects IR as the primary path (so regressions don’t get masked).
* Preserves a clean way to isolate IR or legacy-writer behavior when you need it.
* Keeps the pre-hex suite as documentation/requirements without breaking day-to-day runs.

*We don’t need to “fulfill by the letter” that default excludes IR-only bit anymore.*

*Acceptance:*::
* Default make test runs unit+integration including IR tests, while excluding pre-hex (tests/legacy_pre_hex) by default.
* `@pytest.mark.xlsx_ir` is applied to IR-specific tests; `make test-ir` runs only these.
* If applicable, `@pytest.mark.xlsx_legacy` marks adapter legacy-writer tests; `make test-legacy` runs only these.
* Pre-hex archive remains under `tests/legacy_pre_hex`, marked legacy, ignored by default; optional `make test-prehex` (guarded) exists.
* CI has jobs (or a matrix) for:
  ** unit-fast (default slice),
  ** ir (`test-ir`),
  ** legacy-writer (`test-legacy`, if that path still exists),
  ** (optional scheduled) prehex (`test-prehex`).
  ** Developer help is discoverable (`make help` lists all test targets with descriptions).

*Non-goals:* Forcing IR exclusion by default; running pre-hex tests on every PR.
