===== FTR-VALIDATION-PASS-INFRA - Validation Pass Infrastructure

* *Context* Current spreadsheet validations are adapter-centric. We want neutral constraints that adapters can render.
* *Goal* Introduce neutral validation metadata (`frames.meta['constraints']`) and an IR ValidationPass to apply them in spreadsheet outputs.
* *Rationale* Separates semantic rules from rendering specifics; enables both programmatic checks and Excel validations.
* *Acceptance*
- Domain step `add_validations(rules)` writes neutral constraints into meta.
- IR ValidationPass consumes constraints; XLSX renderer applies them where feasible.
- Tests show constraints propagate and appear as Excel validations (e.g., in-list).
