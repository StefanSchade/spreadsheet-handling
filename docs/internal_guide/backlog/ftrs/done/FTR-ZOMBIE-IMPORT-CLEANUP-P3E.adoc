===== FTR-ZOMBIE-IMPORT-CLEANUP-P3E -- Fix dead imports, cross-layer violations, and backward-compat parameter hacks

*Status:* Open +
*Epic:* Phase 3e -- Shim & Dead-Code Removal

*Purpose:*::
Several modules contain import statements that always fail (defended by try/except), imports that are immediately shadowed by local redefinitions, or imports routed through a backward-compat facade when direct imports are available. These patterns obscure actual dependencies, violate hexagonal layering, and make the codebase harder to reason about.

*Findings (verified):*

[cols="1,2,1"]
|===
| Location | Issue | Severity

| `io_backends/json_backend.py` lines 6-11
| Try/except imports `BackendOptions` from `pipeline.types` (doesn't exist there), falls back to `dict[str, Any]`, then line 21 imports `BackendOptions` from `.base` -- overwriting the fallback. The try/except is dead code defending against the wrong location.
| HIGH

| `domain/transformations/helpers.py` line 9
| `from ...pipeline.types import Step, Frames` -- cross-layer violation (domain -> pipeline). Lines 10-11 immediately redefine `Frames = Dict[str, pd.DataFrame]` and `Step = Callable[[Frames], Frames]`, completely shadowing the import. The import is useless AND violates hexagonal architecture.
| HIGH

| `orchestrator.py` line 9
| `from .pipeline.pipeline import run_pipeline, BoundStep, Frames` -- routes through the backward-compat facade when direct imports from `.pipeline.registry` and `.pipeline.types` are available. Not broken, but adds unnecessary indirection.
| MEDIUM

| `io_backends/xml_backend.py` `write_xml_dir()` (line 209)
| Accepts arguments in either order: `(path, frames)` or `(frames, path)` via runtime type-sniffing. A backward-compat hack that should be standardized to `(frames, path)` matching the `BackendBase.write_multi` convention.
| LOW
|===

*Scope:*::
. `json_backend.py`: Delete lines 6-11 (the try/except block). The `from .base import BackendBase, BackendOptions` on line 21 is the real import.
. `helpers.py`: Delete line 9 (`from ...pipeline.types import Step, Frames`). Keep the local type aliases on lines 10-11.
. `orchestrator.py`: Change line 9 to import from `.pipeline.types` and `.pipeline.registry` directly.
. `xml_backend.write_xml_dir()`: Standardize to `(frames, path, *, options)` signature. Check all callers (router.py, tests) and update.

*Non-Goals:*::
- No changes to the pipeline/pipeline.py facade itself (15+ importers; collapse is separate work).
- No typing improvements (deferred to FTR-TYPING-CONSISTENCY).

*Acceptance:*::
- No try/except import blocks remain that always fail.
- No cross-layer imports from domain -> pipeline.
- `write_xml_dir` has a single, documented parameter order.
- All tests pass.
