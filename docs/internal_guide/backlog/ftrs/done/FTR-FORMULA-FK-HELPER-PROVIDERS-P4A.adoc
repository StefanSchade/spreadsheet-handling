===== FTR-FORMULA-FK-HELPER-PROVIDERS-P4A -- Formula-driven FK helper cells

*Status:* Implemented
*Epic:* Phase 4 -- Feature Expansion and Domain Transformations
*Priority:* High

*Depends-on:*

* <<FTR-FORMULA-PROVIDERS#,FTR-FORMULA-PROVIDERS>>
* <<FTR-FK-HELPER-DOMAIN-EXTRACTION#,FTR-FK-HELPER-DOMAIN-EXTRACTION>>

'''

====== Purpose

Move FK helper cells from copied helper values toward backend-translated formula
providers once the `FormulaSpec` boundary is stable.

'''

====== Scope

This FTR should decide and implement the smallest lookup-formula slice that can
respect the existing layer split:

* domain/pipeline describe FK intent without XLSX/ODS syntax
* rendering carries structured formula intent and stable references
* concrete adapters translate that intent to backend syntax
* tests assert generic semantics separately from backend formula strings
* `FormulaSpec` is expected to grow beyond the current single-case alias,
  likely into a Union or Protocol, before lookup formulas are added

'''

====== Implementation 2026-04-29

Implemented the smallest FK-helper lookup-formula slice:

* `FormulaSpec` now covers `ListLiteralFormulaSpec | LookupFormulaSpec`.
* `LookupFormulaSpec` represents same-row FK helper lookup intent without backend syntax.
* `add_fk_helpers` keeps copied helper values by default and can opt into formula-backed helpers via `defaults.helper_value_mode: formula`.
  Canonical mode names are `values` and `formula`; singular/plural aliases are currently accepted for YAML ergonomics.
* `core.fk.apply_fk_helpers` remains leaf-only and receives only an optional value-provider hook.
* XLSX rendering translates `LookupFormulaSpec` to `XLOOKUP(...)` inside the concrete openpyxl adapter.
* ODS rendering translates `LookupFormulaSpec` to OpenFormula `XLOOKUP(...)` inside the concrete ODS adapter.
* Generic tests assert structured lookup intent; adapter tests assert emitted backend formula syntax and rendered formula cells.
* Formula lookup ranges are derived from the union of data block bounds per referenced sheet, not from the first data block only.

The slice intentionally does not parse lookup formulas back into `LookupFormulaSpec`.
Current spreadsheet read paths still parse cell formula results as cell values where backend support exposes them.

'''

====== Non-goals

This FTR should not weaken the adapter contract or let raw lookup strings leak
into domain, pipeline, or generic rendering modules.

'''

====== Acceptance criteria

. FK helper behavior is represented as structured intent above adapter modules.
. XLSX/ODS-specific lookup syntax is emitted only by adapter translation code.
. Generic tests assert FK/formula semantics without raw backend strings.
. Adapter tests cover the emitted backend syntax for supported lookup formulas.
