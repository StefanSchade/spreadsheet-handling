===== FTR-ENGINE-RETIREMENT-P3E -- Remove dead engine/ package and consolidate config ownership

*Status:* Open +
*Epic:* Phase 3e -- Shim & Dead-Code Removal

*Purpose:*::
The `engine/` package is a backward-compatibility shim that no longer has any consumers. The `Engine` class wraps `core/fk` pure functions but is imported by zero modules. `engine/config.py` defines `EngineConfig` and `SheetMeta` dataclasses that are never used -- `pipeline/config.py` attempts to import from it via a try/except but the import always fails (the names don't match) and the fallback definitions are what actually runs. This creates a zombie import pattern that obscures where config types are actually defined.

*Current State (verified):*::
- `engine/orchestrator.py`: `Engine` class, 80 lines. Wraps `core/fk.build_registry`, `build_id_label_maps`, etc. Zero importers outside `engine/__init__.py`.
- `engine/config.py`: `EngineConfig`, `SheetMeta` dataclasses. Zero external importers. The `pipeline/config.py` try/except targets names (`AppConfig`, `IOConfig`, ...) that don't exist in `engine/config.py`, so the import always fails.
- `engine/__init__.py`: `from .orchestrator import Engine` -- re-exports a class nobody uses.
- `pipeline/config.py`: Lines 4-14 attempt `from ..engine.config import AppConfig, IOConfig, ...` which always raises, then fall through to local dataclass definitions (lines 17-90). These local definitions are the *actual* canonical config types used by all of pipeline and runner.

*Scope:*::
. Delete `engine/config.py` (dead code, zero consumers).
. Delete `engine/orchestrator.py` (dead shim, zero consumers).
. Delete `engine/__init__.py`.
. In `pipeline/config.py`: remove the try/except guard (lines 4-14). The fallback dataclass definitions become the sole, canonical definitions. No import path changes for consumers.
. Verify no test references `engine.Engine` or `engine.config` (grep confirms none).

*Non-Goals:*::
- The top-level `orchestrator.py` (which owns `orchestrate()`) is NOT part of this FTR. That module is actively used by CLI and 4+ test files.
- Runner consolidation (orchestrator.py vs pipeline/runner.py) is covered by FTR-DOMAIN-CONSOLIDATION.

*Risk:*::
Minimal. Zero live importers of the deleted code. The only behavioral change is removing a silent import failure in `pipeline/config.py`.

*Acceptance:*::
- `engine/` package directory is gone.
- `pipeline/config.py` defines config types directly (no try/except from engine).
- All tests pass without change.
- `git log --all -- src/spreadsheet_handling/engine/` confirms the deletion commit.
