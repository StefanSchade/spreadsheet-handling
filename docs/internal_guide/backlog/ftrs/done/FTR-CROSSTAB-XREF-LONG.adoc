===== FTR-CROSSTAB-XREF-LONG - Crosstab → XRef (Long-Form) Transformation

*Status:* Superseded by <<FTR-XREF-CROSSTABLE#,FTR-XREF-CROSSTABLE>> (sub-feature of <<FTR-COMPACT-TRANSFORMS#,FTR-COMPACT-TRANSFORMS>> umbrella)

*Context*::
Business matrices (wide form) need normalization to row-based xref tables for filtering and joining. (See existing planning notes on crosstab-to-xref.)

*Goal*::
Provide `make_crosstab_to_xref_step` to convert matrix-style sheets into row-based references, preserving optional header conditions.

*Rationale*::
Normal form is easier to query, diff, validate, and compose with other steps.

*Acceptance*

- Configurable row keys, column keys, header rows, marker semantics.
- Optionally drop empty rows (`drop_false=true`).
- Roundtrip examples in tests.
