===== FTR-DEPRECATED-ADAPTER-REMOVAL-P3E -- Remove deprecated and never-implemented backend stubs

*Status:* Open +
*Epic:* Phase 3e -- Shim & Dead-Code Removal

*Purpose:*::
Two backend modules exist solely as deprecation/error stubs and should be removed now that the deprecation cycle has passed.

*Findings (verified):*

[cols="1,2"]
|===
| Module | State

| `io_backends/excel_xlsxwriter.py`
| Deprecated adapter. Emits `DeprecationWarning` at import time, re-exports `ExcelBackend` from `xlsx.xlsx_backend`, and defines `_raise()` which throws `DeprecatedAdapterError`. The module exists only so old `from excel_xlsxwriter import ...` paths don't crash immediately. The deprecation warning has been shipping; time to remove.

| `io_backends/ods_backend.py`
| Never-implemented stub. `ODSBackend` raises `DeprecatedAdapterError("ods", ...)` for both `read` and `write`. ODS support was never built. The module serves no purpose.
|===

*Scope:*::
. Delete `io_backends/excel_xlsxwriter.py`.
. Delete `io_backends/ods_backend.py`.
. Check `io_backends/__init__.py` and remove any re-exports of `ODSBackend` or `excel_xlsxwriter`.
. Check `router.py` -- confirm neither backend is registered as a loader/saver (currently they are not).
. Grep for any test imports of these modules and remove/update.

*Risk:*::
Minimal. External callers who still import from `excel_xlsxwriter` will get a clear `ModuleNotFoundError` instead of a `DeprecationWarning` + error on use. This is the expected end of a deprecation cycle.

*Acceptance:*::
- Both stub files are deleted.
- No references to `excel_xlsxwriter` or `ods_backend` remain in source (excluding git history and docs).
- All tests pass.
