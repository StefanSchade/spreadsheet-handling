==== Implementation Quality

*Strengths:*

* `_join_token` uses `object()` sentinel for empty/NaN keys — prevents false
  matches on null join keys without requiring upstream cleansing (tested in
  `test_join_frames_does_not_match_empty_join_keys`)
* Flexible key configuration: `key`, `keys`, `left_key`/`right_key`,
  `left_keys`/`right_keys` — exactly one group must be configured
* Right-unique guard is default-on for left/inner joins, configurable for semi
* Post-join `where` filter reuses the same predicate vocabulary as extraction
  (`equals`, `in`, `non_empty`, `is_null`, `not_null`)
* Collision policy is explicit: `fail` (default) or `suffix` — no silent column
  shadows
* Semi-join uses a numbered-row-id approach to preserve left-side ordering and
  deduplicate correctly after inner-join expansion
* Lifecycle integration: both left and right sources marked `canonical_source`,
  output is `readonly_projection` with `derived_from: [left, right]`
* Temporary join-key column names are collision-safe (`__join_frames_{output}_{n}`)

*Minor observations (non-blocking):*

* Same shared-utility candidates as in `tabular_views.py` (`_is_empty_cell`,
  `_string_list`, `_ensure_string`, `_duplicate_column_names`) — refactor
  candidate for future housekeeping
* `_apply_where` is a near-copy of the extraction predicate logic — could be
  factored out into a shared predicate module if a fourth consumer appears
* Right-frame `_apply_where` runs before join (pre-filter), while `where` runs
  after merge (post-filter) — semantics are correct and well-documented in tests

==== Design Observations

* The `how="semi"` path does not expose right-side columns in output — this is
  intentional and correct for existence-filter semantics
* `right_unique` defaults to `None` → automatically `True` for left/inner,
  implicitly off for semi — good default-safety without forcing configuration
* The implementation correctly avoids pandas merge `validate` for semi-joins
  since the right side is expected to be non-unique (it's a tuple relation)

==== Verdict

Implementation is well-bounded, covers all acceptance criteria including the
critical tuple-relation semi-join case, and handles edge cases defensively.
Accepted without modification.
