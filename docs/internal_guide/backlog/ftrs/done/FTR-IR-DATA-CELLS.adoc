===== FTR-IR-DATA-CELLS — IR RenderPlan: Data Cell Emission

* *Status* Done
* *Context* The IR write path previously relied on a pandas ExcelWriter pre-write step for cell data, with IR formatting overlaid afterwards via `_apply_plan_to_existing()`.
* *Goal* Extend `build_render_plan()` to emit `WriteDataBlock` ops so the IR path is fully self-contained.
* *Acceptance*
- `build_render_plan()` emits `WriteDataBlock` ops containing cell data for each TableBlock.
- `_render_from_plan()` in openpyxl_renderer writes data cells from the plan.
- xlsx_backend IR path no longer uses the pandas ExcelWriter pre-write step.
- `_apply_plan_to_existing()` is no longer called in the IR path (demoted to legacy-only).
- All existing tests remain green; 7 new tests verify data-cell emission and roundtrip.

*Delivery scope*

[cols="1,3"]
|===
| `WriteDataBlock` op | New op in `plan.py` — frozen dataclass with `sheet`, `r1`, `c1`, `data` (tuple of tuples)
| `TableBlock.data` | New field in `ir.py` — row-major 2D data, populated by `compose_workbook`
| `compose_workbook` | `layout_composer.py` — now stores `df.values.tolist()` in `tbl.data`
| `build_render_plan` | `flow.py` — emits `WriteDataBlock` per table when `tbl.data` is populated
| `_render_from_plan` | `openpyxl_renderer.py` — new handler writes cell data from `WriteDataBlock.data`
| `xlsx_backend.py` IR path | Simplified: compose → passes → build_plan → `render_workbook()` (no pandas pre-write)
| `test_ir_call_chain.py` | Updated: expects `render` instead of `apply_plan` in call chain
| `test_ir_data_cells.py` | 7 new tests: plan emission, multi-sheet, MultiIndex, empty DF, roundtrip, numeric, backend integration
|===
