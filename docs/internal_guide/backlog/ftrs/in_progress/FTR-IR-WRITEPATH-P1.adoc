===== FTR-IR-WRITEPATH-P1 — IR write pipeline with openpyxl renderer: “Parity Slice v1” (P1) and wire CI around it

*Current status:* +
basic write path wired; minimal list validation present; more features to follow.

*Goal:* +
IR produces the same workbook “shape” and key UX bits as legacy for a small set of fixtures; run that in CI and keep legacy as the default elsewhere.

*Scope:* +
Header visuals + basic UX: header fill + bold, AutoFilter, optional freeze first row — centralize via a tiny StylePass and render it in openpyxl. (This aligns with FTR-STYLE-THEMES.)

*Validations:* +

* reuse the existing column→area mapping to place a simple in-list validation in the right column (FTR-COLUMN-MAP is “Done”).
* Sheet naming & ordering: match legacy sheet names and order.
* Meta persistence (minimum): write and read a hidden _meta sheet just enough to prove round-trip (FTR-META-PERSISTENCE minimal path).
