===== FTR-COMPACT-MULTIAXIS — Multi-Axis Compact Representation

*Status:* New

*Purpose:*::
Provide a generic approach to represent multiple logical dimensions in compact tabular form and transform them into explicit representations.

*Context:*::
In some tabular models, multiple logical dimensions are combined into compact cell values or layouts to improve readability and maintainability for human users.
For example, a single cell might encode distribution options across several categories as a compact string (e.g. `"P-E"` meaning "Paket + Express"), while the column header identifies a branch — combining a cross-table axis with an encoded cell value.

*Scope:*::
- Support representation of multiple logical dimensions in compact form
- Enable transformation into explicit multi-attribute or relational structures
- Build on top of cross-table and cell codec transformations

*Non-Goals:*::
- No assumption about the meaning or number of dimensions
- No fixed encoding strategy

*Solution:*::
- Combine cross-table expansion with cell token decoding in a composed pipeline step
- Allow configuration of how tokens map to abstract categories or dimensions
- Support optional categorization of tokens via meta configuration
- Provide a `make_compact_multiaxis_step` that chains XREF-CROSSTABLE expansion and CELL-CODEC decoding in sequence

*Dependencies:*::
- <<FTR-XREF-CROSSTABLE#,FTR-XREF-CROSSTABLE>>
- <<FTR-CELL-CODEC#,FTR-CELL-CODEC>>

*Configuration (Meta):*::
- `axes`: list of axis definitions, each naming the source (column header axis, cell token axis)
- `codec`: reference to the cell-codec configuration for token decoding
- `crosstab`: reference to the cross-table configuration for matrix expansion
- optional token categories (grouping tokens into abstract dimensions)
- optional constraints on valid combinations

*Acceptance:*::
- Given a matrix where columns represent one dimension and cell values encode another, the step produces an explicit relational table with one column per dimension
- A fixture with at least 2 axes (e.g. branch × distribution option) round-trips: compact → explicit → compact yields the original
- Empty cells produce no rows in the explicit output (sparse handling)
- Configuration errors (missing codec, missing crosstab config) raise clear diagnostics
- The composed step is usable as a standard pipeline step via `build_steps_from_config`
