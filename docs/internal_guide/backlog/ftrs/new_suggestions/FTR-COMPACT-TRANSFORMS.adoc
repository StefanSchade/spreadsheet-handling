===== FTR-COMPACT-TRANSFORMS — Compact ↔ Explicit Tabular Transformations

*Status:* Open (Umbrella)

*Purpose:*::
Provide a generic capability to transform between compact, human-oriented tabular representations and explicit, relational representations.

*Context:*::
Many tabular data sets are maintained in a visually compact form where multiple aspects of information are encoded into a dense layout (e.g., matrix-like layouts or compact cell values).

The system must support transforming such representations into explicit, structured forms and back, without embedding domain-specific semantics.

This FTR acts as an umbrella coordinating three sub-features. It is considered done when all sub-features are done and integration acceptance is met.

*Scope:*::
- Transformation between matrix/cross-table layouts and explicit relational representations
- Transformation between compact cell encodings and explicit token or attribute representations
- Support for roundtrip-safe transformations (compact → explicit → compact)
- Meta-driven configuration of transformation rules

*Non-Goals:*::
- No domain-specific interpretation of values (e.g., business meaning of tokens)
- No hardcoded semantics for specific industries or use cases
- No assumption of a single compact encoding strategy

*Design Principle:*::
The system provides generic transformation primitives. The interpretation of values is fully defined by configuration (meta) or external application layers.

*Sub-Features:*::
- <<FTR-XREF-CROSSTABLE#,FTR-XREF-CROSSTABLE>> — Matrix ↔ Relation
- <<FTR-CELL-CODEC#,FTR-CELL-CODEC>> — Cell Value ↔ Token Set
- <<FTR-COMPACT-MULTIAXIS#,FTR-COMPACT-MULTIAXIS>> — Multi-Axis composition (depends on the above two)

*Acceptance (integration-level):*::
- End-to-end roundtrip: compact matrix with encoded cells → explicit relational tuples → compact matrix yields identical output
- All three sub-features pass their individual acceptance criteria
- A combined integration test exercises the full chain (crosstab expand → cell decode → re-encode → contract)
- All steps are available as pipeline steps via `build_steps_from_config`

*Supersedes:*::
This umbrella replaces the original monolithic FTR-COMPACT-TRANSFORMS and absorbs the scope of FTR-CROSSTAB-XREF-LONG.
