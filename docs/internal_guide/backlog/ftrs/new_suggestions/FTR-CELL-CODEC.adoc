===== FTR-CELL-CODEC — Cell Value ↔ Token Set Transformation

*Status:* New

*Purpose:*::
Provide a generic mechanism to transform compact cell values into structured token representations and back.

*Context:*::
Cell values in tabular data may encode multiple pieces of information in a compact string form (e.g., delimited tokens).

*Scope:*::
- Parse cell values into a set or list of tokens
- Serialize token sets back into compact string representations
- Validate tokens against a configured value set
- Support configurable delimiters and normalization rules

*Non-Goals:*::
- No domain-specific meaning assigned to tokens
- No hardcoded token semantics

*Solution:*::
- Split cell values using configured delimiters
- Normalize tokens (e.g., trimming, ordering)
- Validate against optional allowed token lists
- Serialize tokens into canonical string representation

*Configuration (Meta):*::
- delimiter (e.g., '-', ',', etc.)
- allowed_tokens (optional)
- canonical_order (optional)
- case normalization rules

*Acceptance:*::
- Parsing and serialization are deterministic
- Roundtrip (string → tokens → string) yields canonical representation
- Invalid tokens can be detected if configuration is provided
- Behavior is fully driven by configuration
