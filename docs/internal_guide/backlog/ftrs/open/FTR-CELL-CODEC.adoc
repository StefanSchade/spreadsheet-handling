===== FTR-CELL-CODEC -- Cell Value <-> Token/Code Transformation

*Status:* New

*Purpose:*::
Provide a generic mechanism to validate, parse, and serialize compact cell
values without assigning domain-specific business meaning to them.

*Context:*::
Some compact spreadsheets use a single code per cell, for example `S`, `E`,
`x`, or `E-R-K`. Others use delimited token strings where one cell contains
several separately meaningful tokens.
+
The first implementation should support the safe case where the whole cell
value is treated as one code. Token splitting is an explicit mode, not the
default assumption.

*Scope:*::

* Validate whole-cell codes against a configured value set.
* Optionally parse delimited cell values into token lists.
* Serialize token/code values back into a deterministic cell representation.
* Support configurable delimiters and normalization rules.
* Allow `allowed_tokens` / `allowed_codes` to reference a legend block.

*Non-Goals:*::

* No domain-specific meaning assigned to tokens or codes.
* No hardcoded token semantics.
* No cross-table or relation transformation.
* No inference of token structure from punctuation such as `-`.

*Solution:*::

* Default mode: treat the entire non-empty cell value as a single code.
* Split mode: split cell values using configured delimiters.
* Normalize only where explicitly configured.
* Validate against optional allowed code/token lists.
* Serialize into a canonical string representation.

*Configuration (Meta):*::

* `mode`: `whole_cell_code` or `split_tokens`
* `delimiter` for split-token mode, for example `-` or `,`
* `allowed_codes` or `allowed_tokens`, optionally sourced from a legend block
* `canonical_order` for split-token mode
* case normalization rules

*Acceptance:*::

* Whole-cell codes such as `E-R-K` can be validated without being split.
* Parsing and serialization are deterministic.
* Roundtrip `cell value -> structured representation -> cell value` yields the
  configured canonical representation.
* Invalid codes/tokens can be detected if configuration is provided.
* A codec can use a `FTR-LEGEND-BLOCKS` codebook as its allowed-value source.
* Behavior is fully driven by configuration.
