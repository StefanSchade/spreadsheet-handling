===== FTR-ODS-PARSER-ROBUSTNESS-P3K — Add edge-case and negative tests for the ODS parser

*Depends-on:* Phase 3j completion

*Context:* The Phase-3j ODS parser was tested against project-generated workbooks and the parity suite covers the sunny-day path well. However, there is currently no coverage for edge cases, malformed input, or workbooks authored outside this project (e.g., manually created in LibreOffice Calc).

*Problem statement:* Without negative and edge-case tests, the parser may fail in unexpected ways when encountering real-world ODS files that deviate from the narrow structure this project writes. Before the ODS adapter is exercised in the demo repository and eventually by end users, basic robustness should be proven.

*Goals:*

. Add tests for degenerate ODS structures: empty sheets, sheets with no header row, sheets with a single data row, sheets with only hidden tables.
. Add tests for unexpected or missing ODS elements: tables without rows, cells without value attributes, missing content-validations, missing database-ranges.
. Add at least one test with a simple hand-authored LibreOffice Calc workbook that was not written by this project.
. Verify the parser does not crash on unsupported structures and produces reasonable fallback behavior.

*Acceptance criteria:*

. The test suite includes at least 5 focused edge-case or negative tests for the ODS parser.
. The parser does not raise unhandled exceptions on any of the tested degenerate inputs.
. Behavior on degenerate input is documented (empty `WorkbookIR`, graceful skip, or clear error).
. At least one test uses a fixture that was created outside this project's write path.

*Non-goals:*

- No attempt to handle every possible ODS variation from arbitrary spreadsheet applications.
- No changes to the write path.
- No expansion of the parser's supported feature scope.

*Why now:* Before the ODS adapter is used in the demo repository and before Phase 4 features exercise it further, basic parser robustness should be established rather than discovered during integration.
