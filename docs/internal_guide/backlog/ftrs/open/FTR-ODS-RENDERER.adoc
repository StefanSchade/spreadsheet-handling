===== FTR-ODS-RENDERER — Parity adapter for OpenDocument Spreadsheet

*Context*::
To support LibreOffice Calc users, an ODS backend is planned. It reuses IR passes but uses a different renderer.

*Goal*::
Provide an `ODSBackend` stub (read/write signatures) and register in router; implement rendering later via `odfpy`.

*Rationale*::
Keeps architecture future-proof; IR makes multi-backend rendering feasible.

*Solution Design*

- Implement ODS renderer using the same IR; gracefully degrade unsupported features.
- Acceptance: core layout renders; fallbacks documented; tests assert no crashes.

*Acceptance*::

- Stub present and discoverable by router.
- Clear NotImplemented errors; tests ensure graceful selection/fallback.
- Follow-up item for ODS Style/Validation parity.
