===== FTR-LOOKUP-HELPER-FORMULA-MODE-P4A -- Let lookup helper enrichment render formulas instead of copied values

*Status:* Open
*Priority:* P4A
*Area:* Domain transforms / formulas

==== Purpose

Add a formula mode for generic lookup helper enrichment so helper columns can
remain visibly linked to their lookup source in the generated workbook.

The framework already has backend-neutral lookup formula intent for FK helper
flows. A prototypical consumer matrix uses `add_lookup_helpers`, which
currently merges helper values into the output frame. For workbook demos,
formulas can make the relationship more transparent and reduce confusion when
users change the key.

==== Consumer Trigger

A prototypical consumer matrix projects helper fields from an entity table by
primary key. For a business-facing workbook, it may be clearer if helper cells
are generated as lookup formulas against the entity sheet rather than copied
static values.

==== Goals

* Add a configured helper value mode to `add_lookup_helpers`, for example
  `values` versus `formula`.
* Reuse the existing backend-neutral `LookupFormulaSpec` where possible.
* Ensure formulas reference configured sheet names, key columns, and helper
  columns after view composition.
* Store logical frame/sheet references in formula specs and resolve them to
  physical sheet names in the renderer after workbook view composition.
* Keep value mode as the default for non-spreadsheet and pure data workflows.
* Implement XLSX and ODS formula rendering/parsing parity at the current
  formula contract level.

==== Non-goals

* Do not introduce raw backend formula strings into generic domain layers.
* Do not implement arbitrary spreadsheet formulas.
* Do not require formulas when the consumer wants stable copied values.

==== Acceptance

* `add_lookup_helpers` can emit copied values or backend-neutral lookup formula
  specs by configuration.
* XLSX renders the lookup helper formulas correctly.
* ODS renders equivalent lookup helper formulas or records a concrete adapter
  gap with tests.
* Tests cover sheet/view renaming so formulas resolve against final rendered
  sheet names, not stale logical names.
* Existing value-mode behavior remains backward compatible.
* Consumer workbooks can choose formula-backed helper columns when that better
  explains the relation to business users.
