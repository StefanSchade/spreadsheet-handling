===== FTR-GRAPH-TOPOLOGY-VALIDATIONS-P5 -- Add optional topology checks for configured graphs

*Status:* Open
*Priority:* P5
*Area:* Domain validations / graph checks

==== Purpose

Extend basic graph reference validation with optional topology checks once a
real consumer workflow proves the need.

The pre-beta graph FTR should stay small: endpoints exist and edges are unique.
Topology checks such as cycle detection and reachability are useful, but they
are also more expensive to specify, test, and explain. They should be added
only when a maintained model needs them.

==== Candidate Checks

* `no_self_edge`
* `acyclic`
* `reachable_from`
* `all_targets_reachable`
* `no_orphan_nodes`

==== Non-goals

* Do not add a graph query language.
* Do not infer graph semantics from column names.
* Do not make topology checks part of the first basic graph validation slice.

==== Acceptance

* At least one topology check is implemented against an explicit graph
  configuration.
* Findings include graph, node/edge, row/key, and check name.
* Tests cover a valid graph and at least one failing topology case.
* The implementation remains optional and does not slow ordinary reference
  validations.
