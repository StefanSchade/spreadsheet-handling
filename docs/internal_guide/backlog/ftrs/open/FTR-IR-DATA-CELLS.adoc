===== FTR-IR-DATA-CELLS - IR RenderPlan: Data Cell Emission

* *Context* The IR write path currently only emits metadata/formatting ops (SetHeader, ApplyHeaderStyle, SetAutoFilter, SetFreeze, AddValidation, WriteMeta). Actual cell data is written by a pandas ExcelWriter fallback in xlsx_backend.py, then IR formatting is overlaid via `_apply_plan_to_existing()`.
* *Goal* Extend `build_render_plan()` to also emit data cell ops (e.g. `WriteDataBlock` or `WriteCell`) so the IR path is fully self-contained and the pandas pre-write workaround can be removed.
* *Rationale* The hybrid pandas+IR approach works but couples the IR renderer to openpyxl's load_workbook round-trip. A fully IR-driven write path keeps concerns separated, enables alternative backends (ODS, CSV-zip), and eliminates the double-open overhead.
* *Acceptance*
- `build_render_plan()` emits data cell content (not just headers) for each TableBlock.
- `_render_from_plan()` in openpyxl_renderer writes data cells from the plan.
- xlsx_backend IR path no longer needs the pandas ExcelWriter pre-write step.
- `_apply_plan_to_existing()` helper removed or demoted to optional decorator.
- All existing tests remain green; parity with legacy path confirmed.
