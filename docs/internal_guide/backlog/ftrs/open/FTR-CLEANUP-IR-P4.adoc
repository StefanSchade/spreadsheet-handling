===== FTR-CLEANUP-IR-P4 — Remove Legacy Renderer and Parity Safeguard

*Context*::
With parity proven in CI, dual-path support and the temporary safeguard must be removed to minimize complexity and maintenance cost.

*Goal*::
Make the IR renderer the **sole** XLSX path; remove legacy code, flags, Make targets, and the parity CI job. Convert parity fixtures to IR-only golden tests.

*Rationale*::
Eliminates conditional branches, reduces surface for regressions, and simplifies releases and future feature work.

*Acceptance*::
* Adapters and entrypoints use the IR renderer **unconditionally** (no env flags, no runtime switches).
* Legacy OpenPyXL write path and all toggles (e.g., `SH_XLSX_BACKEND`, `--legacy-xlsx`) are **deleted**, including Make targets and dead code.
* The **parity-ir** job is removed. Parity fixtures are retained as **golden snapshot tests (IR-only)** using normalized comparisons (structure/styles/validations/meta).
* `docs/legacy_gap.md` is closed: every item either implemented or explicitly deprecated with rationale.
* Release notes state: **“Legacy OpenPyXL path removed; IR renderer is now the sole backend.”**
* CI is green across the supported matrix with IR-only.

*NOTE:* Forward-only strategy (beta). No runtime fallback remains after P4. A temporary rollback branch/tag may exist outside `main` purely for emergency hotfixes, but it is **not** part of the supported code path.

IMPORTANT: After this is done, no remainder (also no safeguard) should remain from the legacy backend
