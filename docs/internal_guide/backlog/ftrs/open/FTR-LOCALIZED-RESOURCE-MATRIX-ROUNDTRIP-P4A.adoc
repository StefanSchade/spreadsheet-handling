===== FTR-LOCALIZED-RESOURCE-MATRIX-ROUNDTRIP-P4A -- Roundtrip localized resources through locale/context matrices

*Status:* Open
*Priority:* P4A
*Area:* Domain transformations / localized resources

==== Purpose

Support the generic localized-resource workbook pattern:

`canonical resource tuples -> locale/context workbook matrices -> edited workbook -> canonical resource tuples`

This covers labels, enum texts, validation messages, error messages, and similar
resource values whose identity is a resource key plus locale plus optional
business context.

==== Canonical Shape

[source,text]
----
localized_resource_values
-------------------------
resource_key
locale
context_id
text
----

The first slice may assume that resource keys, locales, and contexts are already
available as frames or configured axes.

==== Goals

* Project localized resource tuples into one or more locale-specific workbook
  matrices.
* Use context columns such as `default`, product ids, module ids, or screen ids
  as the crosstable axis.
* Expand edited matrices back into canonical tuples.
* Preserve identity from configured keys and axes, not visible labels or sheet
  name guessing.
* Support helper/context columns for descriptions and resource types.
* Integrate with standard key/reference/uniqueness validations.

==== Non-goals

* Do not judge text quality or translation correctness.
* Do not implement a full i18n framework.
* Do not infer fallback behavior from empty cells; use the fallback/override
  semantics FTR for that.
* Do not write target `.properties` files in this FTR.

==== Acceptance

* A localized resource tuple frame can be rendered as locale/context matrices.
* Edited matrices can be recomposed into canonical tuples with stable
  `(resource_key, locale, context_id)` identity.
* Tests cover at least two locales and two non-default contexts.
* Helper columns are ignored or checked according to explicit metadata, not
  silently merged as text payload.
* XLSX and ODS dispositions are explicit.
