===== FTR-VALIDATION-PARITY-SEMANTICS-P3K — Align validation parity tests with canonical constraint meaning

*Depends-on:* Phase 3j completion

*Context:* The Phase-3j parity tests compare `DataValidationSpec.formula` as an exact string between XLSX and ODS. Both adapters currently normalize to the same CSV-quoted form, so the comparison works today. However, the spreadsheet meta semantics documentation explicitly treats `formula` as an adapter-facing, quarantined expression form, with the canonical meaning living in `meta_canonical["constraints"]`.

*Problem statement:* If a third adapter is added or if either adapter changes its formula normalization, the current exact-string parity test will break even though the actual constraint meaning may be unchanged. The parity suite should defend portable constraint semantics, not an implementation-internal formula encoding.

*Goals:*

. Align the parity comparison for validations with canonical constraint meaning rather than adapter-internal formula strings.
. Keep the current parity claim strength while making it less format-encoding-dependent.
. Document the boundary between what parity tests defend (canonical constraint meaning) and what they tolerate (adapter-specific formula encoding).

*Acceptance criteria:*

. The parity suite compares validation semantics through their canonical constraint meaning (reconstructed list values) rather than raw `formula` string equality.
. The comparison still catches real semantic differences (different list values, different constraint types, missing validations).
. A brief code comment or test docstring documents why the comparison is semantic rather than string-based.
. Existing parity assertions remain at least as strong as before.

*Non-goals:*

- No changes to the `DataValidationSpec` model or the `RenderPlan` contract.
- No new validation types.
- No cross-adapter validation DSL.

*Why now:* This is a targeted test-quality improvement that prevents a known fragility point from becoming a real maintenance problem as the adapter set grows.
