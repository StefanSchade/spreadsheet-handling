===== FTR-NESTED-LISTS -- Extract and recompose nested 1:n child tables from hierarchical JSON

*Purpose:*::
Enable roundtripping between hierarchical JSON structures that contain lists
and tabular frames suitable for spreadsheet or CSV representation.

*Scope:*::
Applies to adapters handling hierarchical formats (JSON, XML). Lists are
extracted into separate frames, preserving 1-to-many and nested relationships
through synthetic foreign keys.

This FTR is intentionally about the hard case:

* nested singleton objects are not the real problem and can often stay in the
  flattening / multi-header space,
* nested lists are the point where the hierarchy must be broken into multiple
  related tables.

*Solution:*::

* Introduce a transformation step `normalize_nested_lists(frames, meta)` that:
  - scans JSON-derived frames for columns containing lists of objects or
    primitives,
  - emits new frames for each list field (for example `orders.items` ->
    `orders__items`),
  - propagates parent identifiers into child frames to preserve referential
    integrity.
* Provide the inverse `compose_nested_lists(frames, meta)` to re-embed child
  tables into their parent JSON objects during export.
* Handle recursive/nested lists by repeating the pattern with deterministic
  naming (`__` or configurable separator).
* Capture the mapping manifest in meta for traceability.

*Acceptance:*::

* JSON with nested lists can be unpacked into multiple related tables and
  re-composed without information loss.
* Each emitted table includes a parent-ID column referencing the originating
  object.
* Roundtrip tests cover simple and multi-level nesting (1-to-many-to-many).
* The first implementation documents and enforces a bounded placement strategy
  rather than leaving sheet/table placement accidental.
* Config allows custom naming templates and maximum recursion depth.

*Notes:*::
For detailed algorithms and examples (key propagation, naming templates,
recursive depth control), see the concept work on normalizing nested
structures.

See <<FTR-JSON-BUNDLER#,FTR-JSON-BUNDLER>> for the spreadsheet/table layout
strategy.
