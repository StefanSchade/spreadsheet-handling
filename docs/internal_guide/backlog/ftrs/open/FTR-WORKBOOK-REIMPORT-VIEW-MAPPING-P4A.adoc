===== FTR-WORKBOOK-REIMPORT-VIEW-MAPPING-P4A -- Map edited sheets back to logical frames

*Status:* Open
*Priority:* P4A
*Area:* Pipeline / readback semantics

==== Purpose

Add the first reverse-view slice for workbook readback: map visible sheet names
and workbook views back to logical frame names without guessing from labels.

Rendering already uses `_meta.workbook_view` and `_meta.frame_lifecycle` to
decide which frames become visible. Reimport needs the inverse lookup: which
visible sheet corresponds to which logical view frame, and which canonical
source frame that view may patch or recompose.

==== Prerequisites

* `FTR-DECLARATIVE-WORKBOOK-VIEWS-P4A`
* existing `select_render_frames()` / `workbook_view` policy
* existing `frame_lifecycle` metadata

==== Scope

* Persist or derive a deterministic sheet-to-frame mapping during workbook
  composition.
* Read that mapping during reimport.
* Fail clearly when a required visible sheet is missing or duplicated.
* Keep physical sheet names separate from logical frame names.
* Support renamed sheets through explicit view metadata.

==== Non-goals

* Do not expand matrices back to relations in this slice.
* Do not drop helper columns in this slice.
* Do not infer canonical frame names from arbitrary sheet names.
* Do not implement conflict resolution.

==== Acceptance

* A rendered workbook carries enough metadata to map each configured visible
  sheet back to a logical view frame.
* A reimport pipeline can select workbook sheets by configured view identity,
  not by hard-coded physical sheet names.
* Missing, duplicated, or unknown required sheets produce clear diagnostics.
* Tests cover at least one renamed sheet and one omitted intermediate frame.
* XLSX and ODS readback dispositions are explicit.
