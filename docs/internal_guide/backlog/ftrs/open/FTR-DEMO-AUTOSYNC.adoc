=== FTR-DEMO-AUTOSYNC — Automated dependency update and demo verification

*Purpose:*::
After the demo and library have stabilized, automate version tracking so the demo repo stays current with released library versions without manual intervention.

*Scope:*::
Applies to the public demo repository and optionally to the main project CI configuration.

*Solution:*

- Configure a Dependabot or Renovate workflow that monitors PyPI for new releases of `spreadsheet-handling`.
- Automatically open a pull request in the demo repo updating the version pin in `requirements.txt` or `pyproject.toml`.
- Trigger the demo’s CI pipeline to execute its smoke tests with the updated dependency.
- Optionally report back status to the main repo (e.g., via badge or webhook).
- Define simple labels and reviewers for quick acceptance or rollback.

*Acceptance:*

- New core releases automatically trigger a dependency-update PR in the demo repo within 24 h.
- Demo CI runs successfully against the new version before merge.
- Manual merges remain possible when larger refactors are needed, but routine updates require no manual work.
- Documentation briefly explains the automated update flow and how to pause or override it.
