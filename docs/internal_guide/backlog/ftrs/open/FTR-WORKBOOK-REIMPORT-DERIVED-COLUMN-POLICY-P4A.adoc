===== FTR-WORKBOOK-REIMPORT-DERIVED-COLUMN-POLICY-P4A -- Drop or check derived columns on reimport

*Status:* Open
*Priority:* P4A
*Area:* Pipeline / readback semantics

==== Purpose

Make reimport behavior explicit for helper, derived, and read-only columns in
business-facing workbook views.

Generated workbooks often show context columns next to editable data. During
reimport those columns should not silently patch canonical data unless they are
explicitly configured as editable. The first robust policy is to drop known
helper columns from payload recomposition and optionally check that their
edited values still match canonical lookup sources.

==== Prerequisites

* `FTR-WORKBOOK-REIMPORT-VIEW-MAPPING-P4A`
* `FTR-HELPER-COLUMN-STYLE-METADATA-P4A`
* `FTR-EDITABLE-COLUMNS-AND-PROTECTION-P4A`

==== Policy Modes

* `drop`: ignore derived/helper columns during payload extraction.
* `warn_on_mismatch`: compare derived values to canonical sources and write
  findings when they differ.
* `fail_on_mismatch`: compare derived values to canonical sources and fail the
  pipeline when they differ.

No edited helper value wins silently in this slice.

==== Scope

* Use explicit helper/editability metadata to distinguish payload columns from
  derived display columns.
* Drop helper columns from canonical payload frames.
* Optionally check helper values against their source lookup frame.
* Produce validation-style findings for helper/read-only mismatches.
* Keep protected cells as an editing hint, not as a trust boundary.

==== Non-goals

* Do not implement automatic conflict resolution.
* Do not patch canonical lookup tables from helper columns.
* Do not require spreadsheet protection to be present before reimport.
* Do not detect multi-user Git merge conflicts.

==== Acceptance

* A configured reimport path can produce a payload frame without helper columns.
* Mismatched helper/read-only values are ignored, warned, or failed according
  to explicit policy.
* Findings use the same report shape as standard validations where practical.
* Tests cover unchanged helper columns, edited helper columns in warn/fail
  modes, and payload extraction from a mixed editable/helper sheet.
* XLSX and ODS readback dispositions are explicit.
