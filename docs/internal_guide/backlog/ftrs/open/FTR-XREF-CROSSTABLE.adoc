===== FTR-XREF-CROSSTABLE — Cross-Table (Matrix) ↔ Relation Transformation

*Status:* New

*Purpose:*::
Provide a generic transformation between matrix (cross-table) representations and explicit relational representations.

*Context:*::
A common tabular pattern uses one axis for row identifiers and another for column identifiers, with cell values representing relationships or attributes.

*Scope:*::
- Transform matrix layout into a relation of the form (row_key, column_key, value)
- Transform such relations back into matrix form
- Preserve all values without interpreting their semantics

*Non-Goals:*::
- No interpretation of cell values
- No validation beyond structural consistency

*Solution:*::
- Expand operation: matrix → list of tuples (row_key, column_key, value)
- Contract operation: list of tuples → matrix
- Support configurable row/column key detection

*Acceptance:*::
- Transformation is lossless for all non-ambiguous inputs
- Empty and missing values are handled consistently
- Roundtrip (matrix → relation → matrix) produces identical structure
- Implementation is independent of value semantics
