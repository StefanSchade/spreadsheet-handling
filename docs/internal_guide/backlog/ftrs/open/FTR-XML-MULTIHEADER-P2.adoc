===== FTR-XML-MULTIHEADER-P2 — XML Adapter parity for MultiIndex headers

* *Context* `FTR-MULTIHEADER-P2` delivered MultiIndex parity for XLSX + JSON and the flatten/unflatten edge steps. XML adapter support is not present in the current codebase.
* *Goal* Introduce XML read/write adapter behavior consistent with MultiIndex canonical headers and edge flatten/unflatten pipeline strategy.
* *Rationale* Keep `FTR-MULTIHEADER-P2` scope aligned with implemented adapters while preserving a clear parity roadmap for XML.
* *Acceptance*
- XML writer emits nested structures by default for MultiIndex headers.
- Flat XML output is achievable via `flatten_headers` edge step.
- XML reader can rebuild MultiIndex-compatible frames (or a documented equivalent) for round-trips.
- Tests cover: multi->xml (nested), multi->flatten->xml (flat), and xml->frames->unflatten round-trip conventions.
