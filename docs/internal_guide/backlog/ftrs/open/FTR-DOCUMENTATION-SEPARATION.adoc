===== FTR-DOCUMENTATION-SEPARATION — Split "internal guide" to get the usual set of documents

*Context*::
The "internal guide" is a pragmatic place to collect pieces of documentation that need a place (compare <<Motivation and Scope of the Internal Guide, Motivation and Scope>>) Towards a release we want to provide a structure that is more consumer oriented.

*Goal*::
Create an adequate set of documents you would expect as an observer

*Rationale*::
Project should become mor accessible

*Acceptance*::
* Provide a minimal set of documents you would expect:
- Developer Guide
- Architecture Documentation (a minimalistic arc42 structure)
- Project Planning
- ....
* Each document should have the content / structure an observer would expect
* All information currently in the "internal guide" is retained at an adequate position
