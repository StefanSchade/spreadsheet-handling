===== FTR-PIPELINE-FACADE-AND-SIMPLE-CLI-CLEANUP-P4A -- Retire legacy pipeline facades and realign simple CLI wrappers

*Status:* Open
*Epic:* Phase 4a -- First feature wave on top of the new architecture

*Placement note:*::
This is intentionally planned near the end of Phase 4a.

The current refactoring wave may still reveal a little more cleanup demand in
step registration, domain extraction, and related seams.  The right time to
trim the transitional facades is therefore after the active Phase-4a refactors
have landed and the intended architecture is stable enough to expose directly.

*Purpose:*::
Remove transitional public shims that no longer match the intended architecture:

- legacy simple CLI names such as `sheets-pack` / `sheets-unpack`
- the broad backward-compatibility facade in `pipeline/pipeline.py`
- the equally broad re-export surface in `pipeline/__init__.py`

The goal is not to redesign the product, but to make the public entry points
match the architecture that already exists:

- simple wrappers delegate to `application.orchestrator.orchestrate()`
- pipeline exposes an intentional API instead of "everything still works"

*Context:*::
The current system already has the right internal execution seam:
`application.orchestrator.orchestrate()` owns the end-to-end run path, and the
pipeline layer owns step binding plus step execution.

Two older compatibility surfaces remain from an earlier project phase:

- `sheets-pack` / `sheets-unpack` reflect the old "pack files into Excel" POC
  vocabulary and no longer describe the richer spreadsheet/adapters/pipeline
  model clearly
- `pipeline/pipeline.py` still advertises itself as a backward-compatible
  facade that re-exports types, registry functions, and many step factories
- `pipeline/__init__.py` repeats that broad facade shape at package level

These surfaces were useful during transition, but they now hide the intended
architecture and keep historical import patterns alive longer than necessary.

The demo repository has already moved away from the old `pack` / `unpack`
access path, which lowers the risk of tightening these shims now.

*Goal:*::
At the end of this ticket:

- retained simple CLI wrappers have clearer names for the concrete simple flows
  they cover
- those wrappers are thin and delegate only to
  `application.orchestrator.orchestrate()`
- legacy `pack` / `unpack` naming is no longer the primary documented surface
- the pipeline package exposes an intentional public API rather than a broad
  historical compatibility facade

*Scope:*::
- Replace the current legacy simple-wrapper positioning of `sheets-pack` /
  `sheets-unpack` with clearer, format-explicit thin wrappers for the retained
  simple flows.
- The replacement names should state the concrete simple conversion they cover,
  for example `json2xlsx` / `xlsx2json`, or an equivalent clearer naming chosen
  during implementation.
- Keep these wrappers thin: they may build I/O descriptors and pass defaults,
  but they must delegate into `application.orchestrator.orchestrate()`.
- Do not let simple wrappers reach behind the application orchestrator into
  lower pipeline internals.
- Review `src/spreadsheet_handling/pipeline/pipeline.py` and remove or sharply
  reduce its role as a broad backward-compatible facade.
- Review `src/spreadsheet_handling/pipeline/__init__.py` and keep only the
  intentionally supported pipeline import surface there.
- Adjust internal imports, tests, packaging entry points, and documentation to
  use the intended public surface.
- If compatibility aliases remain temporarily, mark them as compatibility-only
  and keep them as minimal delegates.

*Non-Goals:*::
- No redesign of the generic run path or orchestrator contract.
- No new routing, branching, or workflow-engine behavior.
- No attempt to redesign all casual-user UX beyond these thin wrapper entry
  points.
- No requirement to remove every unrelated compatibility shim elsewhere in the
  package.
- No attempt to decide the final Phase-6 simple-user product story in this
  ticket.

*Solution Direction:*::
- Prefer `application.orchestrator.orchestrate()` as the only internal
  execution seam used by retained simple CLI wrappers.
- Replace old `pack` / `unpack` terminology with clearer names that reflect the
  actual simple flows kept by the project.
- Decide explicitly whether legacy `sheets-pack` / `sheets-unpack` remain as
  documented compatibility aliases for one cycle or are removed immediately if
  no maintained consumer path depends on them.
- Reduce `pipeline/pipeline.py` to a very small intentional surface or remove it
  entirely if all maintained imports can move to the direct modules
  (`pipeline.registry`, `pipeline.types`, `pipeline.steps` where appropriate).
- Reduce `pipeline/__init__.py` to the same intentional public API rather than a
  second broad facade.
- Update the package import surface and tests to make the intended entry points
  obvious:
  - simple execution through `application.orchestrator.orchestrate()`
  - step binding / registry through `pipeline.registry`
  - pipeline types through `pipeline.types`

*Implementation Direction:*::
- Audit current imports of:
  - `spreadsheet_handling.pipeline.pipeline`
  - `spreadsheet_handling.pipeline`
  - `make_*_step` re-exports used only through facades
  - `sheets-pack` / `sheets-unpack` entry points and aliases
- Migrate maintained imports to the direct module that now owns the concept.
- Introduce clearer simple CLI wrapper names and wire them to
  `application.orchestrator.orchestrate()`.
- Update `cli/__init__.py` and packaging entry-point declarations so the simple
  wrappers reflect the chosen maintained names.
- Keep any temporary compatibility aliases visibly secondary and mechanically
  thin.
- Remove or shrink the pipeline facade modules once the maintained imports and
  tests no longer rely on them.

*Acceptance:*::
- The primary documented simple-wrapper commands no longer use the legacy
  `pack` / `unpack` naming.
- Retained simple-wrapper commands delegate to
  `application.orchestrator.orchestrate()` and do not reach behind it.
- Any surviving legacy `sheets-pack` / `sheets-unpack` entry points are clearly
  documented as compatibility aliases, or they are removed.
- `src/spreadsheet_handling/pipeline/pipeline.py` no longer serves as a broad
  "re-export everything" facade, or it is removed entirely.
- `src/spreadsheet_handling/pipeline/__init__.py` exposes only the intentional
  supported pipeline API.
- Tests are updated to the new wrapper names / alias policy and to the intended
  import surface.
- User-facing docs and examples no longer present `pack` / `unpack` as the
  preferred current path.

*Suggested Test Areas:*::
- `tests/unit/test_cli_delegation.py`
- `tests/unit/cli/test_cli_default_pipelines.py`
- `tests/unit/test_orchestrator_smoke.py`
- consumer/import-surface tests such as:
  - `tests/integration/test_consumer_smoke.py`
- a small focused import-surface test for the intended `pipeline` API if the
  final surface becomes substantially narrower

*Relation to Existing Work:*::
- This ticket should run after the active Phase-4a refactor wave has settled,
  especially after:
  - `FTR-PIPELINE-STEP-REGISTRATION-NORMALIZATION`
  - `FTR-FK-HELPER-DOMAIN-EXTRACTION`
- It complements `FTR-ARCHITECTURE-FITNESS-GUARDS-P4X` by removing leftover
  transitional surfaces that would otherwise keep historical import paths alive.
- It pulls a concrete structural subset of `FTR-SIMPLE-CLI-SHIMS-P6` forward:
  cleanup of legacy shim names and legacy pipeline facades.
- The broader later question of the ideal simple-user command set may still
  remain for Phase 6, but this ticket removes the already-clear structural
  legacy.
