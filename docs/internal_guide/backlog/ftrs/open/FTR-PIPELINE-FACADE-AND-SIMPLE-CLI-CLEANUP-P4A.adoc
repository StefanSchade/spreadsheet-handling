===== FTR-PIPELINE-FACADE-AND-SIMPLE-CLI-CLEANUP-P4A -- Tighten pipeline facades and deprecate legacy CLI names

*Status:* Open (rescoped 2026-04-29)
*Epic:* Phase 4a -- First feature wave on top of the new architecture

*Rescope note (2026-04-29):*::
The original version of this FTR proposed renaming `sheets-pack` / `sheets-unpack`
to format-explicit names like `json2xlsx` / `xlsx2json` and removing the legacy
CLI surface. After further reflection, that direction was too aggressive:

1. The *mechanism* of opinionated CLI shortcuts that orchestrate a pipeline
   behind the user's back is valuable and should be preserved as a *pattern*.
   Future use cases (e.g., a `create-calendar` command that reads date ranges
   and orchestrates a workbook pipeline) would use the same pattern. Removing
   it or renaming it to format-specific names would obscure this generality.

2. The `--steps` path was previously considered for removal
   (`FTR-CLI-CONSOLIDATE-REMOVE-STEPS`, discarded). The current `config +
   pipeline` path is a wrapper on the older direct `--steps` model, not a
   replacement. Removing `--steps` would reduce ad-hoc capability without
   simplifying the CLI module. This is already documented in the Phase-3g
   notes.

3. The user base is effectively zero external users with three controlled
   consumer repositories. Breaking CLI names buys nothing.

The rescoped FTR focuses on the parts that are actually useful now:

- tighten the `pipeline/pipeline.py` facade
- deprecate (not remove) `sheets-pack` / `sheets-unpack` in documentation
- keep the simple-shortcut *mechanism* explicitly as an intended pattern

*Purpose:*::
Reduce the transitional pipeline facade surface and mark the legacy CLI names
as deprecated examples rather than the primary user path.

The simple-shortcut mechanism itself -- thin CLI commands that read a few
parameters and delegate to `application.orchestrator.orchestrate()` -- is an
intentional product pattern and must be preserved.

*Context:*::
The current system already has the right internal execution seam:
`application.orchestrator.orchestrate()` owns the end-to-end run path, and the
pipeline layer owns step binding plus step execution.

Two older compatibility surfaces remain from an earlier project phase:

- `sheets-pack` / `sheets-unpack` reflect the old "pack files into Excel" POC
  vocabulary. The names no longer describe the richer model clearly, but the
  *mechanism* (thin CLI -> orchestrator with sensible defaults) is valuable.
- `pipeline/pipeline.py` still advertises itself as a backward-compatible
  facade that re-exports types, registry functions, and many step factories.
  `sheets_pack.py` still imports `make_bootstrap_meta_step` and
  `build_steps_from_yaml` from it instead of using direct module imports.
- `pipeline/__init__.py` repeats that broad facade shape at package level.

The demo repository already uses `sheets-run` as the primary path.

*Goal:*::
At the end of this ticket:

- `pipeline/pipeline.py` is reduced to an intentional API or removed
- `pipeline/__init__.py` exposes only the intentional supported surface
- `sheets-pack` / `sheets-unpack` are marked deprecated in documentation and
  emit a deprecation notice, but remain functional
- the simple-shortcut *pattern* is documented as an intended extension point
  for future domain-specific CLI commands

*Scope:*::
- Review `src/spreadsheet_handling/pipeline/pipeline.py` and remove or sharply
  reduce its role as a broad backward-compatible facade.
- Migrate `sheets_pack.py` imports from `pipeline.pipeline` to the direct
  owning modules (`pipeline.registry`, `pipeline.steps`).
- Review `src/spreadsheet_handling/pipeline/__init__.py` and keep only the
  intentionally supported pipeline import surface there.
- Mark `sheets-pack` / `sheets-unpack` as deprecated in documentation, help
  text, and examples. They should remain functional but visibly secondary.
- Optionally rename the CLI entry-point module directory from
  `cli/apps/sheets_pack.py` to something like `cli/apps/example_pack.py` to
  make the example/template nature visible. This is a soft suggestion, not
  mandatory.
- Document the simple-shortcut pattern: "a thin CLI command that reads
  user-facing parameters and delegates to `orchestrate()` with a built-in
  default pipeline" is an *intended extension point*, not legacy.

*Non-Goals:*::
- No removal of `sheets-pack` / `sheets-unpack` entry points. They stay
  functional.
- No removal of `--steps`. That path is the underlying model and remains.
- No redesign of the generic run path or orchestrator contract.
- No new routing, branching, or workflow-engine behavior.
- No attempt to decide the final simple-user product story. That remains
  deferred to Phase 6 (`FTR-SIMPLE-CLI-SHIMS-P6`).

*Acceptance:*::
- `src/spreadsheet_handling/pipeline/pipeline.py` no longer serves as a broad
  "re-export everything" facade, or it is removed entirely.
- `src/spreadsheet_handling/pipeline/__init__.py` exposes only the intentional
  supported pipeline API.
- `sheets-pack` / `sheets-unpack` emit deprecation notices and documentation
  marks them as legacy examples, not the primary path.
- No maintained code imports from `pipeline.pipeline` when a direct module
  import exists.
- The simple-shortcut mechanism is documented as an intended pattern.
- Tests are updated to use the intended import surface.

*Suggested Test Areas:*::
- `tests/unit/test_cli_delegation.py`
- `tests/unit/cli/test_cli_default_pipelines.py`
- `tests/unit/test_orchestrator_smoke.py`
- a focused import-surface test for the intended `pipeline` API

*Relation to Existing Work:*::
- This ticket should run after the active Phase-4a refactor wave has settled.
- The broader question of the ideal simple-user command set remains for Phase 6
  (`FTR-SIMPLE-CLI-SHIMS-P6`).
- `FTR-CLI-STEPS-DEPRECATION-P3G` handles `--steps` deprecation messaging
  separately.

*Priority reassessment (2026-04-29):*::
The rescoped FTR is smaller and lower risk than the original. The pipeline
facade cleanup is the only part with real architectural value. The CLI
deprecation markers are nice-to-have hygiene.

This FTR can be executed before the beta but is no longer on the critical path.
If time is tight, the pipeline facade cleanup can land as a small commit without
the full CLI deprecation ceremony.
