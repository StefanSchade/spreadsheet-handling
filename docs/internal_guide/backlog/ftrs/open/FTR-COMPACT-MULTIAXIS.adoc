===== FTR-COMPACT-MULTIAXIS -- Multi-Axis Compact Representation

*Status:* New

*Purpose:*::
Compose cross-table expansion and cell-code handling into a generic explicit
representation for compact multi-axis spreadsheets.

*Context:*::
Some spreadsheet views combine several structural ideas at once:

* a row axis, for example features or configuration items
* a column axis, for example product variants grouped by product family
* compact cell codes, for example `E-R-K` or `A-V-Z`
* legend/codebook blocks that explain and optionally validate those codes

The repository example should stay neutral, using products, markets, product
variants, or configuration items rather than private domain vocabulary.

*Scope:*::

* Combine cross-table expansion with cell-code validation or token decoding.
* Produce an explicit generic long form.
* Build on top of `FTR-XREF-CROSSTABLE` and `FTR-CELL-CODEC`.
* Keep domain interpretation outside this generic feature.

*First Explicit Output Form:*::

The first explicit form should be generic long form:

[source,text]
----
row_key | column_key | code | group?
----

Where:

* `row_key` comes from the row-axis entity.
* `column_key` comes from the column-axis entity.
* `code` is either the whole-cell code or one decoded token.
* `group` is optional metadata from a legend/codebook or codec config.

Later application layers may derive richer domain-specific tables from this
generic long form.

*Non-Goals:*::

* No assumption about business meaning.
* No fixed domain schema.
* No hardcoded encoding strategy.
* No private/professional use-case vocabulary in repo fixtures.

*Solution:*::

* Provide a composed pipeline step that chains:
  . XREF cross-table expansion
  . cell-code validation or token decoding
  . optional legend/codebook lookup for group/category metadata
* Allow configuration of how token/code groups map to abstract dimensions.
* Preserve sparse handling: empty cells produce no explicit rows by default.

*Dependencies:*::

* <<FTR-XREF-CROSSTABLE#,FTR-XREF-CROSSTABLE>>
* <<FTR-CELL-CODEC#,FTR-CELL-CODEC>>
* <<FTR-LEGEND-BLOCKS#,FTR-LEGEND-BLOCKS>>

*Configuration (Meta):*::

* `axes`: list of axis definitions
* `codec`: reference to the cell-codec configuration
* `crosstab`: reference to the cross-table configuration
* optional legend/codebook reference
* optional token/code categories
* optional constraints on valid combinations

*Acceptance:*::

* Given a matrix where columns represent one dimension and cell values encode
  another, the step produces generic long form.
* Whole-cell codes can roundtrip without being split.
* A fixture with at least two axes round-trips:
  `compact -> explicit -> compact` yields the original.
* Empty cells produce no rows in the explicit output by default.
* Configuration errors raise clear diagnostics.
* The composed step is usable as a standard pipeline step via
  `build_steps_from_config`.
