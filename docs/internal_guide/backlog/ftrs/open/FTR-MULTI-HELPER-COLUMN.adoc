===== FTR-MULTI-HELPER-COLUMN - Helper Columns & FK Enhancements (Multi-Helper)

*Context*::
Today, helper columns are typically limited to a single label. Real-world cases need multiple attributes (name, category, description, ...).

*Goal*::

- Support multi-helper columns per FK and ensure style/validation/drop logic respects the expanded set.
- Helper columns should sit next to their FK ID columns for readability. desired ordering should be configurable

*Rationale*::
Keeps keys stable while surfacing multiple descriptive attributes for review/UX.

*Acceptance*::

- Configurable helper column list per FK mapping.
- Drop-helpers removes all helper columns for a given prefix.
- Helpers are styled consistently (e.g., shaded).
- Domain step `reorder_helpers_next_to_fk` moves helper columns by prefix pattern.
- Unit tests assert column order (incl. config variant)
