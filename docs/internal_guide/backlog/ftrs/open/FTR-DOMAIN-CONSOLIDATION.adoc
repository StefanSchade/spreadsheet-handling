===== FTR-DOMAIN-CONSOLIDATION — Fold scattered domain code into a tidy layer

*Purpose:*::
Simplify the domain by consolidating partially disconnected packages into a single, well-scoped domain module with clear boundaries to adapters and IR.

*Scope:*::
Domain-only; no adapter behavior changes. Requires a coverage gate before refactor to protect functionality.

*Solution:*

- Establish a **coverage floor** (e.g., 80% domain, 70% overall) and add a CI gate.
- Identify public domain API (functions, data types) and mark the rest internal.
- Merge/relocate `core`/`pk` into `domain/` with subpackages `model`, `transformations`, `validation`.
- Provide deprecation shims (`from domain.old import X` → warnings) for one release.

*Acceptance:*

- CI passes with new coverage gate; public API unchanged (or documented with deprecation warnings).
- Imports from the old packages emit a deprecation warning and still work for one cycle.
- Docs show the new domain layout; internal modules are not referenced from examples.
- No adapter/router changes needed; roundtrip tests remain green.
- Packages, `core` / `pk` are gone (or restructures so they take a role in the domain)
