===== FTR-IR-SHIM-CLEANUP — Remove temporary compatibility shims from new adapters

*Purpose:*::
During the IR refactor several temporary shims (e.g., `render_plan` → `render_workbook`) were introduced to keep legacy tests green.
This feature removes or consolidates those aliases once the test suite and consuming code are fully migrated.

*Scope:*::
Affects functions and symbols within active IR-based backends (not legacy adapters).
Targets files such as `io_backends/xlsx/openpyxl_renderer.py` and related IR renderer modules.

*Solution:*::
- Identify all temporary aliases and `Deprecated:` markers in IR and rendering layers.
- Replace external references with the canonical API.
- Delete the shims and update `__all__` lists.
- Adjust test imports accordingly.

*Acceptance:*::
- No references to deprecated symbols remain in code or tests.
- CI passes with shims removed.
- CHANGELOG and docs mention cleaned-up canonical names.

*Extention for tidy up in context of other FTRs:*::
* *FTR-IR-WRITEPATH-P1*
** Scope:
*** XLSX renderer drops legacy IR `.sheets` acceptance; canonical input is RenderPlan (`.ops`).

** Acceptance
