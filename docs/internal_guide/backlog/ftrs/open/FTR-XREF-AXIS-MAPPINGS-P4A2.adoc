===== FTR-XREF-AXIS-MAPPINGS-P4A2 -- Explicit XRef axis mappings

*Status:* New

*Purpose:*::
Introduce explicit axis mapping metadata for XRef/crosstable views so row and
column axis identity, display labels, ordering, and optional grouping can be
described without parsing visible spreadsheet text.

*Context:*::
`FTR-XREF-CROSSTABLE` intentionally starts with flat matrix columns and explicit
`row_keys`. That is enough for the first compact-transform path, but richer
views need to separate these concerns:

* identity keys used for import/export
* display labels shown to users
* stable order of row and column axis members
* optional grouping such as product family -> product variant

This is not nested JSON. It is a spreadsheet-axis contract for crosstab views.

*Scope:*::

* Define a metadata shape for row and column axis mappings.
* Support axis members sourced from existing frames, for example a
  `product_variants` table.
* Persist resolved axis mappings under workbook metadata.
* Let XRef transforms use axis mappings instead of deriving identity from
  visible labels.
* Preserve deterministic order for both expansion and contraction.

*Non-Goals:*::

* No nested list/object JSON fan-out.
* No direct spreadsheet editing of new axis members inside the matrix.
* No domain-specific product, market, finance, or credit semantics.
* No multi-row header rendering requirement in this FTR.

*Example Configuration:*::

[source,yaml]
----
xref_axis_mappings:
  product_feature_matrix:
    row_axis:
      source: features
      key: feature_id
      label_fields: [feature_label]
      order_by: [sort_key]
    column_axis:
      source: product_variants
      key: variant_id
      label_fields: [variant_label]
      group_fields: [product_family]
      order_by: [product_family_sort, variant_sort]
----

*Resolved Metadata Shape:*::

[source,yaml]
----
xref_axis_mappings:
  product_feature_matrix:
    resolved:
      row_axis:
        - key: f1
          label: Currency
          position: 0
      column_axis:
        - key: FZ-AD
          label: Fixed AD
          groups:
            product_family: Fixed
          position: 0
----

*Acceptance:*::

* Axis mappings can be resolved from ordinary DataFrames.
* Missing required key, label, or order fields fail with clear diagnostics.
* The resolved mapping preserves stable axis order.
* XRef import/export can use the resolved keys without parsing visible labels.
* The metadata shape is documented and does not imply nested JSON support.
