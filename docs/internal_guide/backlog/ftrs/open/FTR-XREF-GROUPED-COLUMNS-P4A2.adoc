===== FTR-XREF-GROUPED-COLUMNS-P4A2 -- Grouped column axis for XRef crosstables

*Status:* New

*Purpose:*::
Allow XRef/crosstable views to represent a structured column axis, for example
product family -> product variant, while keeping the underlying transformation
generic and relation-based.

*Context:*::
The first `FTR-XREF-CROSSTABLE` slice rejects MultiIndex/tuple columns on
purpose. That keeps the base transform flat and safe.
+
The next useful spreadsheet-facing extension is not nested lists. It is a
grouped crosstab header:

[source,text]
----
          Fixed                 Variable
          FZ-AD  FZ-TD          VAR-AD  VAR-TD
feature
currency  E      E              E       E
amount    E-R-K  E-R-K          E-R-K   E-R-K
----

Expanded long form should carry the structured column-axis fields explicitly:

[source,text]
----
feature_id | product_family | variant_id | value
----

*Dependencies:*::

* <<FTR-XREF-CROSSTABLE#,FTR-XREF-CROSSTABLE>>
* <<FTR-XREF-AXIS-MAPPINGS-P4A2#,FTR-XREF-AXIS-MAPPINGS-P4A2>>
* Completed MultiHeader/MultiIndex spreadsheet support from earlier phases

*Scope:*::

* Support grouped column-axis configuration for XRef transforms.
* Expand grouped columns into explicit long-form columns.
* Contract explicit long-form rows back into a grouped-column crosstab.
* Support both:
  ** flat technical column keys with separate axis metadata, and
  ** MultiIndex columns when explicitly configured.
* Preserve identity via axis mappings, not visible labels.

*Non-Goals:*::

* No cell-code parsing; that remains `FTR-CELL-CODEC`.
* No multi-axis composition; that remains `FTR-COMPACT-MULTIAXIS`.
* No nested JSON object/list import or export.
* No automatic inference from arbitrary merged header text.

*Configuration Sketch:*::

[source,yaml]
----
xref:
  matrix: product_feature_matrix
  relation: feature_product_codes
  row_keys: [feature_id]
  value: code
  column_axis:
    key: variant_id
    levels:
      - name: product_family
      - name: variant_id
    mapping: product_feature_matrix
----

*Acceptance:*::

* A grouped-column matrix expands into long form with one output column per
  configured column-axis level.
* Contracting the long form reproduces the grouped matrix with stable ordering.
* MultiIndex/tuple columns remain rejected unless the grouped-column mode is
  explicitly enabled.
* Visible labels are never parsed as identity.
* A neutral fixture uses products/features/variants, not private domain
  vocabulary.
