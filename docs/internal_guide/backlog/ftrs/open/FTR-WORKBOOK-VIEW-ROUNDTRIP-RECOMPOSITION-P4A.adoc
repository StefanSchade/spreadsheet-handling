===== FTR-WORKBOOK-VIEW-ROUNDTRIP-RECOMPOSITION-P4A -- Recompose edited workbook views into canonical frames

*Status:* Open
*Priority:* P4A
*Area:* Pipeline / roundtrip semantics
*Prerequisites:* FTR-WORKBOOK-REIMPORT-VIEW-MAPPING-P4A, FTR-WORKBOOK-REIMPORT-DERIVED-COLUMN-POLICY-P4A

==== Purpose

Make the workbook readback path declarative enough to turn edited spreadsheet
views back into canonical source frames.

Generating a friendly workbook is only half of the business-editing use case.
The important roundtrip is:

`canonical JSON -> editable workbook views -> edited workbook -> canonical JSON`

A prototypical consumer reimport pipeline currently only reads the workbook
back to JSON. Mapping visible sheets and removing helper columns are handled by
the prerequisite reimport FTRs. This FTR focuses on recomposing edited view
payloads, especially matrix-shaped views, into canonical relation frames.

==== Goals

* Use explicit sheet/frame mapping from the reimport mapping FTR.
* Use derived/helper-column policy from the reimport derived-column FTR.
* Support reverse compact-matrix or xref expansion from an editable matrix view
  back to the canonical relation.
* Preserve or deliberately ignore derived/read-only columns according to
  lifecycle/editability metadata.
* Detect payload conflicts such as two editable views producing different
  values for the same canonical tuple, and fail unless a precedence policy is
  configured.
* Keep the readback pipeline inspectable and runnable through `sheets-run`.

==== Non-goals

* Do not require automatic conflict resolution in this slice.
* Do not infer canonical targets from arbitrary sheet names without
  configuration.
* Do not trust protected/read-only cells as a security boundary.
* Do not own helper-column mismatch policy; that belongs to
  `FTR-WORKBOOK-REIMPORT-DERIVED-COLUMN-POLICY-P4A`.
* Do not implement multi-user merge reconciliation here.

==== Acceptance

* A business-facing workbook can be read back through a configured pipeline that
  produces canonical relation frames such as compact matrix code rows.
* Matrix cells can be expanded back into explicit relation rows with legend
  validation still enforced.
* Conflicting editable payload values fail with clear findings unless an
  explicit precedence policy is configured.
* The roundtrip has focused tests for edited matrix values, unchanged matrix
  values, and duplicate/conflicting tuple output.
* ODS and XLSX readback dispositions are explicit for the same configured
  roundtrip path.
