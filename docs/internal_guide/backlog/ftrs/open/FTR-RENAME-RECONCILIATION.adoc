===== FTR-RENAME-RECONCILIATION — Detect and reconcile sheet/column renames against _meta references

*Status:* Open (deferred — post-beta robustness/UX)
*Epic:* Phase 5 — Stabilization and Internal Quality
*Priority:* Low

*Related-to:*

* <<FTR-META-SCHEMA-VALIDATION-P5#,FTR-META-SCHEMA-VALIDATION-P5>>
* <<FTR-SPREADSHEET-PARSE-CONTRACT-P3I#,FTR-SPREADSHEET-PARSE-CONTRACT-P3I>>

'''

====== Purpose

When editors rename sheets or columns in Excel/ODS, `_meta` references
(constraints, legend placements, xref axis mappings, discriminator targets)
break silently. This FTR addresses detection, reporting, and optional
reconciliation of such renames during re-import.

'''

====== Context

The `_meta` sidecar references sheets and columns by name. The pipeline owns
the structure; Excel/ODS is the editing surface for values. However, nothing
prevents an editor from renaming a tab or column header. When this happens, the
next pipeline run either fails opaquely or silently drops configuration that no
longer matches.

Affected `_meta` areas:

* `constraints[].sheet` / `constraints[].column`
* `legend_blocks[].placement.sheet` / `.target`
* xref axis column references
* discriminator-split target patterns
* named-range scopes

'''

====== Problem statement

* Silent breakage: renamed references produce no error, just missing behavior
* Opaque errors: when validation does fail, the message doesn't suggest a rename
* No recovery path: users must manually hunt for what changed

'''

====== Options (to be evaluated post-beta)

====== Option 1: Clear error messages on dangling references (minimum viable)

At re-import time, validate all `_meta` name references against the actual
sheet/column inventory. For any mismatch:

* emit a clear diagnostic: "Sheet `X` referenced in `_meta.constraints` not
  found. Available sheets: [A, B, C]."
* suggest close matches (edit-distance or prefix).

Effort: small — extends existing parse-contract validation.
Value: immediate usability improvement.

====== Option 2: Heuristic reconciliation suggestions

When a referenced name is missing but a plausible rename candidate exists
(same column structure, similar name, same position), suggest reconciliation:

* "Sheet `subject_labels_de` → `Subjects_DE`? (same columns, position 3)"
* Never auto-apply; always require user confirmation or config update.

Matching criteria:

* Column-set overlap (Jaccard similarity > 0.8)
* Edit-distance on name (Levenshtein ≤ 3)
* Position stability (same tab index)
* Row-count similarity

Effort: medium — needs a reconciliation pass and suggestion UI/output.

====== Option 3: Structural fingerprinting

Store a lightweight fingerprint per sheet at render time (hash of column names +
first N data rows). On re-import, match fingerprints to detect renames
regardless of name similarity.

Effort: medium-high — requires fingerprint storage in `_meta` and a matching
algorithm.

====== Option 4: Convention enforcement (documentation + protection)

Document the convention: "Structure comes from the pipeline; Excel edits values
only. Do not rename tabs or columns."

Optionally enforce via Excel sheet-protection on tab names / header rows.

Effort: minimal — documentation + optional protection flag in render config.

'''

====== Recommended phasing

. *Phase 5 (first):* Option 1 — clear error messages. Low effort, high value.
. *Phase 5 (second):* Option 4 — document convention, optional header protection.
. *Post-0.2.0:* Option 2 — heuristic suggestions for power users.
. *Later:* Option 3 — fingerprinting if heuristics prove insufficient.

'''

====== Non-goals

* VBA/macro-based rename tracking (too heavyweight, platform-dependent)
* Automatic silent reconciliation (too risky — renames must be explicit)
* Real-time rename detection (not possible without Excel plugin)

'''

====== Acceptance criteria (to be refined when activated)

. Dangling `_meta` references produce clear, actionable error messages.
. Close-match suggestions are shown when available.
. No silent data loss from undetected renames.
. Convention is documented in user guide.
