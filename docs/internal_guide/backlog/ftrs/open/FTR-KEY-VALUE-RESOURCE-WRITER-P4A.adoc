===== FTR-KEY-VALUE-RESOURCE-WRITER-P4A -- Write deterministic key-value resource files

*Status:* Open
*Priority:* P4A
*Area:* Output generation / localized resources

==== Purpose

Add a generic deterministic writer for key-value resource files such as
`.properties`-style outputs.

Structured YAML output is useful for nested configuration trees, but localized
labels and messages often need flat key-value files per locale and resource
type.

==== Scope

* Write one or more key-value files from frame data.
* Support configured key column and value column.
* Support partitioning output files by locale, resource type, or other columns.
* Support deterministic ordering.
* Support configurable escaping/newline policy.
* Return a generated-file report frame.

==== Example Shape

[source,yaml]
----
- step: write_key_value_resources
  source: localized_resource_values
  output_dir: ./build/resources
  file_pattern: "{resource_type}_{locale}.properties"
  key: resource_key
  value: text
  sort_by: [resource_key]
  encoding: utf-8
----

==== Non-goals

* Do not implement a full template language.
* Do not infer output file names from localized sheet names.
* Do not replace the structured YAML writer.
* Do not perform application-specific fallback resolution unless the input frame
  has already been normalized by the fallback/override semantics.

==== Acceptance

* The writer emits deterministic key-value files from a configured frame.
* Output paths cannot escape the configured output directory.
* Duplicate keys in the same output file fail with a clear error.
* Missing required key/value columns fail with a clear error.
* Tests cover partitioned files, ordering, escaping, duplicate keys, path safety,
  and generated-file reporting.
