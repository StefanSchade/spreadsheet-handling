===== FTR-PARTITION-BY-DOMAIN — Partition/merge by domain component attribute

*Context*::
Teams work on mutually-exclusive domain components. Partitioning tables by a `component` attribute reduces merge conflicts.

*Goal*::
Split/merge frames by an attribute resolved via join on `variable` (configurable), as pre-persist step.

*Rationale*::
Improves Git automerge behavior and local focus.

*Solution*::

- Split/merge frames by a mutually-exclusive ‘component’ attribute (lookup via join on variable) to reduce merge conflicts.

*Acceptance*::

- Deterministic partition layout & stable recomposition.
- Join config explicit; no silent heuristics that could break bijection.
- Tests ensure roundtrip partition→merge is lossless.
- deterministic partition layout; recomposition identical; configurable pre-persist step.
