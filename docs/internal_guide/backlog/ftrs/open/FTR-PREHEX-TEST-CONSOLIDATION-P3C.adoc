===== FTR-PREHEX-TEST-CONSOLIDATION-P3C — Retire, migrate, or promote pre-hex tests after 3a/3b stabilization

*Status:* Open
*Epic:* Phase 3c — Test Consolidation after Architecture Stabilization

*Purpose:*::
Systematically classify the `legacy_pre_hex` test suite so that historical implementation coupling is removed without silently discarding still-relevant behavior expectations.

*Context:*::
The `legacy_pre_hex` suite is intentionally quarantined and no longer part of the default test flow. Some tests preserve obsolete architectural details, while others still describe behavior that must remain true under the new architecture.

*Scope:*::
- Classify pre-hex tests into `retire`, `rewrite`, or `promote`.
- Remove tests that only protect obsolete implementation structure.
- Re-express still-valid behavior expectations in the current architectural test layout.
- Use this work as a confirmation step after 3a and 3b have stabilized the intended structure.

*Classification Rules:*::
- *Retire* -- historical implementation detail, no longer part of target architecture.
- *Rewrite* -- behavior still required, but old entrypoint/test structure is obsolete.
- *Promote* -- behavior already belongs in the regular active suite.

*Non-Goals:*::
- No blanket deletion of all pre-hex tests.
- No preservation of tests purely because they already exist.

*Acceptance:*::
- Every remaining pre-hex test has an explicit classification outcome.
- No still-relevant behavior is left protected only by an ignored legacy test.
- Default active suites cover the intended target architecture rather than historical package structure.
