=== Project Specific Conventions

These conventions are relevant for both AI (in the sense of a primer) and for human collaborators

==== related Repositories

* *spreadsheet-handling* — core tool, published on PyPI under MIT, hosted on GitHub.
* *spreadsheet-handling-demo* — educational usage and plugin examples, hosted on GitHub.

The demo and company repos consume the package from *spreadsheet-handling*.
Repository structures should remain aligned where possible.

==== Technology Choices

* **Language:** Python ≥ 3.10.
* **Docs/Comments:** English. Chat can be in English or German.
* **Tools:** `make`, `ruff`, `black`, `pytest`, `pytest-cov`, `compileall` for syntax.
* **Build Backend:** setuptools + setuptools-scm (no Hatchling).

==== CI/CD

* Public: GitHub Actions.
* Company: GitLab CI.
* Pipeline stages: lint → syntax → tests (+coverage) → build sdist/wheel → publish on tags.

==== Respect Project Conventions

* Follow existing Makefile and env patterns.
* Ensure commands work on Linux and WSL (watch path quoting).
* External interfaces are treated as intentional product surface, even in beta.
* Changes to outward-facing API or schema require explicit user approval before implementation.
* This includes CLI flags, YAML configuration keys, plugin step fields, documented import paths, and consumer-facing examples.
* Internal refactoring must not silently rename or reshape consumer-visible interfaces.

==== Testing

* Prefer small, in-memory DataFrames; assert schema, row count, and keys explicitly.
