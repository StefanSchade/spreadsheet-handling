== Release History

=== Releasenotes 0.1.0b5 -- draft

*Architecture and adapters*

- Completed the Phase-3 spreadsheet-adapter cleanup and added the new ODS/Calc
  adapter as a maintained second spreadsheet backend.
- Added cross-adapter parity coverage and clarified where validation semantics
  are intentionally portable versus adapter-near.
- Hardened the ODS parser with dedicated robustness tests, including
  non-project-generated fixtures and negative cases.

*CLI and run path*

- Kept `sheets-run` as the primary maintained execution path and tightened the
  Phase-3 backlog/demo alignment around that model.
- Clarified that the current beta keeps the demo integration intentionally
  `xlsx`-first even though ODS now exists in the core library.

*Demo integration*

- Re-based the maintained demo flows on checked-in `sheets-run` YAML configs
  instead of relying on `pack` / `unpack` as the primary demo path.
- Demo smoke tests now generate fresh workbook artifacts during test execution
  instead of relying on warmed `tmp/` outputs.

=== Releasenotes 0.1.0b4 — 2025-10-06

*CI, tests, and packaging*

- Hardened release workflow: full fetch depth for tag-based versioning; test selection aligned.
- Test targets cleaned up (markers/filters); legacy tests excluded by default.
- Package metadata and discovery tuned in pyproject.

*Architecture & CLI*

- Documentation and structure aligned toward a single orchestrator (run) with thin CLI wrappers.
- sheets-run refined: clearer profile/pipeline selection and inline I/O in --pipeline-yaml supported.
- sheets-pack / sheets-unpack: simplified to build config and delegate into the common execution path.

*Examples & docs*

- Added example pipeline.yml and sheets.yaml to demonstrate profile-based runs.
