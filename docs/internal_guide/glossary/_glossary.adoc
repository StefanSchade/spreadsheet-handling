== Glossary

[cols="1,10"]
|===

|
term
|
explanation

|
pipeline
|
User-configured sequence of frame transformations and validations. The pipeline
operates on frames and metadata; it does not own spreadsheet rendering.

|
flow
|
Sequence of rendering steps in the Render IR layer that turns selected frames
and canonical metadata into backend-neutral render operations.

|
frame
|
Domain-level tabular data object carried in `Frames`, normally a pandas
`DataFrame` keyed by frame name. A frame is not a workbook sheet.

|
canonical tuple frame
|
Long-form source or target frame whose configured key columns identify each
semantic tuple. Matrix/crosstable views should be recomposable into this shape.

|
matrix frame / crosstable frame
|
Domain-level pivoted frame where row identity remains in columns and a
configured matrix axis becomes payload columns. It may later be projected to a
sheet, but it is not itself a sheet.

|
discriminator
|
Configured column or value axis used to partition a canonical tuple frame into
discriminator-specific frames, for example locale, product, customer,
deployment mode, or software variant.

|
discriminator-specific frame
|
Frame produced by applying a discriminator split. It carries one configured
discriminator value implicitly through split metadata or recomposition config.

|
matrix axis / pivot axis
|
Configured values that become columns in a matrix/crosstable frame and become
the `column_key` values again when expanded back to tuples.

|
payload value
|
Editable semantic cell value carried in the canonical tuple frame and displayed
in matrix-frame payload cells. Helper/display columns are not payload unless
explicitly configured as such.

|
workbook view
|
Rendering policy that selects, orders, names, and annotates frames for
spreadsheet projection. It belongs at the projection boundary, not inside
domain transformation semantics.

|
sheet
|
Concrete spreadsheet tab in a workbook. Sheets are render/readback artifacts;
domain transformations should refer to frames unless they are explicitly
configuring workbook projection.

|
render IR
|
Backend-neutral spreadsheet intermediate representation (`WorkbookIR`,
`SheetIR`, `TableBlock`, `RenderPlan`) used between selected frames and concrete
spreadsheet backends.

|
backend
|
I/O adapter for a concrete storage format such as JSON, CSV, XLSX, or ODS.
Spreadsheet backends render or parse through the spreadsheet contract and
Render IR.

|===
