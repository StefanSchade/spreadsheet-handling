* FTR-COMPACT-TRANSFORMS — Xref ↔ Cross-table ↔ Compact string (E-V-Z) (bijective)
**  *Context:* We need a reversible mapping between multiple layered cross-tables and a single compact encoding (codes + separator).
**  *Goal:* Implement `contract_compact` (merge n cross-tables → compact) and `expand_compact` (compact → n cross-tables/tuples) as bijective pair.
** *Rationale:* Enables human-compact views in Excel while retaining machine-friendly long/xref structures.
** Solution Steps
*** Provide two bijective steps: (1) xref⇄cross-table; (2) stack multiple cross-tables into compact codes using per-table code + separator.

** *Acceptance*::
- Configurable `compact_maps` (code ↔ source), separator, and 2- vs. 3-tuple handling.
- Deterministic code ordering and validation list generation (with cap).
- Bijectivity tests for `expand(contract(X)) == X` and vice versa.
- lossless roundtrip across all three forms; separator + mapping fully meta-driven.
