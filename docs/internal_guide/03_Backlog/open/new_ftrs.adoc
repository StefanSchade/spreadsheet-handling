=== FTR-013: IR Renderer Parity with Legacy OpenPyXL Path

*Status:* New  |  *Epic:* Phase 3 — Functional Parity and IR-driven Features

*Context*::
During migration, the IR-backed XLSX renderer must match legacy outputs for core scenarios (style, validations, sheet naming).

*Goal*::
Achieve parity in layout and key behaviors; provide test fixtures that compare legacy vs IR outputs for selected cases.

*Rationale*::
Confidence for deprecating legacy path; documents expected equivalence.

*Acceptance*::
- IR `compose→passes→render` used by adapter under flag and defaulted in CI once green.
- Parity tests for header styles, filters, validations, meta persistence.
- Legacy-only features documented or re-implemented as passes.
