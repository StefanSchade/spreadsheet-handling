=== FTR-AUTORELEASE-POLICY — Conservative library, aggressive demo automation

** *Purpose:*  Minimize maintenance while avoiding silent breakage.

** *Policy:*
- Demo: Dependabot/ Renovate on PyPI; automerge lib bumps if demo CI + smoke pass.
- Library: Automerge only dev-tool patch bumps; no automerge for runtime deps or feature PRs.
- Release flow: tag RC → publish to TestPyPI → run full + demo smoke → manual promote tag to PyPI.
- Use `constraints.txt` for runtime pins; demo pins lib with `==`.

** *Acceptance:*
- CI workflows exist for RC→TestPyPI and promote-to-PyPI.
- Demo auto-updates itself on new PyPI releases and merges when smoke passes.
- Library never publishes to PyPI without the manual promote step.
