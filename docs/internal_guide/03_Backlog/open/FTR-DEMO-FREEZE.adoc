=== FTR-DEMO-FREEZE — Freeze consumer repo and add smoke guard

** *Purpose:*  Temporarily freeze the *spreadsheet-handling-demo* repository while the core library stabilizes, avoiding redundant sync work during active refactors.

** *Scope:*  Applies to the demo repository and its CI reference in the main project.

** *Solution:*
- Add a short README notice: “Repository frozen until core milestone M3 (IR parity + orchestrator).”
- Pin dependency to the latest released beta (`spreadsheet-handling==0.1.0b4`).
- Disable automatic pipeline syncs or update tasks.
- Add a lightweight CI smoke test in the main project that installs the pinned demo and runs a single `sheets-run` pipeline to verify import and basic roundtrip still function.

** *Acceptance:*
- Demo README includes the freeze statement and version pin.
- Main repo CI runs a minimal consumer smoke test using the frozen demo.
- No content or example updates occur in the demo during the freeze phase.
- Failures in the smoke test are treated as breaking changes in the core interface.

** *Epic:*  Phase 3 — Stabilize and harden CLI / backend interfaces
