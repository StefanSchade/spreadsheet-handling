=== ADR-004 — Meta as Persistent and Mergeable Configuration

**Status:** Accepted

**Context**::
Meta information controls many behavioral aspects: styling, validations, compact mappings, and heuristics.
Initially, meta data was implicit in code or passed through the CLI configuration.
As complexity grew, a unified and persistable source of truth became necessary.

**Decision**::
- Meta is treated as a first-class entity (a structured dictionary) existing both in runtime and persisted artifacts.
- Missing meta is treated as an empty `{}` (fault-tolerant).
- Spreadsheet formats persist meta in a hidden sheet (`_meta`); JSON/YAML may embed it inline or as a sidecar file.
- Early in the pipeline, a dedicated transformation `bootstrap_meta` injects configuration defaults into meta.
- Merge priority: (1) Runtime Config → (2) Persisted Meta → (3) Heuristics/Defaults.

**Consequences**::
- Clear distinction between transient configuration (CLI) and persisted metadata (artifact).
- All backends tolerate missing meta gracefully.
- Deterministic merge behavior and unit-testable precedence rules.
- Provides a consistent foundation for style, validation, and transformation features.
