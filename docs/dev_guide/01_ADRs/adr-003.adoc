=== ADR-003 — Intermediate Representation (IR) Layer for Spreadsheet Rendering

**Status:** Accepted

**Context**::
Multiple spreadsheet adapters (XLSX, ODS) share similar logic for layout, styling, validation, and metadata persistence.
Duplicating this logic in every adapter would lead to divergence and maintenance overhead.

**Decision**::
- Introduce a shared *Intermediate Representation (IR)* layer responsible for rendering spreadsheets.
- The domain produces `frames + meta`; the IR layer handles layout, styling, validations, and meta persistence.
- Only spreadsheet-type backends (XLSX, ODS) use this layer; JSON/YAML backends bypass it entirely.

**Consequences**::
- Rendering logic is centralized and extensible via passes (StylePass, ValidationPass, FormulaPass, MetaPass, etc.).
- Spreadsheet adapters become smaller, simpler, and easier to maintain.
- Non-spreadsheet formats remain lightweight and unaffected by presentation concerns.
