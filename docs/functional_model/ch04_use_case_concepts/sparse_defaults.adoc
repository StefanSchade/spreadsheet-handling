== Use Case: Sparse Defaults In Editable Matrices

Sparse Defaults is a readability pattern for large editable matrices where most
cells carry the same default value. It lets the workbook show only meaningful
deviations while keeping the roundtrip back to the dense value model explicit.

It does not define matrix addressing. XRef Crosstable owns which tuple a matrix
cell represents. Sparse Defaults only answers how a configured default value is
shown, suppressed, and restored in the visible matrix.

=== User Problem

Large crosstables often contain a dominant value such as `nein`, `standard`,
`0`, or another configured default. Showing that value in every cell makes the
spreadsheet noisy and hides the few cells that need attention.

[source,text]
----
feature_id | Product A | Product B
f1         | nein      | ja
f2         | nein      | nein
----

For editing, the useful workbook view may be:

[source,text]
----
feature_id | Product A | Product B
f1         |           | ja
f2         |           |
----

Blank cells in this sparse view do not mean "unknown". They mean the configured
default value for the configured matrix columns.

=== Functional Boundary

Sparse Defaults delegates matrix shape and address meaning to XRef Crosstable.

* *XRef Crosstable* owns matrix projection and recomposition.
* *Sparse Defaults* owns default-value suppression and restoration for
  configured matrix cells.
* *Cell Codec* remains responsible for compact cell values if the non-default
  cell value itself is coded or tokenized.
* *Compact Multiaxis* may compose Sparse Defaults with XRef and Cell Codec as a
  smoother user surface, but must not reimplement sparse/default semantics.

=== Step: Collapse Defaults For Editing

The forward path replaces configured default values with a configured blank
value so editors see a sparse matrix.

==== Information Consumed

* *Matrix frame* -- essential; the matrix to make sparse.
* *Target matrix columns* -- essential; supplied explicitly or resolved from
  unambiguous XRef matrix metadata.
* *Default value* -- essential; the value that may be suppressed.
* *Blank value* -- optional; the visible sparse placeholder, usually empty.
* *Conflict policy* -- optional; determines what happens when pre-existing
  blank cells would become ambiguous.

==== Information Produced

* *Sparse matrix frame* -- same rows and columns, with configured default cells
  suppressed.
* *Sparse default intent* -- records the configured target columns, default
  value, blank value, and conflict policy so the reverse path can restore the
  dense values.

==== Semantic Model Reference

Registry anchors: `sparse_defaults`, `xref_crosstable`.

=== Step: Expand Sparse Blanks Back To Defaults

The reverse path fills configured blank cells with the configured default value
before relation recomposition or persistence.

==== Information Consumed

* *Sparse matrix frame* -- essential; the edited sparse workbook matrix.
* *Sparse default intent* -- essential unless the caller supplies the target
  columns, default value, and blank value explicitly.

==== Information Produced

* *Dense matrix frame* -- same shape, with sparse blanks restored to the
  configured default value.

==== Semantic Model Reference

Registry anchor: `sparse_defaults`.

=== Ambiguity Boundary

Sparse Defaults is reversible only when blank cells are reserved to mean the
configured default for the configured matrix columns. If the source data already
contains intentional blanks in those cells, collapse would make the roundtrip
ambiguous.

The safe behavior is explicit:

* require configured columns or unambiguous XRef metadata;
* reject or warn on pre-existing blank conflicts according to configuration;
* never apply sparse collapse to all frame columns by guessing;
* never infer defaults from visible shape or value frequency.

=== Non-Goals

Sparse Defaults does not:

* decide which tuple a cell addresses;
* add or remove matrix rows or columns;
* infer the dominant default automatically;
* interpret business meaning of the default value;
* replace XRef Crosstable, Cell Codec, or Compact Multiaxis.

Its value is ergonomic: the workbook becomes easier to scan and edit while the
dense default semantics remain explicit.
