=== Step-Slice Convention

Each feature chapter in this section follows the step-slice convention.
One slice per transformation step; slices are independent.

Each slice covers:

* one sentence on what the step does for the pipeline author;
* *Information Consumed* — conceptual name, essential or optional, what
  happens if absent;
* *Information Produced* — conceptual name plus applicable lifecycle
  attributes (`drop_at_persistence` / `drop_if_stale` /
  `carrier_takes_precedence` / `required` / `render_consumed`);
* *Semantic Model Reference* — names of corresponding meta registry /
  semantic model entries; no field paths or producer lists here.

Framework concepts shared across all slices (meta carriers, Type 1 /
Type 2 events, lifecycle attribute model) are in
`ch05_problem_space_overview/03_meta_lifecycle_framework.adoc`.
