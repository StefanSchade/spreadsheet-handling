== Operating Model

The project supports workflows where structured data must remain editable by humans.

[source,text]
----
canonical structured data
  -> spreadsheet projection
  -> human review / edit / enrichment
  -> deterministic re-import
  -> structured downstream use
----

Core relations:

* Canonical data is the stable semantic source.
* Spreadsheet projections are human-facing views.
* Human edits happen in the projection.
* Re-import reconstructs canonical data from the edited projection.
* Downstream systems consume structured data, not spreadsheet conventions.

Design pressure:

* Human workflows need flexibility and dense visual editing.
* Software systems need explicit structure and deterministic behavior.
* The project keeps both sides connected without making the spreadsheet the model.
