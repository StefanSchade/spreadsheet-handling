=== Formatting Pipeline: Styling, Freeze, Validations

The formatting features — header colours, frozen rows, auto-filter, column
drop-downs — are applied through a **pipeline of domain steps** rather than
hard-coded into the exporter.  This section walks through a complete example
that exercises all current formatting capabilities.

The demo files live under `interactive_test/smoke_formatting/`.

==== How it works

[source]
----
JSON files (input data)
    │
    ▼
[ bootstrap_meta ]   - set workbook-wide defaults
    │
    ▼
[ apply_overrides ]  - load per-sheet config from YAML (colours, freeze)
    │
    ▼
[ add_validations ]  - attach dropdown rules to specific columns
    │
    ▼
XLSX (IR-backed renderer with styling, freeze, auto-filter, validations)
----

Formatting metadata lives **only in the YAML files** — the JSON input contains
pure data.  On the return path (XLSX → JSON) only cell values are extracted;
the formatting is not yet persisted (tracked as _FTR-META-PERSISTENCE_).

==== Per-sheet overrides

The overrides file defines workbook-wide defaults and per-sheet overrides.
Here each sheet gets its own header colour and frozen first row.

.`interactive_test/smoke_formatting/overrides.yaml`
[source,yaml]
----
defaults:
  auto_filter: true
  freeze_header: false
  header_fill_rgb: "#4472C4"   # fallback blue (overridden per sheet below)
  helper_fill_rgb: "#FFF2CC"   # light yellow for helper columns
  helper_prefix: "_"

sheets:
  Products:
    freeze_header: true
    header_fill_rgb: "#2E7D32"   # dark green
  Orders:
    freeze_header: true
    header_fill_rgb: "#C62828"   # dark red
----

==== Pipeline configuration

The main config ties together I/O paths, the three pipeline steps, and the
dropdown validation rules.

.`interactive_test/smoke_formatting/demo_config.yaml`
[source,yaml]
----
io:
  input:
    kind: json_dir
    path: ./interactive_test/smoke_formatting/in
  output:
    kind: xlsx
    path: ./tmp/smoke_formatting.xlsx

pipeline:
  - step: bootstrap_meta
    profile_defaults:
      auto_filter: true
      freeze_header: false
      helper_prefix: "_"

  - step: apply_overrides
    overrides_path: ./interactive_test/smoke_formatting/overrides.yaml

  - step: add_validations
    rules:
      - sheet: Products
        column: status
        rule:
          type: in_list
          values: [Active, Discontinued, Pending]
      - sheet: Products
        column: category
        rule:
          type: in_list
          values: [Electronics, Clothing, Books, Other]
      - sheet: Orders
        column: priority
        rule:
          type: in_list
          values: [Low, Medium, High, Urgent]
----

==== Running the demo

[source,powershell]
----
cd <project-root>
$env:PYTHONPATH    = "src"
$env:SH_XLSX_BACKEND = "ir"          # <1>

.venv_win\Scripts\python -m spreadsheet_handling.cli.apps.run `
    --config interactive_test\smoke_formatting\demo_config.yaml `
    -vv
----
<1> The `SH_XLSX_BACKEND=ir` flag activates the IR-based renderer, which
applies all formatting ops.  Without it the legacy pandas renderer is used
and formatting is ignored.

The output is written to `tmp/smoke_formatting.xlsx`.

==== Precedence of formatting settings

Settings are merged left-to-right; later values win:

[source]
----
bootstrap_meta (workbook defaults)
    ↓  overridden by
apply_overrides defaults: section
    ↓  overridden by
apply_overrides sheets.<name>: section
----

For example, `freeze_header` is `false` in `bootstrap_meta` but `true` under
`sheets.Products` — so the Products sheet is frozen.

==== Current limitations

* **Not roundtrip-safe for formatting.**
  Unpacking `tmp/smoke_formatting.xlsx` back to JSON yields the cell values
  only — colours, freeze settings, and validation rules are not recovered.
  Tracked as _FTR-META-PERSISTENCE_ (Phase 3).

* **`SH_XLSX_BACKEND=ir` required.**
  Without the env-var the legacy renderer is used, which ignores all
  IR-based formatting.  Making the IR backend the default is tracked as
  _FTR-PARITY-IR-P3_ / _FTR-CLEANUP-IR-P4_ (Phase 3).

* **Data cell writing is a two-step process.**
  Currently the IR renderer writes cell data via pandas, then overlays
  formatting in a second pass.  Full single-pass IR rendering (including data
  cells) is tracked as _FTR-IR-DATA-CELLS_ (Phase 3).
