=== Compact Transform Pipelines

Compact transform steps convert matrix-shaped workbook data into explicit
relation rows and back. They stay generic: row identity, column axes, cell
codes, and optional code groups are configuration, not inferred business
meaning.

==== Identity and Display Labels

`row_keys` define identity. A display label roundtrips only when it is part of
`row_keys` or carried by another explicit step.

[source,yaml]
----
- step: expand_compact_multiaxis
  matrix: product_matrix
  output: feature_product_codes
  row_keys: [feature_id, feature_label]
  value_columns: [P-001, P-002]
  mode: whole_cell_code
  allowed_from_legend: status_codes
----

If `feature_label` is omitted from `row_keys`, the transform does not recreate it
from a visible label, similar column name, or sheet position.

==== Sparse Versus Strict Shape

`drop_empty: true` is sparse. Empty compact cells produce no explicit rows, so
empty-only row or column groups can disappear on contraction.

[source,yaml]
----
- step: expand_compact_multiaxis
  matrix: product_matrix
  output: feature_product_codes
  row_keys: [feature_id]
  value_columns: [P-001, P-002]
  mode: whole_cell_code
  allowed_from_legend: status_codes
  drop_empty: true
----

Use `drop_empty: false` when empty cells must survive a strict matrix-shape
roundtrip.

[source,yaml]
----
- step: expand_compact_multiaxis
  matrix: product_matrix
  output: feature_product_codes
  row_keys: [feature_id]
  value_columns: [P-001, P-002]
  mode: whole_cell_code
  allowed_from_legend: status_codes
  drop_empty: false
----

`dense_axes` is reserved for a future explicit reconstruction contract. Until it
exists, no transform reconstructs missing rows or columns heuristically.

==== Cell Codes

`whole_cell_code` treats a value such as `E-R-K` as one code. Punctuation alone
does not imply splitting.

[source,yaml]
----
- step: expand_compact_multiaxis
  matrix: product_matrix
  output: feature_product_codes
  row_keys: [feature_id]
  value_columns: [P-001]
  mode: whole_cell_code
  allowed_from_legend: status_codes
----

`split_tokens` splits only when explicitly configured with a delimiter.

[source,yaml]
----
- step: expand_compact_multiaxis
  matrix: product_matrix
  output: feature_product_tokens
  row_keys: [feature_id]
  value_columns: [P-001]
  mode: split_tokens
  delimiter: "-"
  allowed_tokens: [E, R, K]
----

==== Legend Groups

`group` is a generic projection from legend/codebook metadata or explicit
`code_groups`. It does not add business semantics to the compact transform
layer.

[source,yaml]
----
- step: expand_compact_multiaxis
  matrix: product_matrix
  output: feature_product_codes
  row_keys: [feature_id]
  value_columns: [P-001, P-002]
  mode: whole_cell_code
  allowed_from_legend: status_codes
  group: group
----

==== Workbook Routes

The maintained `sheets-run` path can route compact-transform flows to XLSX and
ODS/Calc outputs by configuring the output kind.

[source,yaml]
----
io:
  input: {kind: json_dir, path: data/in}
  output: {kind: calc, path: build/product_matrix.ods}
pipeline:
  - step: expand_compact_multiaxis
    matrix: product_matrix
    output: feature_product_codes
    row_keys: [feature_id]
    value_columns: [P-001, P-002]
    mode: whole_cell_code
    allowed_from_legend: status_codes
----
