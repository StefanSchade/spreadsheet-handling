=== The Core Idea

A pipeline transforms data bidirectionally between a normalized relation form
(YAML-friendly) and a human-editable spreadsheet. The same configuration drives
both directions, so re-import is deterministic.

Practical workflows tend to grow in rings: start with a plain tabular roundtrip
(data in → spreadsheet out → back in), then add FK helper columns for display,
then introduce cross-table or compact-cell encoding for multi-valued relations.
Each ring adds one transform pair from the <<_transformation_catalog,catalog>>.

A runnable walkthrough of this spiral is available in the
link:{demo-url}[spreadsheet-handling-demo repository].

=== Pipeline YAML Structure

A pipeline is a YAML list of named steps. Each step names a registered
transform and supplies its parameters inline:

[source,yaml]
----
pipeline:
  - name: expand_xref
    params:
      matrix: role_permissions
      output: permission_rows
      row_keys: role_id
      drop_empty: true
  - name: write_structured_yaml
    params:
      source: permission_rows
      output_path: out/permissions.yaml
----

Available step names, required parameters, and roundtrip guarantees are listed
in the <<_transformation_catalog,Transformation Catalog>>.
