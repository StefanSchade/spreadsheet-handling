This guide is built from the core repository because it describes the framework
surface of a specific `spreadsheet-handling` release. The main user entry point
is still the demo repository, which links to the published versioned
documentation.

== What This Project Does

* Provides a framework for spreadsheet-oriented data workflows.
* Projects structured data into spreadsheet forms for human review and editing.
* Keeps semantics explicit instead of relying on spreadsheet conventions alone.
* Supports deterministic re-import into structured downstream data.

== Where Users Should Start

* Main user-facing repository: link:https://github.com/StefanSchade/spreadsheet-handling-demo[spreadsheet-handling-demo].
* Published documentation: planned GitHub Pages output built from this repo and
  the matching demo release.
* This repository is the source for framework documentation and release-bound
  user-guide artifacts.

== What Is Not Here

* Demo data, runnable scripts, and integration-flow walkthrough sources.
* Generated website output.
* These belong in the demo repository or the Pages artifact repository,
  depending on whether they are source or generated output.
