=== Quickstart: Integrating in your own repository

When a set of transformations becomes part of your regular workflow,
it is convenient to **collect them in a single configuration file**
and keep it alongside your project data.

This section shows how to integrate *Spreadsheet-Handling*
into your own repository setup.

---

==== 1. Create a configuration file

In your project, add a file such as `configs/sheets.yaml`:

```yaml
io:
  profiles:
    verify:
      pipeline: verify_steps
      input:  { kind: json, path: data/input }
      output: { kind: xlsx, path: build/verify.xlsx }

    extract:
      pipeline: extract_steps
      input:  { kind: xlsx, path: build/verify.xlsx }
      output: { kind: json, path: build/extracted }

pipelines:
  verify_steps:
    - step: validate
    - step: add_helper_columns
    - step: export

  extract_steps:
    - step: filter_rows
      args: { column: status, keep: ["active"] }
    - step: export
```

* `io.profiles` defines named presets for input/output paths and default pipelines.
* `pipelines` lists reusable step sequences referenced by name.

==== 2. Run one of the profiles

To execute one of the workflows defined in the configuration, use:

----
sheets-run --config configs/sheets.yaml --profile verify -v
----

or explicitly select a pipeline:

----
sheets-run --config configs/sheets.yaml --pipeline extract_steps -v
----

Both commands use the same engine as before;
only the way you describe the workflow differs.

==== 3. Benefits of the integrated approach

* All transformations for the repository live in one place.
* Profiles can reference different pipelines without duplicating I/O settings.
* Team members can reuse and modify workflows without touching code.
* CI/CD pipelines or Make targets can trigger named profiles directly.

==== 4. Example repository

A full example is available on https://github.com/StefanSchade/spreadsheet-handling-demo[github]

The demo repository demonstrates:

* how to structure folders for input, output, and configs,
* how to define multiple pipelines,

and how to run them via Make targets such as `make run-profile PROFILE=verify`.

You can use it as a template for integrating Spreadsheet-Handling
into your own data or project repositories.
