=== User-Facing Surfaces

Spreadsheet-Handling deliberately keeps its public surface small while the
project is pre-beta.

==== CLI entry points

* `sheets-run` is the generic main entry point.
* `sheets-example-json-to-xlsx` and `sheets-example-xlsx-to-json` are reference
  shortcut commands demonstrating the supported wrapper pattern.

Shortcut commands are an intended pattern from which real product surfaces may
later be built. The current example shortcut commands are examples/reference
implementations, not the central product UX. The project is pre-beta; removing
the misleading historical `pack`/`unpack` names is intentional and preferable
to carrying compatibility debt.

==== YAML pipeline/config surface

The YAML surface includes:

* registered step names
* step parameter shapes
* `io`, `pipeline`, `pipelines`, and `profiles`

==== Dotted callable extension surface

Plugin steps may call user-provided Python functions via dotted paths, and
registry entries may bind to dotted callables internally. This protocol is part
of the intentional extension surface.

==== Python import surface

The Python import surface is intentionally small. Do not treat every internal
module or barrel export as public API. Prefer `orchestrate(...)`, CLI mains,
and documented pipeline builders as intentional imports.
