=== Quickstart: Reference Shortcut Commands

The fastest way to try *Spreadsheet-Handling* without authoring YAML is through
the reference shortcut commands. These small command-line tools ship with the
package and demonstrate the supported wrapper pattern for common conversions.

`sheets-run` remains the generic main entry point. Shortcut commands are not
deprecated; they are intentionally thin examples of how product-specific
commands can be built on top of `orchestrate()`.

Each shortcut combines:

* predefined input and output formats (for example: JSON -> Excel)
* an optional fixed sequence of pipeline steps
* a few simple command-line options to specify paths

==== Available shortcut examples

[cols="1,4,4"]
|===
| Command | Description | Default flow

| `sheets-example-json-to-xlsx`
| Converts a folder of JSON tables into an Excel workbook as a reference shortcut.
| `json_dir -> [ bootstrap_meta ] -> xlsx`

| `sheets-example-xlsx-to-json`
| Splits an Excel workbook into a JSON directory as a reference shortcut.
| `xlsx -> json_dir`
|===

These commands are installed automatically when you install the package via pip.

==== Example: JSON into Excel

[source,bash]
----
sheets-example-json-to-xlsx \
  ./data/json \
  -o ./build/out.xlsx
----

This reads JSON files from `./data/json`, applies the example default workbook
metadata, and writes them into an Excel workbook at `./build/out.xlsx`.

==== Example: Excel into JSON

[source,bash]
----
sheets-example-xlsx-to-json \
  ./build/out.xlsx \
  -o ./data/json_unpacked
----

This performs the reverse operation: it reads the Excel workbook and exports
each sheet as a JSON file inside `./data/json_unpacked`.

==== When to use shortcut examples

Use the examples when you:

* want a quick conversion between formats,
* want to verify that the package works correctly in your environment,
* or want a compact example of the thin shortcut pattern without defining your own YAML.

Once you need to adjust or extend transformation logic, move on to the next
section and create an ad-hoc steps file, or use `sheets-run` with a reusable
configuration.
