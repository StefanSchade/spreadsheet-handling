=== Quickstart: Using the CLI shims

The fastest way to try *Spreadsheet-Handling* is through the **predefined CLI shims**.
These are small command-line tools that ship with the package and expose
ready-made pipelines for common conversions — no YAML configuration required.

Each shim combines:

* predefined **input and output formats** (for example: JSON → Excel)
* a fixed **sequence of pipeline steps** that performs validation and export
* a few simple command-line options to specify paths and verbosity



==== Available shims

[cols="1,4,4"]
|===
| Shim | Description | Default flow

| `sheets-pack`
| Converts a folder of JSON tables into an Excel workbook.
| `json_dir → [ validate → export_xlsx ] → xlsx`

| `sheets-unpack`
| Splits an Excel workbook into a JSON directory.
| `xlsx → [ parse → export_json_dir ] → json_dir`
|===
These commands are installed automatically when you install the package via pip.

==== Example: pack JSON into Excel

```bash
sheets-pack \
  --in ./data/json \
  --out ./build/out.xlsx \
  -v
```

This reads all JSON files from `./data/json`,
validates their structure,
and writes them into an Excel workbook at `./build/out.xlsx`.

`-v` (or `-vv`) increases logging verbosity;
`--debug` shows full tracebacks if anything goes wrong.

==== Example: unpack Excel into JSON

----
sheets-unpack \
--in ./build/out.xlsx \
--out ./data/json_unpacked \
-v
----

This performs the reverse operation:
it reads the Excel workbook and exports each sheet
as a JSON file inside `./data/json_unpacked`.

==== When to use shims

Use the shims whenever you:

* just need a quick conversion between formats,
* want to verify that the package works correctly in your environment,
* or prefer a one-liner without defining your own YAML.

Once you need to adjust or extend the transformation logic
—for example, to add enrichment or filtering steps—
move on to the next section and create an ad-hoc steps file.
