== Project Context

=== Identity and Scope

* Python library: converts JSON/CSV ↔ XLSX/ODS while preserving FK relationships, indexes, and hierarchies
* Package name: `spreadsheet-handling`, published on PyPI under MIT (beta)
* Related repo: `spreadsheet-handling-demo` — consumer only; do not develop core logic there
* Repository responsibilities and publishing boundaries (core / demo / future `spreadsheet-handling-pages`): see `repository_boundaries.adoc`
* Primary dev environment: headless Debian VM (Linux); avoid WSL-specific assumptions in repo-facing code
* GitHub (public) · GitLab CI (company pipeline)

=== Technology Stack

* Python >= 3.10 · `setuptools` + `setuptools-scm` (no Hatchling)
* Tooling: `make`, `ruff`, `black`, `pytest`, `pytest-cov`, `compileall`
* CI pipeline stages: lint → syntax → tests (+coverage) → build sdist/wheel → publish on tags
* Docs: AsciiDoc; chat language: English or German; all repo-facing code and comments: English

=== AI Use Policy

* AI is used at all levels: requirements, design, coding
* Human oversight and review are mandatory — AI augments but does not replace engineering judgment
* The repo is AI-friendly by design: context files exist so agents can continue work in a new context window

=== Practices

* AI-generated artifacts are reviewed, version-controlled, and tested before acceptance
* Planned changes to outward-facing interfaces must be stated before implementation — not as a surprise in a PR
* Changes affecting the demo repo, YAML schemas, CLI usage, or plugin conventions require explicit user approval
