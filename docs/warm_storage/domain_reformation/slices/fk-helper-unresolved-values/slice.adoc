= Reformation Slice: Fk Helper Unresolved Values
:revdate: 2026-06
:slice-name: fk-helper-unresolved-values

== Status

Draft. First contract-decision slice for the domain/meta reformation workflow.

== Intent

Define the contract for unresolved helper values in FK-helper / helper-column
workbook roundtrips.

The user-facing requirement is narrow: workbook projections may show readable
helper context next to a key, but unresolved helper values must not silently
become ordinary domain strings such as `"nan"` or `""` without a documented
meaning.

This slice exists because `0.2.1` intentionally shipped with one accepted
xfail sentinel. That sentinel should become the first contract-first
reformation slice rather than a quick patch inside the old helper sediment.

== Bridge DSL Contract

Decision pending. The slice should define the smallest `Frames + Meta`
contract needed to represent:

* a canonical source frame with a key or relation-bearing column;
* a helper projection that shows readable context for that key;
* an unresolved helper state when the key has no resolvable target row;
* the reverse path back to canonical frames.

The contract must answer:

* whether unresolved helper values are represented inline;
* whether they are represented as `null`, empty string, a typed unresolved
  token, or a separate finding;
* whether the representation is durable payload, transient workbook support, or
  an error/finding carrier;
* how a no-cleanup diagnostic path differs from the normal cleanup path.

== Lifecycle Semantics

Initial hypothesis:

* helper columns are workbook editing support, not canonical payload;
* helper projection intent is durable enough to regenerate the workbook view;
* materialized helper cell values are transient unless the contract explicitly
  promotes unresolved state into a finding or diagnostic frame;
* normal reimport cleanup should drop helper columns before canonical
  persistence;
* the no-cleanup sentinel path is diagnostic and keeps the helper value visible
  only to force the unresolved-value contract decision.

The slice must avoid promoting `helper lifecycle` as a new durable concept.
That term was already classified as sediment and moved out of the active
Functional Model backlog.

== Contract Tests

Current sentinel:

* `tests/roundtrip/test_pathological_cell_contracts.py`
* `test_missing_fk_key_helper_representation_without_cleanup`

Target change:

* convert the sentinel from accepted xfail to a green contract test after the
  representation decision is made;
* add only the minimum additional tests needed to separate the normal cleanup
  path from the diagnostic no-cleanup path.

The test must express the functional requirement and must not freeze the old
implementation shape.

== Implementation Shape

Pending the contract decision.

Likely implementation shape:

* introduce a small vNext helper-value normalization / representation boundary;
* keep it independent from the old `enrich_lookup` / `derived_column_policy`
  split where possible;
* route the no-cleanup diagnostic path through the same representation rule;
* leave broader FK-helper rewrite work for later slices.

== Old-Path Handling

For this first slice, do not delete broad old FK-helper paths.

Allowed old-path handling:

* delegate a narrow unresolved-value representation point to the new contract
  helper;
* leave unrelated enrichment, formula, cleanup, and provenance paths untouched;
* add comments only if they prevent accidental widening.

Deletion should wait until a later slice has coverage for the surrounding
helper-column workflow.

== Acceptance

* The unresolved-helper-value representation is decided in Functional
  Model-oriented language.
* The xfail sentinel becomes green or is replaced by an equivalent green
  contract test.
* The normal cleanup path still drops helper columns from canonical output.
* No general metadata taxonomy is added.
* No new Project Memory entity hierarchy is created.
* Any durable decision is linked from backlog/PM/ADR only if it needs future
  discoverability beyond this slice.
