= Reformation Slice Tooling How-To
:revdate: 2026-06-24

== Purpose

This is the operational note for the lightweight domain reformation slice
tooling.

The helper is intentionally repo-local and lives in `scripts/`, not `tools/`.
`tools/` is for generic, copyable helper scripts. The reformation helper knows
the core repository's warm-storage layout and is not meant as a cross-repo
utility.

== Create A Slice

[source,bash]
----
make reformation-slice NAME=fk-helper-unresolved-values
----

The target creates:

* `docs/warm_storage/domain_reformation/slices/<name>/slice.adoc`
* an include entry in
  `docs/warm_storage/domain_reformation/slices/_slices.adoc`

Slice names must use lowercase letters, numbers, and hyphens.

== Check A Slice

[source,bash]
----
make reformation-check SLICE=fk-helper-unresolved-values
----

The check verifies that the slice document exists, contains the required
sections, and renders with Asciidoctor when Asciidoctor is available.

Required sections:

* `Intent`
* `Functional Model Source`
* `Bridge DSL Contract`
* `Lifecycle Semantics`
* `Contract Tests`
* `Implementation Shape`
* `Old-Path Handling`
* `Acceptance`
* `Memory / Backlog Impact`
* `Verification Command`

== Print A Driver Prompt

[source,bash]
----
make reformation-driver SLICE=fk-helper-unresolved-values
----

The driver prints a reusable agent prompt for the named slice. It does not
create files, decide the contract, update Project Memory, create backlog or ADR
material, promote semantic vocabulary, delete old paths, or start an
implementation.

Optional source and test hints can reduce maintainer prompt writing:

[source,bash]
----
make reformation-driver \
  SLICE=cell-codec-frames-meta-bridge \
  SOURCES="docs/functional_model/ch04_use_case_concepts/cell_codec.adoc docs/cold_storage/working_context/functional_model_inquiry/cell_codec/cell_codec.adoc" \
  TEST_HINTS="tests/unit/domain/transformations/test_cell_codec.py"
----

== Editing Rule

The tooling scaffolds structure and prints orchestration prompts only. It does
not decide the functional contract, create Project Memory entries, create FTRs,
delete old implementation paths, or promote new semantic vocabulary.

Those decisions remain part of the slice work.

== Current First Slice

[source,bash]
----
make reformation-check SLICE=fk-helper-unresolved-values
make test-file FILE=tests/roundtrip/test_pathological_cell_contracts.py
----

The first command validates the slice note. The second command captures the
current accepted sentinel state for the unresolved-helper-value contract.
