Das Helper-Beispiel spricht daher nicht für eine einzelne allgemeingültige
Metadatenregel. Es zeigt vielmehr, dass jede Informationssorte an konkreten
Prozessgrenzen bewertet werden muss.

Eine Helper-Information kann während des Roundtrips unverzichtbar sein, nach
erfolgreicher Rückführung aber bereinigt werden. Sie kann lokal persistiert
werden, ohne für andere Nutzer verfügbar zu sein. Sie kann durch Bootstrap
rekonstruiert werden, wenn ihre Grundlage in der Konfiguration liegt. Sie kann
aber auch zu fachlichem Drift führen, wenn sie auf dem Rückweg nicht erkannt
oder nicht bereinigt wird.

Die funktionale Frage lautet daher nicht:

### [source,text]

## Welche technische Metadatenklasse hat diese Information?

Sondern zuerst:

## [source,text]

Erst aus diesen Antworten sollte später eine technische oder semantische
Klassifikation abgeleitet werden.
