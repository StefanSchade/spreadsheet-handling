=== ADR-005 — Dual Test Harness (Legacy vs. IR)

**Status:** Accepted

**Context**::
During the migration from the legacy XLSX backend to the new IR-based renderer, both variants must coexist safely within CI pipelines.

**Decision**::
- Introduce two explicit Pytest markers:
* `@pytest.mark.xlsx_ir` — tests that target the new IR-based backend.
* `@pytest.mark.xlsx_legacy` — tests that validate the legacy behavior.
- Extend Makefile with dedicated targets:
* `make test` — default suite (excludes both legacy and IR-only tests).
* `make test-ir` — runs IR-specific tests (`-m xlsx_ir`, `--ignore=tests/legacy`).
* `make test-legacy` — runs legacy tests (`-m xlsx_legacy`).
- Add environment variable `SH_XLSX_BACKEND` to control the active backend (`legacy` | `ir` | `auto`).

**Consequences**::
- Safe parallel execution of both backends with isolated markers.
- CI/CD pipelines remain stable throughout migration.
- Developers can incrementally retag tests once feature parity is reached.
- Simplifies rollout, regression testing, and code review processes.
