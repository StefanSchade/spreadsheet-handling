[[concept-roundtrip-spreadsheet-layout]]
== Round-trip Strategy for Spreadsheet Layouts

_A predictable contract for authoring, editing, and re-importing spreadsheet
workbooks: round-trip is guaranteed for data + metadata (style is normalized),
and tables use one of two clear layout modes — flat anchored tables or named
ranges._

Status: contract concept. We target our own exported structures; arbitrary
third-party workbooks are best-effort only. The in-flight implementation details
that used to live here (the multi-header "pandas-first then IR-overlay" hybrid,
the roadmap position relative to other FTRs) have moved to the backlog —
see `docs/general_info/backlog/REVIEW-roundtrip-and-step-registration-transitional-notes.adoc`.
The render-chain *decisions* live in `docs/human_info/adr/adr-006_render_chain.adoc`
and `docs/human_info/adr/ADR-RENDER-FLOW-VS-DOMAIN-PIPELINE.adoc`.

=== Why this matters

If users edit exported workbooks and re-import them, the system needs a stable,
documented contract for what survives an edit and what does not — otherwise
every user edit is a gamble and every re-import is a debugging session. This
concept defines that contract.

=== The contract

* Round-trip is guaranteed for *data + metadata*. *Style is normalized* — we do
  not promise pixel-faithful style round-trip.
* Two layout modes:
  ** *Scenario A — Flat tables*: a table starts at an anchor cell and expands
     until a sentinel empty row/column.
  ** *Scenario B — Named ranges*: each table is a named range; multiple tables
     per sheet are supported and safe.

Why named ranges? They are supported in Excel, LibreOffice/ODS, and Google
Sheets, and they provide stable, tool-readable boundaries that survive most user
edits.

=== Embedded metadata

The writer emits a hidden `_meta` sheet (or a named range `sh_meta`) carrying
JSON/YAML that describes the workbook version, the tables (id, sheet, start
cell, named range if any), and optional typing/keys/validation descriptors. The
reader (`parse_workbook()` in `io_backends/xlsx/openpyxl_parser.py`) prioritizes
embedded meta when reconstructing the IR.

Bootstrap without embedded meta: a CLI or YAML pipeline can declare anchors and
options (Scenario A) or named-range mappings (Scenario B); a sidecar file
`*.xlsx.meta.json` can carry the same info for batch imports.

=== Editing rules (for users)

* You may append rows and insert columns inside table bounds.
* You may modify cell values and keep formula columns.
* Avoid merges across header rows unless covered by the multi-header feature.
* For multiple tables on one sheet, use named ranges.

=== Concrete example and test strategy

Golden fixture: render → edit (scripted) → parse → compare IR (data + shapes).
Negative tests demonstrate why certain edits invalidate round-trip and document
them as not-supported. A flat-table workbook with one anchored table and a
named-range workbook with two tables on one sheet are the two canonical
fixtures.
