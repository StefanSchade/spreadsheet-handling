= Domain Contracts Registry

This directory contains the first structured domain-contract registry artifact.
It complements `../meta_registry.yaml`; it does not replace that operational
metadata registry.

Transformation contracts are the anchor. The seed tables connect requirements,
domain concepts, transformations, frame/meta/config IO, links, and rules in
flat JSON tables that can roundtrip through spreadsheet review workflows.

This first slice is infrastructure and deliberately small seed data. It is not
full transformation coverage and it is not a code fitness analysis.

Three tables carry the Review 005 Slice 1 white-box model of column schema
maintenance:

* `transformation_features.json` - suboperations of one transformation
  (currently the four column-maintenance operations).
* `meta_reference_surfaces.json` - per-`_meta`-root/path policies for column
  rename/drop reference propagation (supported/blocked/ignored/reported),
  mirrored from `domain/schema_maintenance/meta_refs.py` and guarded by
  `tests/architecture/semantic_invariants/test_meta_reference_surfaces.py`:
  changing either side without the other fails the suite.
* `implementation_gap_findings.json` - explicit, checker-validated gap
  records (e.g. meta paths that escape rename propagation).

== Review Loop Safety

The spreadsheet review loop is:

----
make domain-contracts-export    # canonical JSON -> domain_contracts.ods (+ export stamp)
# edit the workbook in LibreOffice Calc
make domain-contracts-import    # ODS -> staging/ (carries the export stamp)
make domain-contracts-diff-reimport
make domain-contracts-promote   # gated copy staging -> canonical
----

Three gates protect canonical from a stale, foreign, or invalid staging
snapshot (Review 005, FIN-REVIEW-005-P1-2 and follow-up Slice 1b):

* *Freshness*: `domain-contracts-export` writes a stamp of the canonical
  tables next to the workbook (`domain_contracts.ods.stamp.json`, gitignored);
  `domain-contracts-import` copies it into `staging/.export_stamp.json`.
  `domain-contracts-promote` refuses to copy when canonical changed after the
  export that produced the staging snapshot. Re-run the loop from export in
  that case.
* *Workbook binding*: the stamp also records a SHA-256 of the exported
  workbook, and the export embeds the canonical fingerprint into the
  workbook's `_meta` (consumed on reimport). `domain-contracts-import`
  refuses a workbook that neither matches the stamped bytes (unedited case)
  nor carries the matching embedded fingerprint (edited-in-LibreOffice
  case) - a stale copy or foreign workbook cannot be imported against a
  fresh stamp. Old-format stamps are refused; re-run export.
* *Validity*: `domain-contracts-promote` runs
  `tools/domain_contracts/check_contracts.py` against `staging/` and refuses
  to copy when the checker reports errors. The checker also rejects
  ungoverned top-level `*.json` table files (`unexpected_table`), so a rogue
  staging table cannot ride the promote copy into canonical.

The checker requires each table row to carry exactly the fields declared in
`tools/domain_contracts/check_contracts.py`. A canonical schema change must
therefore land together with the matching checker (and, where needed,
workbook-normalizer) change, or `make domain-contracts-check` and the
domain-contracts integration tests fail.

The ODS/staging roundtrip writes every column on every row, so canonical
tables are kept in that homogeneous form: optional-per-row information is
carried as an empty string, never as an absent field.
