= Integration Test Slices

Integration tests cover real product paths across multiple layers. They should
not carry import-direction checks, architecture boundary assertions, or durable
semantic-invariant policy; those belong under `tests/architecture/`.

== Current Slices

[cols="2,4"]
|===
| Slice | Purpose

| `io_backends/json/`
| JSON directory writer/reader filesystem roundtrips.

| `io_backends/yaml/`
| YAML directory writer/reader filesystem roundtrips.

| `io_backends/xlsx/`
| XLSX backend and IR render-flow product paths.

| `io_backends/ods/`
| ODS backend factory, writer, reader, and parser product paths.

| `io_backends/test_ods_xlsx_parity.py`
| Cross-adapter XLSX/ODS parity for portable spreadsheet behavior.

| `roundtrip/`
| Multi-step backend roundtrips and parser read-back product scenarios.

| `spreadsheet_contract/`
| Spreadsheet read-path integration through parser, projection, and backend APIs.

| `pipeline/`
| Application orchestrator, CLI run path, and pipeline runner scenarios using real IO.
|===

== Consolidation Rules

* Keep one representative test per product path unless the variation protects a
  distinct user-visible behavior.
* Prefer parametrization over near-duplicate modules.
* Move architecture assertions into `tests/architecture/`.
* Empty placeholder modules are not retained as integration slices.
