= Test Suite Topology

The active suite is organized by responsibility first and implementation detail second.
When adding a new test, choose the directory that best describes the behavior or boundary it protects.

== Active Layers

* `tests/unit/`: small-scope behavior, helpers, local module collaboration
* `tests/integration/`: real product paths across multiple layers, adapters, or roundtrips
* `tests/architecture/`: dependency guards, contract seams, and durable semantic invariants
* `tests/legacy_pre_hex/`: quarantined historical material, not part of the default active suite

Supporting fixtures and helpers stay under `tests/data/` and `tests/utils/`.

== Placement Guide

* Put pure helper logic under `tests/unit/`, even when the helper lives in a backend package.
* Put real parser, renderer, adapter, CLI-with-IO, or roundtrip paths under `tests/integration/`.
* Put import-direction checks, AST guards, contract seams, and long-lived semantic rules under `tests/architecture/`.
* Do not place historical pre-hex tests into the active tree until they are explicitly promoted or rewritten.

== Common Examples

* Formula translator helpers: `tests/unit/rendering/` or `tests/unit/io_backends/`
* `read_spreadsheet_frames(...)` and parser-boundary product paths: `tests/integration/spreadsheet_contract/`
* JSON/XLSX/ODS roundtrips: `tests/integration/roundtrip/` or `tests/integration/io_backends/`
* Backend import guards or adapter-boundary seams: `tests/architecture/dependency_guards/`
* Meta or carrier invariants: `tests/architecture/semantic_invariants/`

== Guardrails

`tests/architecture/` is for boundary and invariant protection, not a dumping ground for awkward end-to-end tests.
If a test mainly proves user-visible behavior through a real path, it belongs in `integration`, even if the motivating bug was architectural.

`tests/legacy_pre_hex/` remains quarantined on purpose.
Classify those tests deliberately as `promote`, `rewrite`, or `retire` in follow-up work instead of mixing them back into the active suite.
