= Minimal FK Dataset (roundtrip-layer fixture)

Smallest dataset that exercises the roundtrip invariants documented in
`FTR-ROUNDTRIP-TEST-LAYER-P4A`. Owned by the roundtrip test layer; not used
elsewhere.

== Shape

* `entities` -- target frame, two rows. Columns: `id`, `name`.
* `items` -- source frame, three rows. Columns: `id`, `name`, `entity_id`.
* `items.entity_id` -> `entities.id` is the FK relation.
* `ITEM-003` has `entity_id: ""` -- the deliberate unresolved-FK row.

== Why a separate fixture instead of reusing existing test data

The shared `frames_minimal` fixture from `tests/conftest.py` lacks the
empty-FK row and a workbook-view spec. The roundtrip layer needs the
*full* canonical -> workbook -> canonical cycle including the
configure_fk_helpers + add_fk_helpers + configure_workbook_view chain.
Reusing the shared fixture would either omit the helper-related coverage
or pollute the shared fixture for unit/integration callers.

The dataset stays in JSON so it is human-readable and round-trippable
through every backend without format conversion. Pipelines are
generated per-test rather than checked in as YAML, so the fixture's
public surface is the data, not the pipeline shape.
